## 1. 初始化流程

### 1.1 流程目标

应用启动后，`terminal-data` 模块应完成以下初始化任务：

1. **加载本地缓存**：恢复终端身份、凭证、绑定上下文
2. **设备信息采集**：获取设备 ID、指纹、型号等信息
3. **凭证刷新**：如果 token 即将过期，自动刷新
4. **Bootstrap 同步**：从 TCP 获取最新的配置、版本、Topic 定义
5. **TDP 会话建立**：连接 TDP WebSocket，建立实时通道
6. **Projection 同步**：同步所有 Topic 的快照数据
7. **Change 追赶**：追赶离线期间的增量变更
8. **派生树构建**：基于绑定上下文构建组织树视图
9. **运行时状态初始化**：初始化期望配置、期望版本、收件箱

### 1.3 代码实现

#### 1.3.1 Bootstrap 函数

```typescript
// src/application/bootstrap.ts
import type { Dispatch } from '@reduxjs/toolkit'
import { terminalDataCommands } from '../features/commands'
import { selectActivationStatus, selectTerminalCredential } from '../features/selectors'

export async function bootstrapTerminalData(
  dispatch: Dispatch,
  getState: () => RootState
) {
  // 1. 加载本地终端上下文
  await dispatch(terminalDataCommands.loadLocalTerminalContext())

  // 2. 检查激活状态
  const state = getState()
  const activationStatus = selectActivationStatus(state)

  if (activationStatus !== 'ACTIVATED') {
    // 未激活，等待用户激活
    return
  }

  // 3. 检查 token 有效期
  const credential = selectTerminalCredential(state)
  if (credential && isTokenExpiringSoon(credential)) {
    await dispatch(terminalDataCommands.refreshTerminalCredential())
  }

  // 4. 获取 Bootstrap 数据
  await dispatch(terminalDataCommands.fetchTcpBootstrap())

  // 5. 重建派生树
  await dispatch(terminalDataCommands.rebuildDerivedTree())

  // 6. 连接 TDP 会话
  await dispatch(terminalDataCommands.connectTdpSession())

  // 7. 同步所有 Topic（由 fetchTcpBootstrap 触发）
  // Topics 会在 fetchBootstrapActor 中自动启动定期同步
}

function isTokenExpiringSoon(credential: TerminalCredential): boolean {
  const now = Date.now()
  const expiresAt = credential.expiresAt
  const timeUntilExpiry = expiresAt - now
  const threshold = 5 * 60 * 1000 // 5 分钟

  return timeUntilExpiry < threshold
}
```

## 2. 激活流程

### 2.1 流程目标

让终端从未激活状态进入已激活状态，完成以下任务：

1. **获取终端身份**：terminalId、deviceId、deviceFingerprint
2. **获取访问凭证**：accessToken、refreshToken、expiresAt
3. **建立绑定关系**：tenant、brand、store、profile 引用
4. **接入数据平面**：连接 TDP，开始接收数据

### 2.3 代码实现

#### 2.3.1 激活 Actor

```typescript
// src/features/actors/IdentityActor.ts
export const activateTerminalActor = createActor({
  command: terminalDataCommands.activateTerminal,
  execute: async ({ payload, dispatch, getState }) => {
    const { activationCode } = payload

    try {
      // 1. 更新状态为激活中
      dispatch(terminalIdentitySlice.actions.setActivationStatus('ACTIVATING'))

      // 2. 采集设备信息
      const deviceId = await deviceInfoAdapter.getDeviceId()
      const deviceFingerprint = await deviceInfoAdapter.getDeviceFingerprint()
      const deviceModel = await deviceInfoAdapter.getDeviceModel()
      const osVersion = await deviceInfoAdapter.getOSVersion()

      // 3. 调用 TCP 激活接口
      const response = await tcpControlClient.activate({
        deviceId,
        deviceFingerprint,
        deviceModel,
        osVersion,
        activationCode,
      })

      // 4. 更新身份状态
      dispatch(terminalIdentitySlice.actions.setIdentity({
        terminalId: response.terminalId,
        deviceId,
        deviceFingerprint,
        activationStatus: 'ACTIVATED',
        activatedAt: Date.now(),
        lastActivatedAt: Date.now(),
      }))

      // 5. 更新凭证状态
      dispatch(terminalIdentitySlice.actions.setCredential({
        accessToken: response.credential.accessToken,
        refreshToken: response.credential.refreshToken,
        expiresAt: response.credential.expiresAt,
        refreshExpiresAt: response.credential.refreshExpiresAt,
        tokenType: 'Bearer',
        updatedAt: Date.now(),
      }))

      // 6. 更新绑定上下文
      dispatch(terminalBindingSlice.actions.setBindingContext(response.bindingContext))

      // 7. 持久化到本地存储
      await credentialRepository.save({
        identity: {
          terminalId: response.terminalId,
          deviceId,
          deviceFingerprint,
          activationStatus: 'ACTIVATED',
          activatedAt: Date.now(),
          lastActivatedAt: Date.now(),
        },
        credential: {
          accessToken: response.credential.accessToken,
          refreshToken: response.credential.refreshToken,
          expiresAt: response.credential.expiresAt,
          refreshExpiresAt: response.credential.refreshExpiresAt,
          tokenType: 'Bearer',
          updatedAt: Date.now(),
        },
      })

      // 8. 触发后续流程
      dispatch(terminalDataCommands.fetchTcpBootstrap())
      dispatch(terminalDataCommands.rebuildDerivedTree())
      dispatch(terminalDataCommands.connectTdpSession())

    } catch (error) {
      // 激活失败
      dispatch(terminalIdentitySlice.actions.setActivationStatus('UNACTIVATED'))
      dispatch(terminalIdentitySlice.actions.setLastError(
        error instanceof Error ? error.message : 'Activation failed'
      ))
      throw error
    }
  },
})
```

#### 2.3.2 UI 层调用

```typescript
// 2-ui/integration-mixc-retail/src/screens/ActivationScreen.tsx
import { useState } from 'react'
import { useDispatch } from 'react-redux'
import { terminalDataCommands } from '@next/kernel-core-terminal-data'

function ActivationScreen() {
  const [activationCode, setActivationCode] = useState('')
  const [isActivating, setIsActivating] = useState(false)
  const [error, setError] = useState<string | null>(null)
  const dispatch = useDispatch()

  const handleActivate = async () => {
    if (!activationCode) {
      setError('请输入激活码')
      return
    }

    setIsActivating(true)
    setError(null)

    try {
      await dispatch(terminalDataCommands.activateTerminal({ activationCode }))
      // 激活成功，跳转到主界面
      navigation.navigate('Main')
    } catch (err) {
      setError(err instanceof Error ? err.message : '激活失败')
    } finally {
      setIsActivating(false)
    }
  }

  return (
    <View>
      <TextInput
        value={activationCode}
        onChangeText={setActivationCode}
        placeholder="请输入激活码"
        editable={!isActivating}
      />
      {error && <Text style={{ color: 'red' }}>{error}</Text>}
      <Button
        title={isActivating ? '激活中...' : '激活'}
        onPress={handleActivate}
        disabled={isActivating}
      />
    </View>
  )
}
```

## 3. TCP Bootstrap 流程

### 3.1 流程目标

从 TCP 控制平面获取终端运行所需的基础数据，包括：

1. **运行时快照**：当前配置版本、应用版本、在线状态
2. **期望配置**：目标配置版本和配置内容
3. **期望版本**：目标应用版本和下载地址
4. **Topic 目录**：需要订阅的 TDP Topic 列表
5. **初始任务**：待执行的任务投递（可选）

### 3.3 代码实现

#### 3.3.1 Bootstrap Actor

```typescript
// src/features/actors/TcpControlActor.ts
export const fetchBootstrapActor = createActor({
  command: terminalDataCommands.fetchTcpBootstrap,
  execute: async ({ dispatch, getState }) => {
    const state = getState()
    const terminalId = state.terminalIdentity.identity?.terminalId

    if (!terminalId) {
      throw new Error('No terminal ID')
    }

    try {
      // 1. 调用 TCP Bootstrap API
      const bootstrap = await tcpControlClient.fetchBootstrap(terminalId)

      // 2. 更新绑定上下文（如果有变化）
      if (bootstrap.bindingContext) {
        dispatch(terminalBindingSlice.actions.setBindingContext(bootstrap.bindingContext))
      }

      // 3. 更新运行时快照
      if (bootstrap.runtimeSnapshot) {
        dispatch(terminalRuntimeSlice.actions.setRuntimeSnapshot(bootstrap.runtimeSnapshot))
      }

      // 4. 更新期望配置
      if (bootstrap.desiredConfig) {
        dispatch(terminalRuntimeSlice.actions.setDesiredConfig({
          configVersion: bootstrap.desiredConfig.version,
          configPayload: bootstrap.desiredConfig.payload,
          updatedAt: Date.now(),
        }))
      }

      // 5. 更新期望版本
      if (bootstrap.desiredVersion) {
        dispatch(terminalRuntimeSlice.actions.setDesiredVersion({
          appVersion: bootstrap.desiredVersion.appVersion,
          bundleVersion: bootstrap.desiredVersion.bundleVersion,
          downloadUrl: bootstrap.desiredVersion.downloadUrl,
          updatedAt: Date.now(),
        }))
      }

      // 6. 注册 Topics
      if (bootstrap.topics && bootstrap.topics.length > 0) {
        for (const topic of bootstrap.topics) {
          dispatch(terminalDataCommands.startProjectionPeriodicSync({
            topicKey: topic.topicKey,
          }))
        }
      }

      // 7. 重建派生树
      dispatch(terminalDataCommands.rebuildDerivedTree())

    } catch (error) {
      dispatch(terminalRuntimeSlice.actions.setLastError(
        error instanceof Error ? error.message : 'Bootstrap failed'
      ))
      throw error
    }
  },
})
```

### 3.4 Bootstrap 时机

**必须执行 Bootstrap 的场景**：
1. 终端激活成功后
2. 应用启动时（已激活状态）
3. Token 刷新后（可选）

**可选执行 Bootstrap 的场景**：
1. 定期刷新（如每小时一次）
2. 用户手动触发刷新
3. 从错误状态恢复后

## 4. TDP Session 建连流程

### 4.1 流程目标

建立终端与 TDP 数据平面的 WebSocket 长连接，实现：

1. **实时数据推送**：接收 Projection 和 Change 更新
2. **任务投递**：接收控制平面下发的任务
3. **心跳保活**：维持连接活跃状态
4. **断线重连**：网络中断后自动恢复

```typescript
import { useSelector, useDispatch } from 'react-redux'
import { selectUnconsumedInboxItems, terminalDataCommands } from '@next/kernel-core-terminal-data'

function ConfigUpdateHandler() {
  const dispatch = useDispatch()
  const inboxItems = useSelector(selectUnconsumedInboxItems)

  useEffect(() => {
    // 过滤出配置更新任务
    const configTasks = inboxItems.filter(item => item.taskType === 'CONFIG_UPDATE')

    for (const task of configTasks) {
      handleConfigUpdate(task)
        .then(() => {
          // 上报成功
          dispatch(terminalDataCommands.reportTaskResult({
            releaseId: task.payload.releaseId,
            result: 'SUCCESS',
            message: 'Config applied successfully',
          }))

          // 标记为已消费
          dispatch(terminalDataCommands.markDeliveryConsumed({
            envelopeId: task.envelopeId,
          }))
        })
        .catch(error => {
          // 上报失败
          dispatch(terminalDataCommands.reportTaskResult({
            releaseId: task.payload.releaseId,
            result: 'FAILURE',
            message: error.message,
          }))
        })
    }
  }, [inboxItems, dispatch])

  return null
}

  const { configVersion, configPayload } = task.payload
```

### 6.4 为什么需要 Inbox

**Projection vs Inbox 的区别**：

| 维度 | Projection | Inbox |
|------|-----------|-------|
| 语义 | 状态快照 | 待处理动作 |
| 持久化 | 长期保存 | 消费后可清理 |
| 更新方式 | 覆盖或合并 | 追加 |
| 消费模式 | 随时读取 | 消费一次 |
| 典型用途 | 商品数据、会员数据 | 配置下发、版本更新、远程控制 |

**Inbox 的优势**：
1. **明确的消费语义**：每个任务只消费一次
2. **执行状态跟踪**：可以记录 receivedAt、consumedAt
3. **失败重试**：未消费的任务可以重试
4. **业务解耦**：不同业务模块监听不同 taskType

```typescript
// 上层热更新模块
import { useDesiredVersion } from '@next/kernel-core-terminal-data'

function HotUpdateManager() {
  const dispatch = useDispatch()
  const desiredVersion = useDesiredVersion()
  const currentVersion = useSelector(selectCurrentVersion)

  useEffect(() => {
    if (!desiredVersion) return

    // 检查是否需要更新
    if (desiredVersion.appVersion === currentVersion.appVersion &&
        desiredVersion.bundleVersion === currentVersion.bundleVersion) {
      return
    }

    // 执行热更新
    performHotUpdate(desiredVersion)
      .then(() => {
        // 上报成功（重启后执行）
        dispatch(terminalDataCommands.reportVersionApplied({
          appVersion: desiredVersion.appVersion,
          bundleVersion: desiredVersion.bundleVersion,
        }))
      })
      .catch(error => {
        console.error('Hot update failed', error)
      })
  }, [desiredVersion, currentVersion, dispatch])

  return null
}

async function performHotUpdate(desiredVersion: DesiredVersionState) {
  // 1. 下载新版本包
  const bundlePath = await downloadBundle(desiredVersion.downloadUrl)

  // 2. 验证签名
  await verifyBundleSignature(bundlePath)

  // 3. 安装新版本
  await installBundle(bundlePath)

  // 4. 重启应用
  await restartApp()
}
```

```typescript
function OrganizationTreeView() {
  const nodes = useDerivedTree()

  // 查找根节点
  const rootNode = nodes.find(node => node.nodeType === 'ROOT')

  // 递归渲染树
  return <TreeNode node={rootNode} nodes={nodes} />
}

function TreeNode({ node, nodes }: { node: DerivedTreeNode, nodes: DerivedTreeNode[] }) {
  const children = nodes.filter(n => n.parentNodeId === node.nodeId)

  return (
    <View>
      <Text>{node.label}</Text>
      {children.map(child => (
        <TreeNode key={child.nodeId} node={child} nodes={nodes} />
      ))}
    </View>
  )
}
```

### 9.7 关键原则

1. **视图树，非权威树**：这是客户端的视图表示，不参与服务端权威判断
2. **不引入 rootPath**：不使用旧的 `rootPath` 覆盖体系
3. **辅助 Scope 解析**：可用于 UI 展示和 Scope 匹配判断
4. **不持久化**：每次启动或绑定变化时重建

## 10. 错误恢复流程

### 10.1 典型错误类型

| 错误类型 | 触发场景 | 恢复策略 |
|---------|---------|---------|
| **激活失败** | 激活码无效、网络错误 | 提示用户重新输入激活码 |
| **Token 失效** | accessToken 过期 | 自动刷新 token，失败则重新激活 |
| **Bootstrap 失败** | 网络错误、服务端异常 | 延迟重试，使用本地缓存 |
| **Session 连接失败** | 网络错误、服务端拒绝 | 指数退避重连 |
| **心跳超时** | 网络中断、服务端无响应 | 断开连接，触发重连 |
| **Projection 同步失败** | 网络错误、数据格式错误 | 记录错误，延迟重试 |
| **Report 失败** | 网络错误、服务端拒绝 | 加入队列，定期重试 |

### 10.2 Token 刷新流程

```
┌─────────────────────────────────────────────────────────────┐
│  检测到 Token 即将过期                                        │
│  - expiresAt - now < 5 分钟                                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  1. 调用 TCP 刷新接口                                         │
│     POST /api/tcp/v1/terminals/{terminalId}/refresh          │
│     { refreshToken }                                         │
└─────────────────────────────────────────────────────────────┘
                            ↓
                    ┌───────┴───────┐
                    │  刷新成功？    │
                    └───────┬───────┘
                            │
            ┌───────────────┼───────────────┐
            │ 否                            │ 是
            ↓                               ↓
┌─────────────────────┐       ┌─────────────────────────────┐
│  refreshToken 也过期 │       │  2. 更新 credential          │
│  需要重新激活        │       │     - 新的 accessToken       │
│                     │       │     - 新的 refreshToken      │
└─────────────────────┘       │     - 新的 expiresAt         │
                              └─────────────────────────────┘
                                            ↓
                              ┌─────────────────────────────┐
                              │  3. 持久化到本地存储          │

                                            ↓
                              ┌─────────────────────────────┐
```

### 10.3 Session 重连流程（指数退避）

```typescript
// src/foundations/coordinators/SessionCoordinator.ts
export class SessionCoordinator {
  private reconnectAttempts = 0
  private maxReconnectAttempts = 10
  private baseDelay = 1000 // 1 秒

  async reconnect(params: { terminalId: string; accessToken: string }): Promise<void> {
    if (this.reconnectAttempts >= this.maxReconnectAttempts) {
      throw new Error('Max reconnect attempts reached')
    }

    // 计算延迟（指数退避）
    const delay = Math.min(
      this.baseDelay * Math.pow(2, this.reconnectAttempts),
      30000 // 最大 30 秒
    )

    // 等待延迟
    await new Promise(resolve => setTimeout(resolve, delay))

    try {
      // 尝试重连
      await this.tdpSessionClient.connect(params)

      // 重连成功，重置计数
      this.reconnectAttempts = 0
    } catch (error) {
      // 重连失败，增加计数
      this.reconnectAttempts++
      throw error
    }
  }

  resetReconnectAttempts(): void {
    this.reconnectAttempts = 0
  }
}
```

### 10.4 Report 失败重试

```typescript
// src/foundations/coordinators/ReportCoordinator.ts
export class ReportCoordinator {
  private reportQueue: TerminalReportPayload[] = []
  private maxRetries = 3

  async reportWithRetry(
    terminalId: string,
    report: TerminalReportPayload
  ): Promise<void> {
    let retries = 0

    while (retries < this.maxRetries) {
      try {
        await this.tcpClient.reportTaskResult(terminalId, report)
        return // 成功
      } catch (error) {
        retries++
        if (retries >= this.maxRetries) {
          // 达到最大重试次数，加入队列
          this.reportQueue.push(report)
          throw error
        }
        // 等待后重试
        await new Promise(resolve => setTimeout(resolve, 1000 * retries))
      }
    }
  }
}
```

## 12. 总结

### 12.1 关键流程清单

本文档详细阐述了 `terminal-data` 包的 11 个关键流程：

1. **初始化流程**：应用启动时的完整初始化步骤
2. **激活流程**：从未激活到已激活的完整过程
3. **TCP Bootstrap 流程**：获取控制平面基础数据
4. **TDP Session 建连流程**：建立数据平面长连接
5. **Projection 同步流程**：全量同步、增量追平、实时投递
6. **任务投递消费流程**：Inbox 模式的任务处理
7. **配置下发流程**：期望配置的获取和应用
8. **热更新目标流程**：期望版本的获取和应用
9. **派生树构建流程**：组织树视图的构建
10. **错误恢复流程**：各类错误的恢复策略
11. **观察性设计**：调试和监控指标

### 12.2 流程间依赖关系

```
初始化流程
  ├─ 加载本地上下文
  ├─ 激活流程（如果未激活）
  ├─ Token 刷新（如果即将过期）
  ├─ TCP Bootstrap 流程
  │   └─ 派生树构建流程
  ├─ TDP Session 建连流程
  │   ├─ 心跳保活
  │   └─ 实时投递消费
  └─ Projection 同步流程
      ├─ 全量同步
      ├─ 增量追平
      └─ 实时更新
          ├─ 配置下发流程
          ├─ 热更新目标流程
          └─ 任务投递消费流程
```

### 12.3 实现要点

1. **状态驱动**：所有流程通过 Redux state 驱动，避免散落的状态
2. **命令抽象**：所有动作通过 Commands 统一建模
3. **Actor 编排**：所有副作用由 Actor 承担，保持逻辑清晰
4. **错误恢复**：每个流程都有明确的错误处理和恢复策略
5. **观察性**：提供丰富的观察性指标，便于调试和监控

### 12.4 与上层业务的集成

上层业务模块（配置模块、热更新模块、远程控制模块等）应：

1. **监听 State 变化**：使用 Selectors 和 Hooks 监听相关状态
2. **消费 Inbox**：处理收件箱中的任务投递
3. **上报结果**：执行完成后调用 Commands 上报结果
4. **不直接调用 Foundation**：通过 Commands 驱动，不直接调用底层组件
