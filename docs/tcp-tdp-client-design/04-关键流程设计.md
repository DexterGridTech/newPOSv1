# 04-关键流程设计

---

## 1. 初始化流程

### 1.1 目标
应用启动后，应完成：
- 本地身份与缓存装载
- 设备信息采集
- TCP bootstrap 同步
- TDP session 判断与建立
- projection / change catch-up
- runtime state 初始化

### 1.2 流程建议

```text
应用启动
  -> initializeTerminalData
  -> loadLocalTerminalContext
  -> 获取设备信息
  -> 如果已激活：refreshCredential / fetchBootstrap
  -> 重建 binding context refs
  -> rebuildDerivedTree
  -> connectTdpSession
  -> syncProjectionSnapshot
  -> catchUpProjectionChanges
  -> 更新 runtime state
```

---

## 2. 激活流程

### 2.1 目标
让终端从未激活态进入：
- 已激活
- 已获得 credential
- 已建立 binding context
- 已可接入 TDP

### 2.2 流程建议

```text
输入 activationCode
  -> activateTerminal
  -> TCP activate api
  -> 返回 terminal identity + credential + binding bootstrap
  -> 写入 terminalIdentity / terminalBinding / tcpClient
  -> rebuildDerivedTree
  -> connectTdpSession
  -> syncProjectionSnapshot
```

### 2.3 输出结果
激活成功后，客户端至少应有：
- terminalId
- credential
- tenant / brand / store / profile refs
- 初始 projection snapshot

---

## 3. TCP bootstrap 流程

### 3.1 目标
从 TCP 获取控制面需要的基础数据。

### 3.2 bootstrap 建议内容
- terminal identity
- binding context
- runtime snapshot
- desired config/version target
- topic catalog 简版
- 初始任务投递上下文（可选）

### 3.3 流程建议

```text
fetchTcpBootstrap
  -> TcpControlClient.fetchBootstrap
  -> BootstrapAssembler 标准化
  -> 更新 terminalBinding / terminalRuntime / tcpClient
  -> rebuildDerivedTree
```

---

## 4. TDP Session 建连流程

### 4.1 目标
建立终端与 TDP 的长连接或等价会话连接。

### 4.2 流程建议

```text
connectTdpSession
  -> TdpSessionActor.connect
  -> TdpTransportAdapter.connect
  -> 鉴权 / handshake
  -> 成功后更新 tdpSession = CONNECTED
  -> 启动 heartbeat scheduler
  -> 启动 transport message listener
```

### 4.3 重连流程

```text
网络中断 / 心跳超时 / transport error
  -> tdpSession = ERROR / DISCONNECTED
  -> reconnect scheduler backoff
  -> reconnectTdpSession
  -> 恢复后 catchUpProjectionChanges
```

---

## 5. Projection 同步流程

### 5.1 全量同步
用于：
- 首次启动
- 激活后首次接入
- 本地缓存失效后重建

```text
syncProjectionSnapshot
  -> TdpSyncClient.fetchSnapshot
  -> 标准化 ProjectionRecord[]
  -> 更新 tdpProjection
  -> RuntimeActor 解析 desired config/version/inbox
```

### 5.2 增量追平
用于：
- 重连后补偿
- 长连接恢复后追平

```text
catchUpProjectionChanges
  -> TdpSyncClient.fetchChanges(fromCursor)
  -> ChangeRecord[]
  -> merge into tdpProjection
  -> 更新 checkpoint
  -> RuntimeActor 应用变化
```

### 5.3 实时投递消费
用于：
- transport 实时收到新消息

```text
transport message
  -> TdpSyncActor.handleIncomingMessage
  -> 标准化为 ProjectionRecord / ChangeRecord / DeliveryEnvelope
  -> 更新 tdpProjection / tdpInbox
  -> RuntimeActor 计算 desired state
```

---

## 6. 任务投递消费流程

### 6.1 目标
把来自 TCP/TDP 的投递任务，转化为终端侧可执行的本地收件箱项。

### 6.2 流程建议

```text
收到 task delivery projection
  -> 生成 DeliveryInboxItem
  -> 写入 tdpInbox
  -> 上层业务消费该 inbox item
  -> 执行完成后 reportTaskResult
  -> markDeliveryConsumed
```

### 6.3 为什么要 inbox
因为任务投递和投影数据虽然都来源于 TDP，但语义不同：
- projection 更偏状态快照
- inbox 更偏待处理动作

二者分开后，客户端逻辑会更清晰。

---

## 7. 配置下发流程

### 7.1 目标
让终端根据 TDP 的 projection 或 delivery 数据，得到目标配置，并在应用完成后上报给 TCP。

### 7.2 流程建议

```text
TCP 发布配置任务
  -> TDP 生成 tcp.task.release projection
  -> 客户端收到 projection/change
  -> RuntimeActor 解析出 DesiredConfigState
  -> 上层配置模块执行配置应用
  -> 完成后 reportConfigApplied
  -> TCP 更新当前配置版本
```

---

## 8. 热更新目标流程

### 8.1 目标
让终端感知目标版本并择机执行。

### 8.2 流程建议

```text
TCP 指定目标版本
  -> TDP 分发 terminal.runtime.config / upgrade topic
  -> 客户端解析 DesiredVersionState
  -> 上层热更新模块执行下载/安装/重启
  -> 完成后 reportVersionApplied
  -> TCP 更新 terminal runtime snapshot
```

---

## 9. 派生树构建流程

### 9.1 目标
在客户端内部得到一个可浏览、可选择、可辅助 scope 解析的统一树视图。

### 9.2 输入
- binding context
- tenant / brand / store refs
- profile ref
- terminal identity

### 9.3 输出示意

```text
ROOT
 └─ TENANT
     └─ BRAND
         └─ STORE
             ├─ TERMINAL
             └─ PROFILE (可挂为附属节点或视图节点)
```

### 9.4 关键原则
- 这是视图树，不是服务端权威树
- 不参与服务端正式 scope 权威判断
- 不引入旧 `rootPath` 覆盖体系

## 10. 错误恢复流程

### 10.1 典型错误类型
- 激活失败
- token 失效
- bootstrap 失败
- session connect 失败
- heartbeat timeout
- projection merge 失败
- report 失败

### 10.2 恢复策略建议
- token 失效 -> refresh 或重新激活
- session 断开 -> backoff reconnect
- snapshot 失败 -> fallback 本地缓存 + 延迟重试
- report 失败 -> 本地队列重试
- merge 失败 -> 记录 diagnostics 并上报错误

---

## 11. 观察性建议

为了支持联调与排障，建议内核暴露：
- 最后一次 bootstrap 时间
- 最后一次 session connect 结果
- 最后一次 heartbeat 时间
- 最后一次 snapshot sync 时间
- 最后一次 report 结果
- projection cache 大小
- inbox 未消费数量
- 最近错误栈 / 错误码
