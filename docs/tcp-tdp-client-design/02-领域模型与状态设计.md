# 02-领域模型与状态设计

---

## 1. 领域模型总览

新 `terminal-data` 的领域模型分成 6 组：

- Identity 实体
- Binding / Organization 引用实体
- TCP 控制面实体
- TDP 数据面实体
- Runtime 运行时实体
- DerivedTree 派生视图实体

## 8. Redux 状态建议

建议新包至少包含以下 slice：

### 8.1 `terminalIdentity`
保存：
- TerminalIdentity
- TerminalCredential
- activation status

### 8.2 `terminalBinding`
保存：
- TerminalBindingContext
- tenant / brand / store / profile / template refs

### 8.3 `terminalRuntime`
保存：
- TerminalRuntimeSnapshot
- DesiredConfigState
- DesiredVersionState
- TerminalCapabilityState

### 8.4 `tcpClient`
保存：
- 最近一次 bootstrap
- 最近一次 report 状态
- 最后一次错误
- refresh token 状态

### 8.5 `tdpSession`
保存：
- TdpSessionStatus
- TdpSessionDescriptor
- reconnect info
- connection history

### 8.6 `tdpProjection`
保存：
- projection by `topicKey + scopeKey`
- change logs by topic / time
- topic catalog
- sync checkpoint

### 8.7 `tdpInbox`
保存：
- DeliveryInboxItem 列表
- consume 状态

### 8.8 `terminalDerivedTree`
保存：
- DerivedTreeNode 集合
- rootNodeIds
- 最近构建时间

## 9. 为什么不再用旧 `UnitDataState`

旧 `UnitDataState` 的 key 是 `unitData.id`，其语义建立在：
- 单位树
- path 覆盖
- 组数据更新
之上。

新模型下：
- TDP 的核心是 `ProjectionRecord`
- 变化核心是 `ChangeRecord`
- 本地运行时目标是 `DesiredConfigState / DesiredVersionState / DeliveryInbox`

因此不再以 `UnitDataState` 作为状态中心。

---

## 10. 状态设计总结

新状态设计的核心变化是：

- 从“树覆盖式状态”转向“显式引用 + 投影缓存式状态”
- 从“kernel unitData group”转向“topic / projection / change / inbox”
- 从“万能 terminal 包”转向“identity / tcp / tdp / runtime / derived view”多子域状态
