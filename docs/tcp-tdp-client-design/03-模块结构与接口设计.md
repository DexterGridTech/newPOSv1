## 4. modulePreSetup 设计

### 4.1 作用
沿用旧 `terminal` 的思路，但调整为新的 TCP/TDP 体系。

### 4.2 建议职责
- 注册 TCP API token request interceptor
- 初始化 foundation adapter registry
- 初始化 session / sync coordinator
- 预加载本地缓存快照
- 注册必要生命周期监听器

### 4.3 不建议职责
- 直接发业务请求
- 直接构造 UI 逻辑
- 直接依赖项目级远控行为

## 5. supports/apis 设计

### 5.1 TCP API 分组
建议定义：

```ts
export const terminalDataTcpApis = {
  activateTerminal,
  refreshCredential,
  fetchBootstrap,
  reportTaskResult,
  reportConfigApplied,
  reportVersionApplied,
  reportLifecycleState,
}
```

### 5.2 TDP API / Transport 分组
建议定义：

```ts
export const terminalDataTdpApis = {
  connectSession,
  heartbeatSession,
  disconnectSession,
  fetchSnapshot,
  fetchChanges,
}
```

### 5.3 为什么不把 TDP 全部写死为 HTTP
因为未来 TDP 可能通过：
- WS
- SSE
- HTTP long polling
- native push channel
多种方式接入。

所以建议：
- `supports/apis` 提供默认协议描述
- `foundations/adapters` 提供 transport 抽象

## 8. actors 设计

### 8.1 `InitializeActor`
负责：
- 设备信息采集
- 本地缓存装载
- 触发 bootstrap
- 触发 session 初始化

### 8.2 `IdentityActor`
负责：
- 激活
- refresh token
- 身份重置
- credential 存储

### 8.3 `TcpControlActor`
负责：
- 调用 TCP HTTP 接口
- 获取 bootstrap
- 上报执行结果
- 同步控制面状态

### 8.4 `TdpSessionActor`
负责：
- connect / heartbeat / disconnect
- reconnect
- 更新会话状态

### 8.5 `TdpSyncActor`
负责：
- 拉取 snapshot
- 拉取 changes
- 消费 transport 收到的 projection / delivery 消息
- 更新 projection 与 changeLog 状态

### 8.6 `RuntimeActor`
负责：
- 根据 projection 更新本地 desired config/version/runtime state
- 根据任务结果刷新 inbox 状态

### 8.7 `DerivedTreeActor`
负责：
- 根据 binding context 与 refs 生成派生树
- 更新 scope 辅助索引

## 9. slices 设计

建议 slices 如下：

- `terminalIdentity`
- `terminalBinding`
- `terminalRuntime`
- `tcpClient`
- `tdpSession`
- `tdpProjection`
- `tdpInbox`
- `terminalDerivedTree`

### persist 策略建议

#### 需要持久化
- `terminalIdentity`
- `terminalBinding`
- `terminalRuntime`
- `tdpProjection`（可按容量策略裁剪）
- `tdpInbox`
- `terminalDerivedTree`

#### 不建议持久化
- `tdpSession` 的实时连接态
- 瞬时错误态

### syncType 建议
根据 `kernel-core-interconnection` 的约定：
- 大多数数据内核状态建议 `MASTER_TO_SLAVE`
- 纯实时会话态可设为 `ISOLATED` 或不做持久同步

## 11. 错误与参数设计

### errorMessages
应至少覆盖：
- 激活失败
- token refresh 失败
- TCP bootstrap 失败
- TDP connect 失败
- heartbeat 超时
- projection sync 失败
- report 失败
- 派生树构建失败

### parameters
可定义：
- heartbeat interval
- reconnect backoff
- snapshot sync interval
- change catch-up batch size
- projection cache size
- inbox retain count
