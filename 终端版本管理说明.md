## 1. 适用范围

- `1-kernel/1.1-base/*`

- `2-ui/2.1-base/*`
- `2-ui/2.3-integration/retail-shell`
- `3-adapter/android/adapter-android-v2`

## 3. 当前版本字段

### 3.1 Assembly Version

用途：安装包主版本。

位置：

- `4-assembly/android/mixc-retail-assembly-rn84/release.manifest.json`

字段：

- `assemblyVersion`

示例：

```json
"assemblyVersion": "1.0.0"
```

### 3.2 Build Number

用途：构建号，要求递增。

位置：

- `release.manifest.json` 中的 `buildNumber`
- `android/app/build.gradle` 中的 `versionCode`

### 3.3 Bundle Version

用途：JS Bundle / OTA 版本。

位置：

- `release.manifest.json` 中的 `bundleVersion`

示例：

```json
"bundleVersion": "1.0.0+ota.11"
```

### 3.4 Runtime Version

用途：标识运行时兼容范围。

位置：

- `release.manifest.json` 中的 `runtimeVersion`

示例：

```json
"runtimeVersion": "android-mixc-retail-rn84@1.0"
```

### 3.5 Package Version

用途：kernel/ui 包内部版本展示与追踪。

来源：

- 各包自己的 `package.json.version`

同步方式：

- `node scripts/release/generate-package-version-info.cjs`

## 5. 日常开发

### 5.1 只开发功能，不发版

如果只是日常开发，不需要改 `release.manifest.json`。

正常做法：

- 修改代码
- 跑 `type-check`
- 跑相关测试
- 跑本地 dev / Android 联调

### 5.2 修改某个 kernel/ui 包的 `package.json.version` 后

执行：

```bash
node scripts/release/generate-package-version-info.cjs
```

然后再执行对应包的 `type-check` / `test`。

## 6. Android 安装包发布

### 6.1 一次性完整流程

推荐直接执行：

```bash
node scripts/release/release-assembly-android-full.cjs --app assembly-android-mixc-retail-rn84 --version 1.1.0 --channel production
```

这个脚本会自动完成：

1. 更新 `assemblyVersion`
2. 递增 `buildNumber`
3. 刷新 `targetPackages`
4. 注入 git 信息
5. 校验 manifest
6. 同步 assembly `package.json.version`
7. 同步 Android `versionCode / versionName`
8. 生成 `src/generated/releaseInfo.ts`
9. 执行 assembly `type-check`
10. 构建 Android release 包
11. 写回产物信息到 manifest

### 6.2 拆开执行

```bash
node scripts/release/bump-assembly-version.cjs --app assembly-android-mixc-retail-rn84 --version 1.1.0
node scripts/release/bump-build-number.cjs --app assembly-android-mixc-retail-rn84
node scripts/release/prepare-release-manifest.cjs --app assembly-android-mixc-retail-rn84 --channel production
node scripts/release/inject-git-metadata.cjs --app assembly-android-mixc-retail-rn84
node scripts/release/validate-release-manifest.cjs --app assembly-android-mixc-retail-rn84
node scripts/release/sync-assembly-version.cjs --app assembly-android-mixc-retail-rn84
node scripts/release/sync-android-version.cjs --app assembly-android-mixc-retail-rn84
node scripts/release/generate-assembly-release-info.cjs --app assembly-android-mixc-retail-rn84
corepack yarn workspace @impos2/assembly-android-mixc-retail-rn84 type-check
bash -lc 'source ~/.zshrc && cd 4-assembly/android/mixc-retail-assembly-rn84/android && ./gradlew assembleRelease'
node scripts/release/finalize-release-manifest.cjs --app assembly-android-mixc-retail-rn84
node scripts/release/generate-release-note.cjs --app assembly-android-mixc-retail-rn84
```

## 8. 修改版本时的原则

### 8.1 不要手改安装包版本

不要手改：

- `android/app/build.gradle`
- assembly `package.json.version`
- `src/generated/releaseInfo.ts`

正确做法：

- 用 `bump-assembly-version.cjs`
- 再执行同步脚本或 full-flow

### 8.2 不要手改 bundleVersion

正确做法：

- 用 `bump-bundle-version.cjs`
- 或直接用 `release-bundle-full.cjs`

### 8.3 不要手改 Android versionCode

正确做法：

- 用 `bump-build-number.cjs`
- 再执行 `sync-android-version.cjs`

## 9. 新包如何纳入当前版本体系

### 9.1 对 `1-kernel` / `2-ui` 新包

要求：

- 有 `package.json`
- 有 `src/index.ts`

执行：

```bash
node scripts/release/generate-package-version-info.cjs
```

### 9.2 对 `4-assembly` 新包

如果以后新增新的 Android assembly，需要补：

1. `release.manifest.json`
2. `src/generated/releaseInfo.ts`
3. `scripts/release/shared.cjs` 的 appId 映射
4. `scripts/release/generate-assembly-release-info.cjs`
5. `scripts/release/finalize-release-manifest.cjs`
6. `scripts/release/release-bundle-full.cjs`
