# 后台使用手册

> 适用对象：第一次使用 `Mock Terminal Platform` 后台的小白管理员、实施同学、测试同学、产品同学。  
> 阅读目标：看着后台页面，就知道“这个页面是干什么的”“我应该什么时候点什么按钮”“点完后应该去哪里继续看结果”。

---

## 1. 这份手册怎么用

这份手册是**对照后台截图**来写的。

你可以这样看：

1. 先把后台跑起来
2. 打开对应页面
3. 一边看页面，一边看这份手册
4. 按手册里的“推荐操作顺序”点一遍

如果你完全不懂 TCP / TDP，也没关系。  
你可以先把这份文档当成“后台导览手册”。

---

## 2. 启动后台

在仓库根目录运行：

```bash
corepack yarn mock:platform:dev
```

启动后打开浏览器：

- Web 后台：`http://localhost:5820`
- Server 接口：`http://127.0.0.1:5810`

如果你只想看后台界面，只需要打开 Web 地址即可。

---

## 3. 后台整体长什么样

整个后台由 5 个主要页面组成：

- `总览`
- `TCP 控制面`
- `TDP 数据面`
- `场景引擎`
- `故障注入`

左侧是导航，右侧是当前页面内容。

页面顶部还有 3 个通用操作：

- `刷新数据`
- `预览导出数据`
- `下载导出文件`

这些按钮在所有页面都能看到。

### 你可以这样理解这 5 个页面

- **总览**：先看全局情况
- **TCP 控制面**：做终端治理和任务下发
- **TDP 数据面**：看数据是否真正下发到了终端视角
- **场景引擎**：快速跑一套预设测试流程
- **故障注入**：模拟异常情况

## 4.1 总览页是干什么的？

总览页是后台的“首页”。  
如果你不知道当前平台状态如何，第一眼就应该来看这里。

它主要回答这几个问题：

- 现在系统里有多少终端？
- 有多少在线终端？
- 当前有多少任务在跑？
- 当前有多少活跃会话？
- 当前定义了多少 Topic？
- 最近是否发生过故障命中？

---

## 4.2 总览页里的几个区域分别怎么看

### 区域一：运行状态

这里是一排统计卡片。

你主要看：

- **终端总量**：系统里一共有多少终端
- **在线终端**：当前有多少终端在线
- **运行中任务**：有多少任务处于执行中
- **活跃会话**：当前有多少 TDP Session 是连接状态
- **主题数量**：系统里有多少 Topic
- **故障命中**：最近故障规则被触发了多少次

### 什么时候看这里

- 你刚进后台时
- 你想知道当前测试环境是否“活着”
- 你刚跑完一个场景，想看整体数字是否变化

### 小白建议

如果你刚执行了某个动作，比如：
- 批量造终端
- 发布配置任务
- 运行场景模板

那你可以回到总览页看看这些统计数字是否变化。这是最简单的验收方式。

---

### 区域二：默认沙箱

这里告诉你当前平台默认工作在哪个沙箱里。

你会看到：
- 沙箱名称
- 状态
- 用途
- 资源上限

### 什么时候看这里

- 你想确认当前是不是默认联调环境
- 你想知道环境是不是激活状态
- 你想了解这个环境大概支持多少数据量

### 小白理解方式

把它理解成：
> 你现在正在这个“练习场”里操作。

---

### 区域三：联调建议

这里会用白话告诉你当前适合做什么事情，比如：

- 当前在线终端有多少台
- Projection / Change Log 有多少条
- 有多少场景模板
- 有多少故障规则

### 什么时候看这里

- 你不知道下一步点哪个页面
- 你想快速知道当前环境有没有“可玩内容”

### 小白建议

如果你完全不知道下一步做什么，就按这个顺序：

1. 看终端联调
2. 看 TDP 观察
3. 跑测试场景
4. 做故障调试

---

### 区域四：审计日志与导出预览

这里分成两块：

- 左边：审计日志
- 右边：导出数据预览

### 审计日志是干什么的？

告诉你：
- 谁做了什么动作
- 什么时候做的
- 目标对象是谁
- 详细参数是什么

### 什么时候看这里

- 你刚点了某个按钮，想确认后台是否真的执行了
- 你想回看是谁创建了任务、谁导入了模板
- 你在排查误操作

### 真实场景例子

比如你刚运行了“门店批量上线”场景，审计日志里就会看到一条 `RUN_SCENE_TEMPLATE` 记录。

### 导出预览是干什么的？

当你点击顶部 `预览导出数据` 按钮时，右边会显示当前平台导出的 JSON 数据。

### 什么时候用

- 你想快速看当前平台里都有哪些对象
- 你想备份当前测试数据
- 你想把这份数据给别的同学复用

---

### 区域五：模板库与导入

这里主要有 3 块：

- Topic 模板库
- Fault 模板库
- 导入预检 / 导入区域

### 它是干什么的？

帮助你：
- 不从零开始定义 Topic
- 不从零开始写故障规则
- 导入一批测试模板

### 什么时候用

- 你是新手，不知道 Topic 怎么建
- 你想快速套用一条故障规则
- 你想导入一批现成模板

### 小白推荐操作

第一次使用时，可以先：

1. 点击一个 `Topic 模板` 的“套用”
2. 再去 `TDP 数据面` 看表单有没有自动填充
3. 点击 `Fault 模板` 的“套用”
4. 再去 `故障注入` 看表单有没有自动填充

这样你会很快理解“模板库”的作用。

## 4.3 总览页推荐操作顺序

如果你是第一次用后台，建议在总览页做以下 3 个动作：

### 动作 1：点击“刷新数据”
确认页面不是旧数据。

### 动作 2：点击“预览导出数据”
看一下导出的 JSON 长什么样。

### 动作 3：翻一下审计日志
确认最近是不是有人跑过场景、创建过任务、注入过故障。

## 5.1 TCP 页面是干什么的？

这是“终端治理后台”。

你在这里主要做的是：

- 造终端
- 生成激活码
- 激活终端
- 发布任务
- 强制改终端状态
- 查看任务实例和链路追踪

如果你想做“终端管理”和“控制动作”，大多数都来这个页面。

---

## 5.2 页面里的几个区域分别怎么用

### 区域一：TCP 控制台动作

这里有几个最常用的按钮：

- `批量造 10 台终端`
- `生成 5 个激活码`
- `发布配置任务`
- `首台终端置为告警`

### 这些按钮分别是什么场景用的？

#### 1）批量造 10 台终端

**什么时候用：**
- 你刚搭好环境，里面没什么数据
- 你要快速做联调演练
- 你想模拟一个更真实的门店或批量环境

**点完后你应该去哪里看：**
- 看本页的终端总览
- 看总览页的终端总量是否变化

---

#### 2）生成 5 个激活码

**什么时候用：**
- 你要模拟新终端装机
- 你要测试激活流程
- 你要给终端开发同学准备激活数据

**点完后你应该去哪里看：**
- 本页下方的激活码列表
- 审计日志

---

#### 3）发布配置任务

**什么时候用：**
- 你要测试配置下发流程
- 你想验证 TCP -> TDP 联动是否正常
- 你想让终端模拟接收一条任务

**点完后你应该去哪里看：**
- 先看本页的任务发布 / 任务实例
- 再切到 `TDP 数据面`
- 看 `Projection` 和 `Change Log`

这是后台最推荐小白先跑的一条主链路。

---

#### 4）首台终端置为告警

**什么时候用：**
- 你想模拟终端异常
- 你想验证前端状态显示
- 你要做健康状态联调

**点完后你应该去哪里看：**
- 本页终端列表
- 看健康状态是否变成告警
- 看审计日志是否记录了这次动作

---

### 区域二：激活实操流

这里有两个输入项和一个按钮：

- 激活码
- 设备指纹
- 执行终端激活

### 什么时候用

- 你刚生成了激活码
- 你要模拟终端首次接入
- 你要验证激活是否能成功绑定进系统

### 推荐操作顺序

1. 先点击“生成 5 个激活码”
2. 把其中一个激活码填到输入框里
3. 点击“执行终端激活”
4. 看终端列表是否新增或状态变化
5. 看审计日志

### 小白理解

这相当于：
> 模拟一台终端拿着激活码来系统报到。

---

### 区域三：终端总览

这里会展示终端列表，并支持筛选。

你能看到每台终端的：
- 终端 ID
- 所属门店
- 生命周期状态
- 在线状态
- 健康状态
- App 版本
- 查看详情

右侧还会显示：
- 当前选中终端的基础信息
- 当前选中终端对应的 TDP 快照与变化

### 什么时候用

- 查某台终端当前状态
- 看这台终端属于哪个门店
- 看当前版本是多少
- 看这台终端在 TDP 里是否已经有快照

### 小白推荐操作

1. 在筛选框里输入某个终端 ID
2. 点击该终端的“查看”
3. 看详情抽屉
4. 再看右侧 JSON 面板里的快照与 changes

### 最典型的使用场景

#### 场景 A：排查某终端为什么没更新配置
看这里：
- 当前配置版本
- 当前 TDP 快照
- 是否有对应的变更记录

#### 场景 B：排查某终端是否已经升级
看这里：
- `App 版本`
- 终端详情
- 任务实例和结果

---

### 区域四：激活码与任务发布

左边看激活码，右边看任务发布单。

### 什么时候用

- 你想确认激活码是否已生成
- 你想确认某个发布单是否已创建
- 你想知道发布单的目标范围和更新时间

### 小白推荐操作

- 刚生成激活码后，来这里确认是否出现新记录
- 刚发布配置任务后，来这里确认是否出现新的发布单

---

### 区域五：任务实例调试与链路追踪

这是 TCP 页面里最重要的调试区之一。

它会告诉你：
- 每台终端对应的任务实例是什么
- 当前状态如何
- 投递状态如何
- 更新时间是什么时候
- 链路追踪数据是什么

### 什么时候用

- 你要排查某台终端上的任务为何失败
- 你想确认 TDP 是否已经接到任务
- 你想把伪结果写回去做联调

### 推荐操作顺序

1. 在任务筛选框输入关键字
2. 选中一条任务实例
3. 点击“查看”
4. 看右侧 `链路追踪`
5. 必要时点击“给首个实例写入成功结果”

### 真实场景例子

#### 场景：配置下发后，终端说自己没收到
你应该：
1. 来看任务实例是否存在
2. 看投递状态是不是 `PENDING` / `DELIVERED` / 其他状态
3. 看 trace 里是否能关联到 TDP 的 Projection / ChangeLog

## 6. TDP 数据面怎么用

### 对应截图

- `0-mock-server/mock-terminal-platform/docs/screenshots/03-tdp.png:1`

---

## 6.1 TDP 页面是干什么的？

这个页面是“数据下发和终端视角观察台”。

你在这里主要做的是：

- 创建 Topic
- 注入 Projection
- 建 Session / 发心跳 / 断开 Session
- 看 Topic 定义
- 看 Projection 快照
- 看 Change Log 历史

如果 TCP 像“指挥中心”，那 TDP 页面就是“配送和投递观察台”。

---

## 6.2 页面里的几个区域分别怎么用

### 区域一：TDP Topic / Schema / Scope 治理

这里主要用来：
- 定义一个新的 Topic
- 为某个 Topic 写一条 Projection

表单里通常会有：
- Topic Key
- Topic 名称
- Scope 类型
- Projection Payload(JSON)

按钮：
- `注册 Topic`
- `注入 Projection`

---

### 什么时候点击“注册 Topic”？

当你遇到下面这些情况时：

- 你需要定义一种新的终端被动感知数据
- 平台里还没有这个 Topic
- 你想试验一类新的投递数据结构

### 真实场景例子

你要给终端新增一种“运行时参数配置”，那你就可以注册一个新的 Topic，比如：

- `terminal.runtime.config`

---

### 什么时候点击“注入 Projection”？

当你遇到下面这些情况时：

- 你已经有 Topic 了
- 你想直接给某个终端 / 门店写入一份当前快照
- 你想跳过完整业务链路，直接模拟 TDP 当前态

### 真实场景例子

你想测试终端收到下面这类配置时会怎样处理：

```json
{
  "desiredVersion": "2.4.0",
  "rollout": "gray"
}
```

这时你就可以直接注入 Projection，而不一定非要先走完整的 TCP 任务流程。

### 小白推荐操作

1. 先套用一个 Topic 模板
2. 再来这里点“注册 Topic”
3. 然后写入 Projection
4. 看下方列表有没有变化

---

### 区域二：Session 实操

这里主要有 3 个动作：

- 建立 Session
- 发送心跳
- 断开首个 Session

### 什么时候用

- 你想模拟终端上线
- 你想模拟终端还活着（心跳）
- 你想模拟终端断开连接

### 小白理解方式

这相当于：
> 我现在在后台假装有一台终端上线了。

### 推荐操作顺序

1. 输入一个终端 ID
2. 点击“建立 Session”
3. 观察 Session 列表是否出现新记录
4. 点击“发送心跳”
5. 看“最近心跳时间”是否更新
6. 点击“断开首个 Session”
7. 看状态是否变化

---

### 区域三：TDP 会话与主题

左边是 Session 列表，右边是 Topic 列表。

#### Session 列表主要看什么
- 哪台终端连上了
- 当前连接状态是什么
- 用的是什么客户端版本和协议版本
- 最近心跳是什么时候

#### Topic 列表主要看什么
- 当前有哪些 Topic
- 每个 Topic 的名称
- Payload 模式是什么
- Scope 类型是什么
- 保留时长是多久

### 什么时候用

- 排查终端是否在线
- 看某个 Topic 是否已经建好
- 看某个 Topic 是否适合当前业务

### 小白推荐操作

点击这些表格里的 `查看` 按钮，打开详情抽屉。  
这是最适合新手理解数据对象的方法。

---

### 区域四：Scope 统计 / Projection / Change Log

这是整个 TDP 页面最关键的观察区。

它一般分成 3 块：

- Scope 统计
- Projection 列表
- Change Log 列表

---

#### Scope 统计是干什么的？

告诉你当前不同 Topic / Scope 下大概有多少数据。

**什么时候看：**
- 你想知道当前系统里哪些 Topic 最活跃
- 你想知道数据大概分布在哪些范围里

---

#### Projection 列表是干什么的？

告诉你：
- 哪个 Topic
- 对哪个范围
- 当前 revision 是多少
- 最近更新时间是什么时候

**什么时候看：**
- 你刚发布了一个配置任务
- 你刚注入了一条 Projection
- 你想看当前终端最新应该看到什么数据

**最典型场景：**
发布配置任务后，先来这里确认是否真的写出了 Projection。

---

#### Change Log 列表是干什么的？

告诉你：
- 某次变更的 ID 是什么
- 对应哪个 Topic
- revision 变成了多少
- 来源发布单是谁
- 时间是什么时候

**什么时候看：**
- 你想追历史变化
- 你想验证某次任务是否真的产生了变更
- 你想对照 TCP 里的发布单看是否打通了链路

**最典型场景：**
点击 TCP 的“发布配置任务”之后，来这里看是否新出现了一条 Change Log。

---

### 区域五：当前选中快照

这里通常显示两块 JSON：
- 当前选中的 Projection
- 当前选中的 Change Log

### 什么时候看

- 你想看完整 JSON 内容
- 你要给开发或终端同学复制数据样例
- 你要做精确排查

### 小白建议

如果你只想快速理解：
- `Projection` 看“当前值”
- `Change Log` 看“这次变更怎么来的”

---

## 6.3 TDP 页面最推荐新手跑的一条链路

### 操作顺序

1. 先去 `TCP 控制面`
2. 点击“发布配置任务”
3. 再回到 `TDP 数据面`
4. 看 Projection 是否新增或 revision 是否变化
5. 看 Change Log 是否新增
6. 点击“查看”打开详情

### 你会学到什么

你会直观地明白：
> TCP 不是直接“让终端执行”，而是先把治理动作委托到 TDP，TDP 再把终端应该看到的数据写出来。

## 7. 场景引擎怎么用

### 对应截图

- `0-mock-server/mock-terminal-platform/docs/screenshots/04-scene.png:1`

---

## 7.1 场景引擎是干什么的？

场景引擎是“快速造真实联调环境”的地方。

你不需要手工一个个点很多按钮，只要运行一个场景模板，就能快速得到一套比较完整的数据变化。

---

## 7.2 页面里的内容怎么看

### 区域一：场景模板库

这里列出当前可运行的场景模板，比如：
- 门店批量上线
- 灰度升级演练

每条模板会告诉你：
- 模板名称
- 类别
- 描述
- 主要步骤
- 运行按钮

### 什么时候用

- 你刚拿到环境，想快速生成一套可联调数据
- 你要复现一个典型业务流程
- 你想让测试同学快速进入状态

---

### 区域二：Scene DSL 草案

这里目前更偏“设计展示”，告诉你后续场景是怎么抽象成脚本的。

### 小白怎么理解

你暂时可以把它理解成：
> 未来系统可能会把复杂场景变成一套可脚本化的流程定义。

首期使用后台时，不需要深究这块。

---

## 7.3 两个最常用场景怎么用

### 场景一：门店批量上线

**适合什么时候用：**
- 环境里数据太少
- 你想快速生成一批终端和一批任务
- 你想让后台看起来更像真实环境

**点完后应该去哪里看：**
- 回总览页，看终端总量是否增加
- 去 TCP 页面，看终端列表和任务发布是否增加
- 去 TDP 页面，看 Projection / Change Log 是否增加

---

### 场景二：灰度升级演练

**适合什么时候用：**
- 你在做热更新联调
- 你要模拟某些终端升级、某些终端不升级
- 你要观察升级目标是如何分发下去的

**点完后应该去哪里看：**
- TCP 页的任务发布和任务实例
- TDP 页的 Projection / Change Log
- 审计日志

---

## 7.4 场景页推荐给新手的操作顺序

### 动作 1：先运行“门店批量上线”
这是最容易看出变化的场景。

### 动作 2：去总览页看数字变化
确认场景真的生效了。

### 动作 3：去 TCP / TDP 看联动
理解这不是“只改了一页的数据”，而是整体联动。

## 8. 故障注入怎么用

### 对应截图

- `0-mock-server/mock-terminal-platform/docs/screenshots/05-fault.png:1`

---

## 8.1 故障注入页是干什么的？

这个页面是专门用来：
- 模拟异常
- 验证系统容错能力
- 给开发 / 测试制造真实问题场景

如果场景引擎像“搭舞台”，那故障注入就像“往舞台里制造麻烦”。

---

## 8.2 页面里的几个区域分别怎么用

### 区域一：故障注入控制台

这里主要有：
- 规则名称
- Matcher(JSON)
- Action(JSON)
- 新增故障规则
- 更新首条规则
- 模拟首条规则命中

---

### 什么是 Matcher？

Matcher 可以理解成：
> “这条故障规则应该匹配谁？”

比如：
- 只匹配 `APP_UPGRADE` 任务
- 只匹配某类配置下发
- 只匹配某种目标类型

---

### 什么是 Action？

Action 可以理解成：
> “匹配上之后，我要制造什么故障？”

比如：
- 延迟
- 超时
- 失败
- 伪结果

---

### 什么时候用“新增故障规则”？

- 你想新建一条异常场景
- 现有模板不满足你的测试需求

### 什么时候用“更新首条规则”？

- 你只是想在已有规则基础上调整一下参数
- 比如把延迟从 3 秒改成 10 秒

### 什么时候用“模拟首条规则命中”？

- 你想快速验证故障规则是否能生效
- 你不想等待真实链路自然命中

---

### 区域二：故障规则列表

这里会展示当前所有故障规则：
- 名称
- 目标类型
- Matcher
- Action
- 命中次数
- 更新时间

### 什么时候看这里

- 你想确认规则是否创建成功
- 你想确认某条规则最近有没有命中
- 你想回看规则当前内容

### 小白推荐操作

1. 先套用一个 Fault 模板
2. 来故障页看看表单有没有自动填好
3. 点击“新增故障规则”
4. 再点击“模拟首条规则命中”
5. 看命中次数有没有变化
6. 去审计日志看看有没有记录

---

### 区域三：调试原则

这里不是操作区，而是提醒区。  
它主要告诉你这个 Mock 平台的使用边界：

- TCP 和 TDP 不混职责
- Mock 能力是增强调试的，不是替代真实主链路的
- 所有异常都应尽量可观测、可追溯

### 小白建议

这一块不用背，但最好理解：
> 这里的很多操作是“为了联调方便”，不代表真实生产系统会允许这样随意改状态。

---

## 8.3 故障页推荐新手先跑的一条链路

### 操作顺序

1. 到总览页或模板库先套用一个 Fault 模板
2. 切到故障页
3. 看表单是否自动填好
4. 点击“新增故障规则”
5. 点击“模拟首条规则命中”
6. 看命中次数
7. 再去审计日志确认动作已记录

### 你会学到什么

你会理解：
> Mock 平台不仅能造正常数据，还能帮你模拟“不正常的数据和链路”。

---

## 9. 顶部 3 个通用按钮怎么用

这 3 个按钮在任何页面都能看到。

### 9.1 刷新数据

**作用：** 重新从后台拉取最新数据。

**什么时候用：**
- 刚执行完一个动作
- 页面看起来没变化
- 你怀疑自己看到的是旧数据

---

### 9.2 预览导出数据

**作用：** 在当前页面中显示导出的 JSON 数据。

**什么时候用：**
- 你想看当前环境里有哪些对象
- 你想快速核对数据结构
- 你想给开发同学贴一份数据示例

---

### 9.3 下载导出文件

**作用：** 把当前环境数据下载成文件。

**什么时候用：**
- 你要备份测试环境
- 你要把当前数据交给别的同学复现
- 你要留存某次测试前后的环境快照

---

## 10. 小白管理员最常见的 6 个工作场景

这一章最实用。你可以对号入座。

---

## 场景 1：我刚拿到环境，不知道从哪看起

### 建议操作
1. 打开 `总览`
2. 看运行状态卡片
3. 看默认沙箱
4. 看联调建议
5. 翻一页审计日志

### 目标
先知道环境里是不是已经有数据、最近有人做过什么。

---

## 场景 2：我想快速造一套联调环境

### 建议操作
1. 打开 `场景引擎`
2. 运行“门店批量上线”
3. 回到 `总览` 看数字变化
4. 去 `TCP 控制面` 看终端和任务
5. 去 `TDP 数据面` 看 Projection / Change Log

### 目标
快速得到“像真的一样”的联调数据。

---

## 场景 3：我想测试终端激活

### 建议操作
1. 打开 `TCP 控制面`
2. 点击“生成 5 个激活码”
3. 在激活区填一个激活码
4. 点击“执行终端激活”
5. 看终端列表和审计日志

### 目标
验证终端首次接入流程。

---

## 场景 4：我想测试配置下发

### 建议操作
1. 打开 `TCP 控制面`
2. 点击“发布配置任务”
3. 看任务发布和任务实例
4. 切到 `TDP 数据面`
5. 看 Projection / Change Log 是否更新

### 目标
验证 TCP -> TDP 联动主链路。

---

## 场景 5：我想测试终端断线或连接变化

### 建议操作
1. 打开 `TDP 数据面`
2. 在 Session 区输入终端 ID
3. 点击“建立 Session”
4. 看会话列表
5. 点击“发送心跳”
6. 再点击“断开首个 Session”

### 目标
验证终端连接生命周期。

---

## 场景 6：我想模拟异常

### 建议操作
1. 打开 `故障注入`
2. 套用一个 Fault 模板或直接填规则
3. 点击“新增故障规则”
4. 点击“模拟首条规则命中”
5. 看规则命中次数
6. 看审计日志

### 目标
验证异常场景下系统是否能正常观测和联调。

---

## 11. 推荐给小白管理员的后台操作路线

如果你是第一次正式使用后台，我建议按下面顺序走一遍：

### 第 1 步：看总览
先认识系统整体情况。

### 第 2 步：在 TCP 页面发布配置任务
理解控制面。

### 第 3 步：在 TDP 页面看 Projection / Change Log
理解数据面。

### 第 4 步：在场景页运行一个模板
理解快速造数。

### 第 5 步：在故障页模拟一次命中
理解异常联调。

这 5 步走完，你基本就能上手这个后台了。

---

## 12. 小白管理员最容易犯的错误

### 错误 1：只看 TCP，不看 TDP
这样你只能看到“任务发出去了”，却不知道“终端视角的数据是否真的出来了”。

### 错误 2：只看 TDP，不看 TCP
这样你能看到数据变了，却不知道“这次变更是哪个发布单引起的”。

### 错误 3：点完按钮不刷新也不看审计
这样很容易误以为系统没反应。

### 错误 4：故障规则加了，但没回看命中次数和日志
这样你无法确认故障是否真的生效。

### 错误 5：一上来就自己手写复杂 JSON
建议先套模板，再逐步改。

---

## 13. 给小白管理员的最后建议

你可以用一句话记住后台的使用方式：

- **总览**：看全局
- **TCP**：做治理
- **TDP**：看投递
- **场景**：快造数
- **故障**：造异常

如果你每次操作后都记得：

1. 看当前页结果
2. 看另一侧联动页结果
3. 看审计日志

那你基本就不会迷路。
