# 12-分布式事务与一致性设计

## 1. 概述

TCP 服务涉及多个关键流程需要分布式事务保证：
- **激活流程**：激活码消费、终端创建、凭证生成
- **换绑流程**：旧关系解除、新关系建立、凭证更新
- **任务发布流程**：任务创建、实例生成、TDP投递
- **结果回报流程**：任务状态更新、终端状态更新、统计更新

本文档定义分布式事务处理机制，确保数据一致性。

## 2. 事务处理策略

### 2.1 Saga 模式

对于跨多个聚合根的长事务，使用 Saga 模式：

```
Saga 模式特点：
- 将长事务拆分为多个本地事务
- 每个本地事务更新一个聚合根并发布事件
- 通过补偿操作处理失败场景
- 最终一致性保证
```

### 3.1 事务边界

```

├─────────────────────────────────────────────────────────────┤
│ 步骤2: 本地事务1 - 创建/更新终端实例                         │
│   - 创建 TerminalInstance                                    │
│   - 发布 TerminalCreatedEvent 到 outbox                      │
│   - 提交事务                                                 │
├─────────────────────────────────────────────────────────────┤
│ 步骤3: 本地事务2 - 消费激活码                                │
│   - 更新 ActivationCode.usedCount                           │
│   - 更新 ActivationCode.status                              │
│   - 发布 ActivationCodeUsedEvent 到 outbox                  │
│   - 提交事务                                                 │
├─────────────────────────────────────────────────────────────┤
│ 步骤4: 本地事务3 - 生成凭证                                  │
│   - 创建 TerminalCredential                                 │
│   - 提交事务                                                 │
├─────────────────────────────────────────────────────────────┤
│ 步骤5: 异步 - 发布事件到消息队列                             │
│   - OutboxEventPublisher 轮询 outbox 表                     │

└─────────────────────────────────────────────────────────────┘
```

### 3.2 补偿操作

```java
/**
 * 激活流程补偿服务
 */
@Service
public class ActivationCompensationService {

    /**
     * 补偿操作：撤销激活
     */
    @Transactional
    public void compensateActivation(String terminalId, String activationCodeId) {
        // 1. 删除终端实例（软删除）
        TerminalInstance terminal = terminalInstanceRepository
            .findById(terminalId)
            .orElseThrow(() -> new TerminalNotFoundException());

        terminal.markDeleted("激活失败补偿");
        terminalInstanceRepository.save(terminal);

        // 2. 释放激活码
        ActivationCode code = activationCodeRepository
            .findById(activationCodeId)
            .orElseThrow(() -> new ActivationCodeNotFoundException());

        code.releaseUsage();  // usedCount--
        activationCodeRepository.save(code);

        // 3. 删除凭证
        terminalCredentialService.revokeCredential(terminalId);

        // 4. 记录补偿日志
        log.warn("激活流程补偿完成: terminalId={}, activationCodeId={}",
            terminalId, activationCodeId);
    }
}
```

### 3.3 幂等性设计

```java
/**
 * 激活流程幂等性保证
 */
@Service
public class ActivationApplicationService {

    @Transactional
    public ActivateTerminalResponse activateTerminal(ActivateTerminalRequest request) {
        // 1. 幂等键：deviceFingerprint
        String idempotentKey = "activation:idempotent:" + request.getDeviceFingerprint();

        // 2. 检查是否已激活
        String existingTerminalId = redisTemplate.opsForValue().get(idempotentKey);
        if (existingTerminalId != null) {
            // 返回已有结果
            return getActivationResult(existingTerminalId);
        }

        // 3. 使用分布式锁防止并发激活
        String lockKey = "activation:lock:" + request.getActivationCode();
        RLock lock = redissonClient.getLock(lockKey);

        try {
            // 尝试获取锁，最多等待10秒，锁持有30秒
            if (!lock.tryLock(10, 30, TimeUnit.SECONDS)) {
                throw new ConcurrentActivationException("激活码正在被使用");
            }

            // 4. 执行激活流程
            ActivateTerminalResponse response = doActivate(request);

            // 5. 设置幂等键（24小时）
            redisTemplate.opsForValue().set(
                idempotentKey,
                response.getTerminalId(),
                24,
                TimeUnit.HOURS
            );

            return response;

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new ActivationException("激活被中断");
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}
```

### 4.2 补偿操作

```java
/**
 * 换绑流程补偿服务
 */
@Service
public class RebindCompensationService {

    /**
     * 补偿操作：回滚换绑
     */
    @Transactional
    public void compensateRebind(
            String terminalId,
            String oldStoreId,
            String oldProfileId,
            String oldTemplateId) {

        // 1. 恢复终端归属
        TerminalInstance terminal = terminalInstanceRepository
            .findById(terminalId)
            .orElseThrow(() -> new TerminalNotFoundException());

        terminal.rollbackRebind(oldStoreId, oldProfileId, oldTemplateId);
        terminalInstanceRepository.save(terminal);

        // 2. 撤销新凭证
        terminalCredentialService.revokeLatestCredential(terminalId);

        // 3. 恢复旧凭证
        terminalCredentialService.restorePreviousCredential(terminalId);

        // 4. 取消配置刷新任务
        taskReleaseService.cancelPendingTasks(terminalId, TaskType.CONFIG_PUBLISH);

        // 5. 记录补偿日志
        log.warn("换绑流程补偿完成: terminalId={}, 恢复到 storeId={}",
            terminalId, oldStoreId);
    }
}
```

### 5.2 幂等性设计

```java
/**
 * 任务发布幂等性保证
 */
@Service
public class TaskReleaseApplicationService {

    @Transactional
    public CreateTaskReleaseResponse createTaskRelease(CreateTaskReleaseRequest request) {
        // 1. 幂等键：releaseId（客户端生成）
        String releaseId = request.getReleaseId();
        if (releaseId == null) {
            releaseId = IdGenerator.generateReleaseId();
        }

        // 2. 检查是否已存在
        Optional<TaskRelease> existing = taskReleaseRepository.findById(releaseId);
        if (existing.isPresent()) {
            // 返回已有结果
            return buildResponse(existing.get());
        }

        // 3. 创建任务发布单（使用客户端提供的 releaseId）
        TaskRelease release = TaskRelease.create(
            releaseId,  // 使用确定的 ID
            request.getTaskType(),
            request.getTaskName(),
            request.getTargetScope(),
            request.getPayload(),
            request.getScheduledAt(),
            request.getCreatedBy()
        );

        taskReleaseRepository.save(release);

        return buildResponse(release);
    }
}
```

### 5.3 补偿操作

```java
/**
 * 任务发布补偿服务
 */
@Service
public class TaskReleaseCompensationService {

    /**
     * 补偿操作：取消任务发布
     */
    @Transactional
    public void compensateTaskRelease(String releaseId) {
        // 1. 查询任务发布单
        TaskRelease release = taskReleaseRepository
            .findById(releaseId)
            .orElseThrow(() -> new TaskReleaseNotFoundException());

        // 2. 取消任务发布
        release.cancel("发布失败补偿");
        taskReleaseRepository.save(release);

        // 3. 取消所有待执行的任务实例
        List<TerminalTaskInstance> instances = terminalTaskInstanceRepository
            .findByReleaseIdAndStatus(releaseId, TaskInstanceStatus.PENDING);

        for (TerminalTaskInstance instance : instances) {
            instance.cancel("发布单取消");
        }
        terminalTaskInstanceRepository.saveAll(instances);

        // 4. 通知 TDP 取消投递
        tdpClient.cancelTaskDelivery(releaseId);

        // 5. 记录补偿日志
        log.warn("任务发布补偿完成: releaseId={}", releaseId);
    }
}
```

### 6.2 幂等性设计

```java
/**
 * 结果回报幂等性保证
 */
@Service
public class TaskResultApplicationService {

    @Transactional
    public void reportTaskResult(ReportTaskResultRequest request) {
        // 1. 查询任务实例
        TerminalTaskInstance instance = terminalTaskInstanceRepository
            .findById(request.getTaskInstanceId())
            .orElseThrow(() -> new TaskInstanceNotFoundException());

        // 2. 幂等性检查：如果已经是终态，直接返回
        if (instance.isTerminalStatus()) {
            log.info("任务实例已处于终态，跳过: taskInstanceId={}, status={}",
                instance.getTaskInstanceId(), instance.getStatus());
            return;
        }

        // 3. 使用乐观锁更新任务实例
        if (request.getStatus() == TaskInstanceStatus.SUCCESS) {
            instance.markSuccess(request.getResult());
        } else {
            instance.markFailed(request.getErrorCode(), request.getErrorMessage());
        }

        try {
            terminalTaskInstanceRepository.save(instance);
        } catch (OptimisticLockException e) {
            // 乐观锁冲突，重新查询并检查状态
            log.warn("乐观锁冲突，重新检查任务状态: taskInstanceId={}",
                instance.getTaskInstanceId());
            throw new ConcurrentUpdateException("任务状态已被其他请求更新");
        }

        // 4. 更新终端状态（如果需要）
        if (shouldUpdateTerminalStatus(instance)) {
            updateTerminalStatus(instance);
        }

        // 5. 更新发布单统计
        updateReleaseStats(instance.getReleaseId());
    }
}
```

## 7. 并发控制策略

### 7.1 分布式锁

使用 Redis 分布式锁防止并发冲突：

```java
/**
 * 分布式锁工具
 */
@Component
public class DistributedLockService {

    @Autowired
    private RedissonClient redissonClient;

    /**
     * 执行带锁的操作
     */
    public <T> T executeWithLock(
            String lockKey,
            long waitTime,
            long leaseTime,
            TimeUnit unit,
            Supplier<T> action) {

        RLock lock = redissonClient.getLock(lockKey);

        try {
            if (!lock.tryLock(waitTime, leaseTime, unit)) {
                throw new LockAcquisitionException("获取锁失败: " + lockKey);
            }

            return action.get();

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new LockAcquisitionException("获取锁被中断: " + lockKey);
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}
```

**使用场景：**

1. **激活码消费**：防止并发超卖
   ```java
   String lockKey = "activation:lock:" + activationCode;
   distributedLockService.executeWithLock(
       lockKey, 10, 30, TimeUnit.SECONDS,
       () -> consumeActivationCode(activationCode)
   );
   ```

2. **任务发布**：防止重复发布
   ```java
   String lockKey = "task:release:lock:" + releaseId;
   distributedLockService.executeWithLock(
       lockKey, 5, 20, TimeUnit.SECONDS,
       () -> executeTaskRelease(releaseId)
   );
   ```

## 8. 事件发布保证

### 8.1 Outbox 模式

```java
/**
 * Outbox 事件发布器
 */
@Component
public class OutboxEventPublisher {

    @Autowired
    private OutboxEventRepository outboxEventRepository;

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    /**
     * 定时任务：发布待发布的事件
     */
    @Scheduled(fixedDelay = 1000)  // 每秒执行一次
    public void publishPendingEvents() {
        // 1. 查询待发布的事件（限制100条）
        List<OutboxEvent> events = outboxEventRepository
            .findByStatusOrderByCreatedAtAsc(EventStatus.PENDING, PageRequest.of(0, 100));

        if (events.isEmpty()) {
            return;
        }

        // 2. 批量发布
        for (OutboxEvent event : events) {
            try {
                // 发布到 RocketMQ
                rocketMQTemplate.convertAndSend(
                    event.getEventType(),  // topic
                    event.getEventPayload()
                );

                // 标记为已发布
                event.markPublished();
                outboxEventRepository.save(event);

            } catch (Exception e) {
                // 增加重试次数
                event.incrementRetryCount();

                // 超过最大重试次数，标记为失败
                if (event.getRetryCount() >= 10) {
                    event.markFailed();
                }

                outboxEventRepository.save(event);

                log.error("发布事件失败: eventId={}, retryCount={}",
                    event.getEventId(), event.getRetryCount(), e);
            }
        }
    }
}
```

## 9. 本文结论

分布式事务与一致性设计展示了完整的事务处理机制：

- **Saga 模式**：将长事务拆分为多个本地事务，通过补偿操作处理失败
- **本地消息表**：使用 Outbox 模式保证事件最终发布
- **幂等性设计**：使用唯一键、状态机、分布式锁保证操作幂等
- **补偿操作**：提供撤销激活、回滚换绑、取消任务等补偿机制
- **并发控制**：分布式锁（激活码）、乐观锁（任务实例）、悲观锁（换绑）
- **事件发布保证**：Outbox 模式 + 定时轮询 + 重试机制

所有关键流程都有明确的事务边界、补偿逻辑和幂等性保证。
