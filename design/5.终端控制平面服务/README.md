## 设计原则

### 1. 主数据与运行态分离
- **主数据**：终端是谁、机型是什么、归属哪里、能力有哪些
- **运行态**：当前在线吗、当前版本是什么、当前配置版本是什么、最后任务结果是什么

### 2. "定义任务"与"投递任务"分离
- TCP 负责任务的定义、目标圈选、发布窗口、灰度批次、结果追踪
- TDP 负责实际异步投递

### 3. 统一任务模型
- 配置下发、热更新、远程控制统一抽象为控制面任务
- 任务模板 → 任务发布单 → 终端任务实例 → 执行结果

### 4. 发布与结果闭环
- 定义目标 → 生成发布单 → 生成终端实例任务 → 委托 TDP 投递 → 终端执行 → 回报结果 → 汇总状态

## 与其他服务的关系

### TCP vs TDP

- **TCP（控制面）**：管终端是谁、该做什么、做得怎么样
- **TDP（数据面）**：管如何可靠送达、如何补偿、如何感知在线

### TCP vs 业务系统

- TCP 不承载业务数据（订单、菜单、工单等）
- TCP 只管理终端资产和控制面任务
- 业务系统通过 TDP 推送业务数据到终端
