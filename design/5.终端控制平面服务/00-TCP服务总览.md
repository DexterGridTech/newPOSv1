# 00-TCP服务总览

## 1. 服务概述

终端控制平面服务（Terminal Control Plane，简称 TCP）是购物中心餐饮中台的**终端资产管理与运维控制中心**，负责终端设备的全生命周期管理、配置管理、版本管理和远程控制。

**核心职责**：
- 终端设备主数据管理（设备型号、设备模板、终端实例）
- 终端生命周期管理（注册、激活、绑定、换绑、退役）
- 终端身份与凭证管理（激活码、访问令牌、权限控制）
- 终端配置管理（配置模型、配置版本、配置发布、回滚）
- 终端版本管理（应用版本、热更新包、灰度发布、回滚）
- 终端远程控制（重启、日志上传、强制同步、锁屏等）
- 任务执行追踪（任务状态、执行结果、审计日志）
- 运维后台支持（资产视图、任务视图、发布视图、审计视图）

**核心聚合根**：
- TerminalInstance（终端实例）
- DeviceProfile（设备能力模板）
- TerminalTemplate（终端模板）
- ConfigVersion（配置版本）
- AppVersion（应用版本）
- ControlTask（控制任务）
- ActivationCode（激活码）

**服务端口**：8092
**数据库**：tcp_db

## 2. 服务定位与战略价值

### 2.1 为什么需要 TCP

在企业级餐饮终端体系中，除了业务数据分发（TDP 负责），还存在另一类同样关键但性质完全不同的问题：

**问题一：终端不是"匿名客户端"，而是需要被持续治理的资产**
- POS、KDS、PDA、叫号屏、自助机都不是一次性连接体
- 每个终端都有机型、能力、版本、门店归属、安装位置、责任人、状态等长期属性
- 这些属性不适合放在业务系统里零散维护

**问题二：终端侧存在大量"控制面动作"，不属于业务数据平面**
- 激活、绑定、换绑、退役
- 配置参数变更
- 热更新发布、灰度、回滚
- 远程控制（重启、日志上传、强制同步、锁屏等）
- 运维审计与任务结果追踪

**问题三：控制动作需要权威管理，但不一定需要实时直连投递**
- 配置下发、热更新、远控任务等，本质是"管理任务"
- 这些任务的定义、审批、灰度、审计应由管理中心统一控制
- 但具体投递可以复用 TDP 的离线补偿、终端在线感知、可靠推送能力

**问题四：原有 POC 把设备、终端、激活、远控等能力混在同一个 mock 服务中，边界不清**
- 终端资产管理与终端数据分发混在一起，会导致职责失控
- 热更新、配置、远控、主数据、在线会话、业务投影混在一起后，后续很难演进到正式生产形态

### 2.2 TCP 与 TDP 的关系

二者并列而非包含关系：

- **TCP 是控制面（Control Plane）**
  - 管终端是谁
  - 管终端属于谁
  - 管终端应该用什么配置和什么版本
  - 管终端应该执行什么运维任务
  - 管任务审计、灰度、回滚、结果

- **TDP 是数据面（Data Plane）**
  - 管终端如何接收被动下行数据
  - 管终端在线连接与离线补偿
  - 管 Revision / Cursor / Replay / Snapshot
  - 管 Topic 订阅与消息投递

一句话总结：

- **TCP 决定发什么、发给谁、为什么发**
- **TDP 负责怎么送达、怎么补偿、怎么感知在线终端**

### 2.3 TCP 在中台中的位置

```
终端（POS/KDS/PDA/KIOSK/Display/App）
        │
        ├──── 主动业务请求（HTTP） ────► 各业务系统
        │
        ├──── 激活 / 绑定 / 结果回报（HTTP） ────► TCP
        │
        └──── 控制面异步任务接收（WS/推送） ◄──── TDP
                                      ▲
                                      │
                         TCP 生成控制任务并委托 TDP 投递
```

### 2.4 TCP 明确不做的事情

1. 不承载订单、菜单、工单等业务投影数据
2. 不负责高频业务数据广播
3. 不替代 TDP 的 Session / Sync / Projection / Replay 协议
4. 不作为所有业务系统的统一网关
5. 不直接控制终端连接层，不替代 TDP 的可靠投递引擎

## 3. 领域边界

### 3.1 属于 TCP 的对象

- 设备型号（VendorModel）
- 设备能力模板（DeviceProfile）
- 终端模板（TerminalTemplate）
- 终端实例（TerminalInstance）
- 终端安装关系（TerminalInstallation）
- 激活码 / 激活单（ActivationCode / ActivationRecord）
- 终端凭证 / Token（TerminalCredential）
- 配置模型 / 配置版本 / 配置发布（ConfigModel / ConfigVersion / ConfigRelease）
- 应用版本 / 热更新包 / 升级发布（AppVersion / UpdatePackage / UpgradeRelease）
- 远程控制任务（RemoteControlTask）
- 任务执行记录（TaskExecutionRecord）
- 运维审计记录（AuditLog）

### 3.2 属于 TDP 的对象

- Session
- Topic 订阅
- Projection Store
- Change Log
- Replay Cursor
- 在线推送 / 离线补偿
- 点对点异步控制消息的最终投递

### 3.3 属于业务系统的对象

- 订单
- 菜单
- 履约工单
- 预订数据
- 支付数据
- 商品库存
- 营销活动

## 4. 技术架构

### 4.1 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                      接入层 (Controller Layer)                │
│  REST API + WebSocket (激活、配置、版本、远控、审计)          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    应用服务层 (Application Layer)             │
│  ├─ TerminalApplicationService (终端管理)                    │
│  ├─ ActivationApplicationService (激活绑定)                  │
│  ├─ ConfigApplicationService (配置管理)                      │
│  ├─ VersionApplicationService (版本管理)                     │
│  ├─ ControlTaskApplicationService (远程控制)                 │
│  └─ AuditApplicationService (审计查询)                       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    领域层 (Domain Layer)                      │
│  聚合根 + 实体 + 值对象 + 领域服务 + 领域事件                  │
│  (纯业务逻辑，不依赖基础设施)                                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                  基础设施层 (Infrastructure Layer)             │
│  ├─ 数据持久化: PostgreSQL + Citus (分片)                     │
│  ├─ 缓存: Redis Cluster (终端状态、在线会话)                  │
│  ├─ 消息队列: RocketMQ (事件驱动)                            │
│  ├─ TDP 客户端: 任务委托、状态同步                            │
│  ├─ 对象存储: MinIO / OSS (更新包、日志文件)                 │
│  └─ 服务治理: Nacos + Sentinel                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 核心技术栈

| 技术分类 | 技术选型 | 版本 | 选型理由 |
|---------|---------|------|---------|
| 编程语言 | Java | 17 LTS | 与中台主体一致 |
| 应用框架 | Spring Boot | 3.2.x | 微服务标准框架 |
| 数据库 | PostgreSQL + Citus | 16.x + 12.x | 分布式能力、JSON 支持 |
| 消息队列 | Apache RocketMQ | 5.1.x | 事务消息、延迟消息 |
| 缓存 | Redis Cluster | 7.2.x | 分布式缓存 |
| 对象存储 | MinIO / OSS | Latest | 文件存储 |
| 服务注册 | Nacos | 2.3.x | 服务注册 + 配置中心 |
| 流量控制 | Sentinel | 1.8.x | 限流熔断降级 |

### 4.3 数据存储方案

**PostgreSQL + Citus 分片**：
- 按 `store_id` 分片（与业务系统一致）
- 支持水平扩展
- 支持 JSONB 存储复杂配置

**Redis Cluster**：
- 终端在线状态缓存
- 任务执行状态缓存
- 激活码使用次数计数
- 分布式锁

**MinIO / OSS**：
- 热更新包存储
- 终端日志文件存储
- 配置文件备份

## 5. 核心业务流程

### 5.1 终端激活流程

```
1. 运维后台生成激活码
2. 终端安装后输入/扫码激活码
3. 终端调用 TCP POST /api/v1/activation/activate
4. TCP 校验激活码、设备指纹、使用次数、有效期
5. TCP 创建或绑定 TerminalInstance
6. TCP 生成控制面凭证
7. TCP 返回：terminalId、归属信息、profile/template、初始 token、TDP 接入材料
8. 终端保存本地身份材料
9. TCP 发布 TerminalActivated 事件
```

### 5.2 配置发布流程

```
1. 运维后台创建配置版本
2. 运维后台创建配置发布单（指定目标终端、灰度策略）
3. TCP 生成终端级配置任务实例
4. TCP 委托 TDP 投递配置任务
5. 终端接收配置任务并应用
6. 终端通过 HTTP 回报 TCP 应用结果
7. TCP 汇总状态并支持回滚/重试
8. TCP 发布 ConfigApplied 事件
```

### 5.3 热更新发布流程

```
1. 运维后台上传热更新包到对象存储
2. 运维后台创建应用版本记录
3. 运维后台创建升级发布单（指定目标终端、灰度策略）
4. TCP 生成终端级升级任务实例
5. TCP 委托 TDP 投递升级任务
6. 终端接收升级任务并下载更新包
7. 终端应用更新并重启
8. 终端通过 HTTP 回报 TCP 升级结果
9. TCP 汇总状态并支持回滚/重试
10. TCP 发布 AppUpgraded 事件
```

### 5.4 远程控制流程

```
1. 运维后台发起远程控制任务（重启、日志上传、强制同步等）
2. TCP 创建远程控制任务实例
3. TCP 委托 TDP 投递控制命令
4. 终端接收控制命令并执行
5. 终端通过 HTTP 回报 TCP 执行结果
6. TCP 记录执行结果并支持审计查询
7. TCP 发布 RemoteControlExecuted 事件
```

## 6. 核心设计理念

### 6.1 主数据与运行态分离

- **主数据**：终端是谁、机型是什么、归属哪里、能力有哪些
- **运行态**：当前在线吗、当前版本是什么、当前配置版本是什么、最后任务结果是什么

主数据由 TCP 权威维护；运行态由终端回报、TDP 同步、TCP 汇总。

### 6.2 "定义任务"与"投递任务"分离

TCP 负责任务的定义、目标圈选、发布窗口、灰度批次、结果追踪；TDP 负责实际异步投递。

这样可以保证：
- 控制逻辑统一
- 投递链路复用 TDP 的可靠机制
- TCP 不需要自己维护大规模长连接

### 6.3 统一任务模型

配置下发、热更新、远程控制，本质上都是"面向终端的控制面任务"。

因此统一抽象为：
- 任务模板（TaskTemplate）
- 任务发布单（TaskRelease）
- 终端任务实例（TerminalTaskInstance）
- 执行结果（TaskExecutionResult）

仅任务类型不同：
- `CONFIG_PUBLISH`
- `APP_UPGRADE`
- `REMOTE_CONTROL`

### 6.4 发布与结果闭环

完整闭环必须包含：
1. 定义目标
2. 生成发布单
3. 生成终端实例任务
4. 委托 TDP 投递
5. 终端接收并择机执行
6. 终端通过 HTTP 回报 TCP
7. TCP 汇总状态并支持回滚/重试/审计

### 6.5 灰度发布策略

支持多种灰度策略：
- 按百分比灰度（10% → 50% → 100%）
- 按终端列表灰度（指定 terminalId）
- 按门店灰度（指定 storeId）
- 按标签灰度（指定 tag）
- 按时间窗口灰度（指定发布时间）

### 6.6 版本管理策略

- 应用版本采用语义化版本号（Semantic Versioning）：`major.minor.patch`
- 配置版本采用时间戳 + 序列号：`cfg-20260406-001`
- 热更新包采用 Bundle 版本号：`2026.04.06.001`
- 支持版本回滚（回滚到上一个稳定版本）
- 支持版本强制更新（安全漏洞修复）

## 7. 与 TDP 协同协议

### 7.1 任务委托协议

TCP 通过 HTTP API 向 TDP 提交控制任务：

```http
POST /api/v1/tdp/tasks
Content-Type: application/json

{
  "taskId": "task-001",
  "taskType": "CONFIG_PUBLISH",
  "targetScope": {
    "orgScopeId": "store:s-001",
    "profileScopeId": "device-model:pos-standard",
    "terminalIds": ["pos-001", "pos-002"]
  },
  "payload": {
    "configVersion": "cfg-20260406-001",
    "configData": {...}
  },
  "priority": "NORMAL",
  "expiresAt": "2026-04-07T00:00:00Z"
}
```

### 7.2 状态同步协议

TDP 通过 WebHook 向 TCP 同步终端在线状态：

```http
POST /api/v1/tcp/terminal-status
Content-Type: application/json

{
  "terminalId": "pos-001",
  "status": "ONLINE",
  "lastSeenAt": "2026-04-06T10:00:00Z",
  "sessionId": "session-001"
}
```

### 7.3 结果回报协议

终端通过 HTTP API 向 TCP 回报任务执行结果：

```http
POST /api/v1/tcp/task-results
Content-Type: application/json

{
  "taskId": "task-001",
  "terminalId": "pos-001",
  "status": "SUCCESS",
  "executedAt": "2026-04-06T10:05:00Z",
  "result": {
    "appliedConfigVersion": "cfg-20260406-001"
  }
}
```

## 9. 监控与告警

### 9.1 监控指标

**业务指标**：
- 激活成功率
- 配置发布成功率
- 热更新成功率
- 远程控制成功率
- 任务执行平均时长
- 终端在线率

**技术指标**：
- JVM 指标（堆内存、GC、线程）
- 接口 QPS、响应时间、错误率
- 数据库连接数、慢查询
- 缓存命中率
- 消息队列积压

### 9.2 告警规则

**P0 告警（紧急）**：
- 激活成功率 < 95%
- 配置发布失败率 > 5%
- 服务不可用
- 数据库连接池耗尽

**P1 告警（高）**：
- 接口响应时间 P99 > 1000ms
- 热更新失败率 > 10%
- 终端在线率 < 80%
- 消息队列积压 > 10000

**P2 告警（中）**：
- JVM 堆内存使用率 > 80%
- 数据库慢查询 > 100 次/分钟
- 磁盘使用率 > 80%

## 10. 安全设计

### 10.1 认证与授权

**终端认证**：
- 激活时颁发 JWT Token
- Token 包含 terminalId、storeId、profileId
- Token 有效期 7 天，支持刷新
- 退役终端的 Token 立即失效

**管理后台认证**：
- RBAC 权限控制
- 平台管理员、项目运维、租户运维、门店运维
- 敏感操作（发布、回滚）需要二次确认

### 10.2 数据安全

**敏感数据加密**：
- 设备序列号脱敏
- 激活码加密存储
- 配置敏感字段加密（数据库密码、API Key）

**审计日志**：
- 所有敏感操作记录审计日志
- 审计日志包含操作人、操作时间、操作内容、操作结果
- 审计日志保留 1 年

- **终端主数据管理**：设备型号、设备模板、终端实例

- **配置管理**：配置模型、配置版本、配置发布、回滚
- **版本管理**：应用版本、热更新包、灰度发布、回滚
- **远程控制**：重启、日志上传、强制同步、锁屏等
- **任务追踪**：任务状态、执行结果、审计日志

TCP 与 TDP 协同工作：
- **TCP 决定发什么、发给谁、为什么发**
- **TDP 负责怎么送达、怎么补偿、怎么感知在线终端**
