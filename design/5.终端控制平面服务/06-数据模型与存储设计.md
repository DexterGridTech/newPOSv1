# 06-数据模型与存储设计

## 1. 数据库架构

### 1.1 技术选型

- **数据库**：PostgreSQL 16.x + Citus 12.x
- **分片策略**：按 `store_id` 分片（与业务系统一致）
- **连接池**：HikariCP
- **ORM**：Spring Data JPA + Hibernate

### 1.2 分片设计

```sql
-- 设置分片键
SELECT create_distributed_table('terminal_instances', 'store_id');
SELECT create_distributed_table('config_versions', 'project_id');
SELECT create_distributed_table('app_versions', 'project_id');
SELECT create_distributed_table('control_tasks', 'created_by');
```

## 2. 核心表结构

### 2.1 终端实例表

```sql
CREATE TABLE terminal_instances (
    terminal_id VARCHAR(64) PRIMARY KEY,

    -- 设备信息
    device_id VARCHAR(128) NOT NULL,
    device_sn VARCHAR(128),
    device_fingerprint VARCHAR(256) NOT NULL,

    -- 组织归属
    project_id VARCHAR(64) NOT NULL,
    tenant_id VARCHAR(64) NOT NULL,
    brand_id VARCHAR(64),
    store_id VARCHAR(64) NOT NULL,

    -- 模板与配置
    profile_id VARCHAR(64) NOT NULL,
    template_id VARCHAR(64),

    -- 状态
    status VARCHAR(32) NOT NULL,
    connect_status VARCHAR(32) NOT NULL,
    health_status VARCHAR(32) NOT NULL,

    -- 版本信息
    current_app_version VARCHAR(32),
    current_bundle_version VARCHAR(32),
    current_config_version VARCHAR(64),

    -- 时间信息
    installed_at TIMESTAMP,
    activated_at TIMESTAMP,
    last_seen_at TIMESTAMP,
    retired_at TIMESTAMP,

    -- 标签与能力
    tags JSONB,
    capabilities JSONB,

    -- 审计字段
    version INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_terminal_store ON terminal_instances(store_id);
CREATE INDEX idx_terminal_profile ON terminal_instances(profile_id);
CREATE INDEX idx_terminal_status ON terminal_instances(status);
CREATE INDEX idx_terminal_fingerprint ON terminal_instances(device_fingerprint);
CREATE INDEX idx_terminal_last_seen ON terminal_instances(last_seen_at);
```

### 2.2 设备能力模板表

```sql
CREATE TABLE device_profiles (
```

### 2.3 激活码表

```sql
CREATE TABLE activation_codes (
    activation_code_id VARCHAR(64) PRIMARY KEY,
```

### 2.8 应用版本表

```sql
CREATE TABLE app_versions (
```

## 3. 数据库配置

### 3.1 application.yml

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/tcp_db
    username: tcp_user
    password: ${DB_PASSWORD}
    driver-class-name: org.postgresql.Driver
    hikari:
      maximum-pool-size: 200
      minimum-idle: 10
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000

  jpa:
    database-platform: org.hibernate.dialect.PostgreSQLDialect
    hibernate:
      ddl-auto: validate
    properties:
      hibernate:
        format_sql: true
        use_sql_comments: true
        jdbc:
          batch_size: 50
        order_inserts: true
        order_updates: true
```

## 4. 本文结论

数据模型与存储设计展示了完整的数据库实现：

- **12 张核心表**：覆盖终端管理、配置管理、版本管理、远程控制
- **分片策略**：按 `store_id` 分片，支持水平扩展
- **索引优化**：针对高频查询场景建立索引
- **JSONB 支持**：灵活存储配置数据、标签、能力等
- **审计追踪**：完整的审计日志记录
