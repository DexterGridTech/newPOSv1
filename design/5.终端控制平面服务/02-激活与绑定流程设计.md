# 02-激活与绑定流程设计

## 1. 激活与绑定概述

激活与绑定是终端接入 TCP 的第一道门，决定了：
- 终端是谁（terminalId）
- 终端归属哪里（projectId, tenantId, brandId, storeId）
- 终端使用什么模板（profileId, templateId）
- 终端拿什么身份去访问后续服务（token, credentials）

---

## 2. 终端首次激活流程

### 2.1 流程概览

```
1. 运维后台生成激活码批次
2. 运维后台分配激活码到具体门店/模板
3. 终端安装后输入/扫码激活码
4. 终端调用 TCP POST /api/v1/activation/activate
5. TCP 校验激活码（有效期、使用次数、状态）
6. TCP 校验设备指纹（防止重复激活）

【事务边界1：创建/更新终端实例】
7. TCP 创建或更新 TerminalInstance
   - 使用分布式锁防止并发激活（lockKey: activation:lock:{activationCode}）
   - 发布 TerminalCreatedEvent 到 outbox 表
   - 提交事务

【事务边界2：消费激活码】
8. TCP 标记激活码已使用
   - 更新 ActivationCode.usedCount
   - 更新 ActivationCode.status
   - 发布 ActivationCodeUsedEvent 到 outbox 表
   - 提交事务

【事务边界3：生成凭证】
9. TCP 生成终端凭证（JWT Token）
   - 创建 TerminalCredential
   - 提交事务

10. TCP 返回激活结果（terminalId、归属信息、token、TDP 接入材料）
11. 终端保存本地身份材料
12. TCP 异步发布事件到消息队列（OutboxEventPublisher）

【补偿操作】
- 如果步骤7-9任一失败，执行补偿：
  - 删除终端实例（软删除）
  - 释放激活码使用次数
  - 删除凭证
  - 详见：12-分布式事务与一致性设计.md
```

### 2.4 激活失败场景

| 错误码 | 错误信息 | 原因 |
|-------|---------|------|
| 40001 | 激活码不存在 | 激活码无效 |
| 40002 | 激活码已过期 | 超过有效期 |
| 40003 | 激活码已使用 | 使用次数达到上限 |
| 40004 | 激活码已撤销 | 激活码被管理员撤销 |
| 40005 | 设备指纹冲突 | 同一设备已激活 |
| 40006 | 激活频率过高 | 短时间内多次激活 |

## 3. 应用服务实现

### 3.1 ActivationApplicationService

```java
@Service
@Transactional
public class ActivationApplicationService {

    @Autowired
    private ActivationCodeRepository activationCodeRepository;

    @Autowired
    private TerminalInstanceRepository terminalInstanceRepository;

    @Autowired
    private ActivationRecordRepository activationRecordRepository;

    @Autowired
    private TerminalCredentialService terminalCredentialService;

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    @Autowired
    private DomainEventPublisher eventPublisher;

    /**
     * 激活终端
     *
     * 并发控制：
     * - 使用 Redis 分布式锁防止激活码并发超卖
     * - 使用幂等键防止重复激活
     * - 使用频率限制防止暴力激活
     */
    public ActivateTerminalResponse activateTerminal(ActivateTerminalRequest request) {
        // 1. 幂等检查（防止重复激活）
        String idempotentKey = "activation:idempotent:" + request.getDeviceFingerprint();
        if (redisTemplate.hasKey(idempotentKey)) {
            throw new DuplicateActivationException("设备已激活，请勿重复操作");
        }

        // 2. 频率限制（防止暴力激活）
        String rateLimitKey = "activation:ratelimit:" + request.getDeviceFingerprint();
        Long count = redisTemplate.opsForValue().increment(rateLimitKey);
        if (count == 1) {
            redisTemplate.expire(rateLimitKey, 1, TimeUnit.HOURS);
        }
        if (count > 10) {
            throw new RateLimitExceededException("激活频率过高，请稍后再试");
        }

        // 3. 使用分布式锁防止激活码并发消费
        String lockKey = "activation:lock:" + request.getActivationCode();
        RLock lock = redissonClient.getLock(lockKey);

        try {
            // 尝试获取锁，最多等待10秒，锁持有30秒
            if (!lock.tryLock(10, 30, TimeUnit.SECONDS)) {
                throw new ConcurrentActivationException("激活码正在被使用，请稍后再试");
            }

            // 4. 查询激活码
            ActivationCode activationCode = activationCodeRepository
                .findByCode(request.getActivationCode())
                .orElseThrow(() -> new ActivationCodeNotFoundException("激活码不存在"));

            // 5. 校验激活码可用性
            if (!activationCode.isAvailable()) {
                throw new ActivationCodeUnavailableException(
                    "激活码不可用：" + activationCode.getStatus());
            }

            // 6. 检查设备指纹是否已存在
            Optional<TerminalInstance> existingTerminal = terminalInstanceRepository
                .findByDeviceFingerprint(request.getDeviceFingerprint());

            TerminalInstance terminal;
            if (existingTerminal.isPresent()) {
                // 设备已存在，更新激活信息
                terminal = existingTerminal.get();
                if (terminal.getStatus() == TerminalStatus.RETIRED) {
                    throw new TerminalRetiredException("设备已退役，无法重新激活");
                }
            } else {
                // 创建新终端实例
                terminal = TerminalInstance.create(CreateTerminalCommand.builder()
                    .deviceId(request.getDeviceId())
                    .deviceSn(request.getDeviceSn())
                    .deviceFingerprint(request.getDeviceFingerprint())
                    .projectId(activationCode.getProjectId())
                    .tenantId(activationCode.getTenantId())
                    .brandId(activationCode.getBrandId())
                    .storeId(activationCode.getStoreId())
                    .profileId(activationCode.getProfileId())
                    .templateId(activationCode.getTemplateId())
                    .capabilities(request.getCapabilities())
                    .build());

                terminalInstanceRepository.save(terminal);
            }

            // 7. 激活终端
            terminal.activate(activationCode);
            terminalInstanceRepository.save(terminal);

            // 8. 标记激活码已使用
            activationCode.use(terminal.getTerminalId());
            activationCodeRepository.save(activationCode);

            // 9. 记录激活记录
            ActivationRecord record = ActivationRecord.create(
                activationCode.getActivationCodeId(),
                terminal.getTerminalId(),
                request.getDeviceId(),
                request.getDeviceSn(),
                request.getDeviceFingerprint(),
                request.getAppVersion(),
                request.getPlatform()
            );
            activationRecordRepository.save(record);

            // 10. 生成终端凭证
            TerminalCredential credential = terminalCredentialService
                .generateCredential(terminal);

            // 11. 设置幂等键（24小时有效）
            redisTemplate.opsForValue().set(
                idempotentKey,
                terminal.getTerminalId(),
                24,
                TimeUnit.HOURS
            );

            // 12. 发布领域事件
            eventPublisher.publish(terminal.getDomainEvents());
            eventPublisher.publish(activationCode.getDomainEvents());

            // 13. 构建响应
            return ActivateTerminalResponse.builder()
                .terminalId(terminal.getTerminalId())
                .projectId(terminal.getProjectId())
                .tenantId(terminal.getTenantId())
                .brandId(terminal.getBrandId())
                .storeId(terminal.getStoreId())
                .profileId(terminal.getProfileId())
                .templateId(terminal.getTemplateId())
                .tcpAccessToken(credential.getAccessToken())
                .tcpRefreshToken(credential.getRefreshToken())
                .tokenExpiresAt(credential.getExpiresAt())
                .tdpIdentity(buildTdpIdentity(terminal))
                .initialConfig(getInitialConfig(terminal))
                .build();

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new ActivationException("激活被中断");
        } finally {
            // 释放分布式锁
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }

    /**
     * 构建 TDP 身份信息
     */
    private TdpIdentity buildTdpIdentity(TerminalInstance terminal) {
        return TdpIdentity.builder()
            .terminalId(terminal.getTerminalId())
            .orgScopeId("store:" + terminal.getStoreId())
            .profileScopeId("device-model:" + terminal.getProfileId())
            .build();
    }

    /**
     * 获取初始配置
     */
    private InitialConfig getInitialConfig(TerminalInstance terminal) {
        // 从配置服务获取终端的初始配置
        // 这里简化处理，实际应该调用 ConfigApplicationService
        return InitialConfig.builder()
            .configVersion("cfg-default")
            .configData(new HashMap<>())
            .build();
    }
}
```

## 5. 终端换绑流程

### 5.1 换绑场景

换绑是控制面敏感动作，必须严格审计。

典型场景：
- 设备从一个门店调拨到另一个门店
- 同店更换为不同 profile
- 安装模板切换

### 5.2 换绑要求与事务边界

```
【事务边界1：更新终端归属】
1. 更新 TerminalInstance.storeId/profileId/templateId
   - 使用悲观锁防止并发换绑
   - 发布 TerminalReboundEvent 到 outbox 表
   - 提交事务

【事务边界2：撤销旧凭证】
2. 更新 TerminalCredential.status = REVOKED
   - 提交事务

【事务边界3：生成新凭证】
3. 创建新 TerminalCredential
   - 提交事务

【事务边界4：创建配置刷新任务】
4. 创建 TaskRelease (CONFIG_PUBLISH)
   - 创建 TerminalTaskInstance
   - 提交事务

【补偿操作】
- 如果任一步骤失败，执行补偿：
  - 恢复终端归属到旧值
  - 撤销新凭证
  - 恢复旧凭证
  - 取消配置刷新任务
  - 详见：12-分布式事务与一致性设计.md
```

### 5.3 换绑请求


### 5.4 换绑应用服务

```java
@Service
@Transactional
public class TerminalApplicationService {

    @Autowired
    private TerminalInstanceRepository terminalInstanceRepository;

    @Autowired
    private TerminalCredentialService terminalCredentialService;

    @Autowired
    private ConfigApplicationService configApplicationService;

    @Autowired
    private DomainEventPublisher eventPublisher;

    /**
     * 换绑终端
     */
    public RebindTerminalResponse rebindTerminal(RebindTerminalRequest request) {
        // 1. 查询终端
        TerminalInstance terminal = terminalInstanceRepository
            .findById(request.getTerminalId())
            .orElseThrow(() -> new TerminalNotFoundException(request.getTerminalId()));

        // 2. 执行换绑
        terminal.rebind(
            request.getNewStoreId(),
            request.getNewProfileId(),
            request.getNewTemplateId(),
            request.getReason()
        );

        // 3. 保存终端
        terminalInstanceRepository.save(terminal);

        // 4. 失效旧凭证
        terminalCredentialService.revokeCredential(terminal.getTerminalId());

        // 5. 生成新凭证
        TerminalCredential newCredential = terminalCredentialService
            .generateCredential(terminal);

        // 6. 触发配置刷新任务
        configApplicationService.triggerConfigRefresh(terminal.getTerminalId());

        // 7. 发布领域事件
        eventPublisher.publish(terminal.getDomainEvents());

        // 8. 构建响应
        return RebindTerminalResponse.builder()
            .terminalId(terminal.getTerminalId())
            .status(terminal.getStatus())
            .reboundAt(LocalDateTime.now())
            .newAccessToken(newCredential.getAccessToken())
            .newRefreshToken(newCredential.getRefreshToken())
            .build();
    }
}
```

## 6. 终端凭证管理

### 6.1 凭证模型

```java
@Entity
@Table(name = "terminal_credentials")
public class TerminalCredential {

    @Id
    private String credentialId;

    private String terminalId;

    // JWT Token
    private String accessToken;
    private String refreshToken;
    private LocalDateTime expiresAt;

    // 凭证状态
    @Enumerated(EnumType.STRING)
    private CredentialStatus status;  // ACTIVE, REVOKED

    // 版本号（用于 Token 轮换）
    private Integer tokenVersion;

    private LocalDateTime createdAt;
    private LocalDateTime revokedAt;

    // 领域方法

    /**
     * 生成凭证
     */
    public static TerminalCredential generate(TerminalInstance terminal) {
        TerminalCredential credential = new TerminalCredential();
        credential.credentialId = IdGenerator.generateCredentialId();
        credential.terminalId = terminal.getTerminalId();
        credential.tokenVersion = 1;
        credential.status = CredentialStatus.ACTIVE;
        credential.createdAt = LocalDateTime.now();

        // 生成 JWT Token
        JwtTokenPair tokenPair = JwtTokenGenerator.generate(
            terminal.getTerminalId(),
            terminal.getStoreId(),
            terminal.getProfileId(),
            credential.tokenVersion
        );

        credential.accessToken = tokenPair.getAccessToken();
        credential.refreshToken = tokenPair.getRefreshToken();
        credential.expiresAt = tokenPair.getExpiresAt();

        return credential;
    }

    /**
     * 撤销凭证
     */
    public void revoke() {
        this.status = CredentialStatus.REVOKED;
        this.revokedAt = LocalDateTime.now();
    }

    /**
     * 刷新凭证
     */
    public void refresh(TerminalInstance terminal) {
        // 增加版本号
        this.tokenVersion++;

        // 生成新 Token
        JwtTokenPair tokenPair = JwtTokenGenerator.generate(
            terminal.getTerminalId(),
            terminal.getStoreId(),
            terminal.getProfileId(),
            this.tokenVersion
        );

        this.accessToken = tokenPair.getAccessToken();
        this.refreshToken = tokenPair.getRefreshToken();
        this.expiresAt = tokenPair.getExpiresAt();
    }
}

/**
 * 凭证状态
 */
public enum CredentialStatus {
    ACTIVE,   // 活跃
    REVOKED   // 已撤销
}
```

## 7. 安全与风控

### 7.1 安全要求

1. **激活码安全**
   - 激活码必须有有效期
   - 激活码必须支持撤销
   - 激活码使用次数限制
   - 激活码加密存储

2. **设备指纹校验**
   - 设备指纹变更要审计
   - 同一设备短时间多次激活应限流
   - 设备指纹冲突要告警

3. **Token 安全**
   - Token 有效期 7 天
   - Token 支持刷新
   - Token 轮换要可追踪
   - 退役终端的 Token 必须立即失效

4. **换绑安全**
   - 异常换绑需要二次确认
   - 换绑操作必须审计
   - 换绑后旧 Token 立即失效

### 7.2 风控策略

```java
@Service
public class ActivationRiskControlService {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    /**
     * 检查激活频率
     */
    public void checkActivationRate(String deviceFingerprint) {
        String key = "activation:ratelimit:" + deviceFingerprint;
        Long count = redisTemplate.opsForValue().increment(key);

        if (count == 1) {
            redisTemplate.expire(key, 1, TimeUnit.HOURS);
        }

        if (count > 10) {
            throw new RateLimitExceededException("激活频率过高");
        }
    }

    /**
     * 检查设备指纹冲突
     */
    public void checkDeviceFingerprintConflict(String deviceFingerprint) {
        String key = "activation:fingerprint:" + deviceFingerprint;
        if (redisTemplate.hasKey(key)) {
            throw new DeviceFingerprintConflictException("设备指纹冲突");
        }
    }

    /**
     * 记录激活成功
     */
    public void recordActivationSuccess(String deviceFingerprint, String terminalId) {
        String key = "activation:fingerprint:" + deviceFingerprint;
        redisTemplate.opsForValue().set(key, terminalId, 24, TimeUnit.HOURS);
    }
}
```

## 8. 本文结论

激活与绑定流程设计展示了完整的终端接入实现：

- **激活流程**：激活码校验、设备指纹校验、终端创建、凭证生成
- **绑定流程**：终端与门店/模板的关系建立
- **换绑流程**：终端调拨、模板切换、凭证更新
- **凭证管理**：JWT Token 生成、刷新、撤销
- **安全风控**：频率限制、设备指纹校验、审计日志
