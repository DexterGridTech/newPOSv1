### 2.3 扩容策略

#### 垂直扩容

**触发条件：**
- CPU 使用率持续 > 70%
- 内存使用率持续 > 80%
- 磁盘 IOPS 接近上限

**扩容方案：**
- 应用服务器：4C8G → 8C16G → 16C32G
- PostgreSQL：增加 CPU/内存/磁盘
- Redis：增加内存

#### 水平扩容

**触发条件：**
- 单节点 QPS 接近上限
- 连接数接近上限
- 垂直扩容无法满足需求

**扩容方案：**
- L1：增加节点数（2 → 3 → 5）
- L2：每个商场增加节点数（1 → 2 → 3）
- PostgreSQL：主从 → 分片
- Redis：单机 → Cluster

## 3. 部署流程

### 3.1 L1 中心节点部署

#### 环境准备

```bash
# 1. 安装 Java 17
sudo apt install openjdk-17-jdk

# 2. 安装 PostgreSQL 14
sudo apt install postgresql-14

# 3. 安装 Redis 7
sudo apt install redis-server

# 4. 创建数据库
sudo -u postgres psql
CREATE DATABASE tdp_production;
CREATE USER tdp_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE tdp_production TO tdp_user;
```

#### 初始化数据库

```bash
# 执行 DDL 脚本
psql -U tdp_user -d tdp_production -f schema.sql

# 初始化 Topic Registry
psql -U tdp_user -d tdp_production -f init_topics.sql
```

#### 部署应用

```bash
# 1. 上传应用包
scp tdp-service-1.0.0.jar user@server:/opt/tdp/

# 2. 配置文件
cat > /opt/tdp/application.yml <<EOF
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/tdp_production
    username: tdp_user
    password: your_password
  redis:
    host: localhost
    port: 6379

tdp:
  node:
    id: l1-central-01
    type: L1
    region: beijing
  federation:
    enabled: true
EOF

# 3. 启动服务
java -jar /opt/tdp/tdp-service-1.0.0.jar \
  --spring.config.location=/opt/tdp/application.yml \
  --server.port=8080
```

#### 配置 Systemd

```bash
cat > /etc/systemd/system/tdp-l1.service <<EOF
[Unit]
Description=TDP L1 Service
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=tdp
WorkingDirectory=/opt/tdp
ExecStart=/usr/bin/java -jar /opt/tdp/tdp-service-1.0.0.jar
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

systemctl enable tdp-l1
systemctl start tdp-l1
```

## 8. 故障处理

### 8.1 L1 节点故障

**症状：**
- L1 节点无响应
- 业务写入失败
- L2 节点报告上游断连

**处理步骤：**
1. 检查 L1 节点状态
2. 如果节点宕机，启动备用节点
3. 如果数据库故障，切换到从库
4. 恢复服务后，验证数据一致性

---

### 8.2 L2 节点故障

**症状：**
- L2 节点无响应
- 终端连接失败
- 终端自动切换到其他 L2 或 L1

**处理步骤：**
1. 检查 L2 节点状态
2. 如果节点宕机，重启节点
3. 节点恢复后，自动从 L1 同步数据
4. 验证终端连接恢复

---

### 8.3 数据库故障

**症状：**
- 数据库连接失败
- 写入操作失败
- 查询超时

**处理步骤：**
1. 检查数据库状态
2. 如果主库故障，切换到从库
3. 如果磁盘满，清理过期数据
4. 如果连接数耗尽，重启应用或增加连接池

---

### 8.4 Redis 故障

**症状：**
- Redis 连接失败
- 缓存命中率下降
- Session 丢失

**处理步骤：**
1. 检查 Redis 状态
2. 如果 Redis 宕机，重启 Redis
3. 如果内存不足，清理过期键或增加内存
4. 如果数据丢失，从 PostgreSQL 重建缓存

```java
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
            .antMatchers("/v1/projections/**").authenticated()
            .antMatchers("/actuator/health").permitAll()
            .and()
            .oauth2ResourceServer()
            .jwt();
    }
}
```
