## 1. 核心数据模型

### 1.1 Projection Item（投影项）

**概念：** 某个 Scope 下某个 Topic/Item 的当前有效值。

**属性：**
```
ProjectionItem {
  id: Long                    # 主键
  tenantId: String            # 租户 ID
  topic: String               # Topic 名称
  itemKey: String             # Item 键
  scopeType: String           # Scope 类型
  scopeId: String             # Scope ID
  payload: JSONB              # 载荷数据
  schemaVersion: String       # Schema 版本
  revision: Long              # 全局版本号
  occurredAt: Timestamp       # 业务发生时间
  updatedAt: Timestamp        # 更新时间
  expiresAt: Timestamp?       # 过期时间（可选）
  tombstone: Boolean          # 是否已删除
  sourceSystem: String        # 来源系统
  idempotencyKey: String      # 幂等键
}
```

**唯一约束：**
```
UNIQUE(tenantId, topic, itemKey, scopeType, scopeId)
```

**说明：**
- 同一 `(tenantId, topic, itemKey, scopeType, scopeId)` 只保留最新值
- `tombstone=true` 表示已删除，但记录仍保留（用于推送 delete 操作）
- `expiresAt` 到期后，后台任务自动清理

---

### 1.2 Projection Change Log（投影变更日志）

**概念：** 记录所有投影变更的时序日志，用于增量同步。

**属性：**
```
ProjectionChangeLog {
  id: Long                    # 主键
  revision: Long              # 全局版本号（唯一）
  tenantId: String            # 租户 ID
  topic: String               # Topic 名称
  itemKey: String             # Item 键
  scopeType: String           # Scope 类型
  scopeId: String             # Scope ID
  operation: String           # 操作类型（upsert / delete）
  payload: JSONB?             # 载荷数据（delete 时为 null）
  occurredAt: Timestamp       # 业务发生时间
  publishedAt: Timestamp      # 发布时间
  sourceSystem: String        # 来源系统
  idempotencyKey: String      # 幂等键
}
```

**唯一约束：**
```
UNIQUE(revision)
```

**索引：**
```
INDEX idx_revision_range (revision)
INDEX idx_tenant_topic (tenantId, topic, revision)
INDEX idx_published_at (publishedAt)
```

**保留策略：**
- 默认保留 7 天
- 7 天后归档到冷存储或删除
- 归档前确保所有 L2 节点已同步

---

### 1.3 Terminal Session（终端会话）

**概念：** 记录终端当前连接会话信息。

**属性：**
```
TerminalSession {
  sessionId: String           # 会话 ID（主键）
  terminalId: String          # 终端 ID
  tenantId: String            # 租户 ID
  orgScopeId: String          # 组织作用域 ID
  profileScopeId: String      # 终端画像 ID
  nodeId: String              # 当前连接的节点 ID
  region: String              # 区域
  connectedAt: Timestamp      # 连接时间
  lastHeartbeatAt: Timestamp  # 最后心跳时间
  lastAppliedRevision: Long   # 最后应用的 Revision
  subscribedTopics: JSONB     # 订阅的 Topic 列表
  appVersion: String          # 终端应用版本
  capabilities: JSONB         # 终端能力列表
}
```

**索引：**
```
INDEX idx_terminal (terminalId)
INDEX idx_node (nodeId)
INDEX idx_heartbeat (lastHeartbeatAt)
INDEX idx_org_scope (tenantId, orgScopeId)
```

**存储：**
- 主存储：Redis（快速查询）
- 备份存储：PostgreSQL（持久化 + 审计）

**清理策略：**
- 心跳超时 5 分钟后标记为离线
- 离线 24 小时后从 Redis 删除（PostgreSQL 保留）

### 1.4 Topic Registry（Topic 注册表）

**概念：** 记录所有注册的 Topic 元数据。

**属性：**
```
TopicRegistry {
  topic: String               # Topic 名称（主键）
  topicType: String           # Topic 类型
  ownerDomain: String         # 归属业务域
  schemaVersion: String       # Schema 版本
  payloadSchema: JSONB        # JSON Schema 定义
  allowedScopeTypes: String[] # 允许的 Scope 类型
  maxPayloadSize: Int         # 最大载荷大小（字节）
  retentionPolicy: String     # 保留策略
  ttlSeconds: Int?            # TTL 秒数（可选）
  createdAt: Timestamp        # 创建时间
  updatedAt: Timestamp        # 更新时间
}
```

**topicType 枚举：**
- `retained_state`：保留态数据
- `durable_event`：耐久事件
- `ephemeral_cmd`：点对点命令

**retentionPolicy 枚举：**
- `permanent`：永久保留
- `ttl`：TTL 过期后删除
- `tombstone`：标记删除后保留一段时间

### 1.5 Federation Subscription（联邦订阅）

**概念：** 记录 L2 节点订阅的 Scope 集合。

**属性：**
```
FederationSubscription {

  subscribedScopes: JSONB     # 订阅的 Scope 列表
  lastCursor: Long            # 最后同步的 Revision
  createdAt: Timestamp        # 创建时间
  updatedAt: Timestamp        # 更新时间
}
```

**索引：**
```
INDEX idx_l2_node (l2NodeId)
```

## 3. Revision 生成机制

### 3.1 全局单调递增

**实现方式：** PostgreSQL Sequence

```sql
CREATE SEQUENCE projection_revision_seq
    START WITH 1
    INCREMENT BY 1
    NO CYCLE;
```

**获取 Revision：**
```sql
SELECT nextval('projection_revision_seq');
```

**说明：**
- L1 中心节点独占 Sequence，确保全局唯一
- L2 边缘节点不生成 Revision，只消费 L1 分配的 Revision

---

### 3.2 Revision 与 Cursor

**Revision：** 全局单调递增的版本号，由 L1 分配。

**Cursor：** 终端或 L2 已消费到的 Revision 位置。

**关系：**
```
Cursor ≤ HighWatermark

HighWatermark = MAX(revision) FROM projection_change_log
```

**增量同步：**
```sql
SELECT * FROM projection_change_log
WHERE revision > :cursor
ORDER BY revision ASC
LIMIT 100;
```

## 4. 缓存策略

### 4.1 Redis 缓存结构

#### Projection Store 缓存

```
# Hash 结构：按 Scope + Topic 缓存
tdp:projection:{tenantId}:{scopeType}:{scopeId}:{topic} → Hash {
  itemKey1: payload1,
  itemKey2: payload2,
  ...
}

# 示例
tdp:projection:t-001:store:s-001:order.pending → {
  "order:20240405001": "{\"orderId\":\"order_20240405001\",\"status\":\"PAID\"}",
  "order:20240405002": "{\"orderId\":\"order_20240405002\",\"status\":\"ACCEPTED\"}"
}

# TTL：1 小时（热数据）
EXPIRE tdp:projection:t-001:store:s-001:order.pending 3600
```

#### HighWatermark 缓存

```
# String 结构
tdp:highWatermark → 10042

# 更新策略：每次写入后更新
SET tdp:highWatermark 10043
```

#### Topic Registry 缓存

```
# Hash 结构
tdp:topic:{topic} → {
  topicType: "retained_state",
  ownerDomain: "trading-service",
  allowedScopeTypes: ["store"],
  maxPayloadSize: 65536
}

# TTL：永久（Topic 元数据变更极少）
```

---

### 4.2 缓存更新策略

**写入流程：**
1. 写入 PostgreSQL `projection_items` 表
2. 追加 PostgreSQL `projection_change_log` 表
3. 更新 Redis `tdp:projection:*` 缓存
4. 更新 Redis `tdp:highWatermark`
5. 推送到在线终端

**缓存失效：**
- Projection 缓存：1 小时 TTL
- Session 缓存：5 分钟 TTL（心跳刷新）
- Topic Registry 缓存：永久

**缓存穿透保护：**
- 使用 Bloom Filter 判断 Projection 是否存在
- 不存在的 Projection 缓存空值（TTL 1 分钟）

## 5. 数据清理策略

### 5.1 过期 Projection 清理

**定时任务：** 每小时执行一次

```sql
-- 删除已过期的 Projection
DELETE FROM projection_items
WHERE expires_at IS NOT NULL
  AND expires_at < NOW();
```

**说明：**
- `expiresAt` 到期后自动删除
- 适用于 `order.urge`、`workunit.recalled` 等短生命周期数据

---

### 5.2 Tombstone Projection 清理

**定时任务：** 每天执行一次

```sql
-- 删除 tombstone 超过 24 小时的 Projection
DELETE FROM projection_items
WHERE tombstone = TRUE
  AND updated_at < NOW() - INTERVAL '24 hours';
```

**说明：**
- Tombstone 保留 24 小时，确保离线终端重连后能收到 delete 操作
- 24 小时后删除，释放存储空间

---

### 5.3 Change Log 归档

**定时任务：** 每天执行一次

```sql
-- 归档 7 天前的 Change Log
INSERT INTO projection_change_log_archive
SELECT * FROM projection_change_log
WHERE published_at < NOW() - INTERVAL '7 days';

DELETE FROM projection_change_log
WHERE published_at < NOW() - INTERVAL '7 days';
```

**归档存储：**
- 选项 1：PostgreSQL 归档表（冷存储）
- 选项 2：对象存储（S3 / OSS）
- 选项 3：ClickHouse（时序数据库）

---

### 5.4 离线 Session 清理

**定时任务：** 每小时执行一次

```sql
-- 删除离线超过 24 小时的 Session
DELETE FROM terminal_sessions
```

## 6. 数据一致性保证

### 6.1 写入事务

**单条写入：**
```java
@Transactional
public void writeProjection(ProjectionWriteRequest request) {
    // 1. 分配 Revision
    Long revision = revisionSequence.nextval();

    // 2. 写入 projection_items（UPSERT）
    projectionItemRepository.upsert(
        request.getTenantId(),
        request.getTopic(),
        request.getItemKey(),
        request.getScopeType(),
        request.getScopeId(),
        request.getPayload(),
        revision,
        request.getOccurredAt(),
        request.getIdempotencyKey()
    );

    // 3. 追加 projection_change_log
    changeLogRepository.insert(
        revision,
        request.getTenantId(),
        request.getTopic(),
        request.getItemKey(),
        request.getScopeType(),
        request.getScopeId(),
        "upsert",
        request.getPayload(),
        request.getOccurredAt(),
        request.getIdempotencyKey()
    );

    // 4. 更新 Redis 缓存（事务外）
    redisTemplate.opsForHash().put(
        buildCacheKey(request),
        request.getItemKey(),
        request.getPayload()
    );

    // 5. 更新 HighWatermark
    redisTemplate.opsForValue().set("tdp:highWatermark", revision);

    // 6. 推送到在线终端（异步）
    distributionService.pushToTerminals(request, revision);
}
```

**批量写入：**
```java
@Transactional
public void batchWriteProjections(List<ProjectionWriteRequest> requests) {
    // 1. 批量分配 Revision
    Long startRevision = revisionSequence.nextval();

    // 2. 批量写入 projection_items
    projectionItemRepository.batchUpsert(requests, startRevision);

    // 3. 批量追加 projection_change_log
    changeLogRepository.batchInsert(requests, startRevision);

    // 4. 批量更新 Redis 缓存
    // 5. 批量推送到在线终端
}
```

### 6.2 幂等保证

**幂等键约束：**
```sql
CONSTRAINT uk_idempotency UNIQUE(tenant_id, topic, item_key, scope_type, scope_id, idempotency_key)
```

**重复写入处理：**
```java
try {
    writeProjection(request);
} catch (DuplicateKeyException e) {
    // 检查是否是相同 Payload
    ProjectionItem existing = projectionItemRepository.findByIdempotencyKey(
        request.getTenantId(),
        request.getTopic(),
        request.getItemKey(),
        request.getScopeType(),
        request.getScopeId(),
        request.getIdempotencyKey()
    );

    if (existing.getPayload().equals(request.getPayload())) {
        // 相同 Payload，返回成功（幂等）
        return WriteResult.success(existing.getRevision());
    } else {
        // 不同 Payload，返回冲突错误
        throw new IdempotencyConflictException();
    }
}
```

---

### 6.3 Redis 与 PostgreSQL 一致性

**策略：** PostgreSQL 为权威，Redis 为缓存。

**不一致场景：**
1. Redis 写入失败 → 终端下次同步时从 PostgreSQL 获取
2. Redis 过期 → 终端下次同步时从 PostgreSQL 获取
3. Redis 数据错误 → 定期对账任务修正

**对账任务：**
```java
@Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨 2 点
public void reconcileRedisWithPostgres() {
    // 1. 从 PostgreSQL 加载所有活跃 Projection
    List<ProjectionItem> items = projectionItemRepository.findAllActive();

    // 2. 重建 Redis 缓存
    for (ProjectionItem item : items) {
        redisTemplate.opsForHash().put(
            buildCacheKey(item),
            item.getItemKey(),
            item.getPayload()
        );
    }
}
```

### 7.3 存储优化建议

**PostgreSQL：**
- 使用 JSONB 存储 Payload（支持索引和查询）
- 按 `tenant_id` HASH 分区（水平扩展）
- 按 `published_at` 时间分区 Change Log（便于归档）
- 定期 VACUUM 和 ANALYZE

**Redis：**
- 使用 Redis Cluster（水平扩展）
- 合理设置 TTL（避免内存溢出）
- 使用 Redis Persistence（RDB + AOF）
