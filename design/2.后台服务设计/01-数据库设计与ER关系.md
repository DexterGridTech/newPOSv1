# 01-数据库设计与ER关系

## 1. 文档目的

本文档提供餐饮中台所有数据库表的全景视图、ER关系图、完整CREATE TABLE语句、索引设计、分库分表策略和数据归档策略，帮助理解整体数据架构和表之间的关联关系。

---

## 2. 设计原则

### 2.1 微服务数据隔离

**每个服务独立数据库**：严格遵循微服务数据隔离原则，避免跨服务直接访问数据库。

### 2.2 命名规范

- 表名：小写下划线命名（snake_case），复数形式
- 字段名：小写下划线命名（snake_case）
- 主键：`{entity}_id`，VARCHAR(64)
- 外键字段：`{referenced_entity}_id`
- 时间字段：TIMESTAMP类型，`created_at`、`updated_at`
- 状态字段：VARCHAR(32)，使用枚举字符串
- 金额字段：BIGINT，单位为分
- JSON字段：JSONB 类型（PostgreSQL）
- 索引前缀：`idx_`
- 唯一约束：`UNIQUE` 关键字

## 7. 所有数据库表定义（CREATE TABLE）

### 7.1 交易域数据库 (transaction_db)

#### sale_orders 表（订单主表）

```sql
CREATE TABLE sale_orders (
    -- 主键与标识
    order_id VARCHAR(64) PRIMARY KEY,
    order_no VARCHAR(32) UNIQUE NOT NULL,

    -- 租户与门店
    store_id VARCHAR(64) NOT NULL,
    tenant_id VARCHAR(64) NOT NULL,
    brand_id VARCHAR(64),
    entity_id VARCHAR(64),

    -- 三维度统一模型
    channel_type VARCHAR(32) NOT NULL,  -- MINI_PROGRAM, H5, POS_DIRECT, KIOSK, MEITUAN, ELEME, DOUYIN
    order_scene VARCHAR(32) NOT NULL,   -- DINE_IN, TAKEAWAY_SELF, DELIVERY_ONESELF, DELIVERY_PLATFORM
    fulfillment_type VARCHAR(32) NOT NULL, -- TABLE_SERVICE, SELF_PICKUP, THIRD_PARTY_DELIVERY, PLATFORM_DELIVERY, STORE_DELIVERY

    -- 外部订单号
    external_order_no VARCHAR(64),

    -- 顾客信息
    customer_id VARCHAR(64),
    customer_name VARCHAR(128),
    customer_phone VARCHAR(32),
    customer_phone_suffix VARCHAR(8),

    -- 金额信息（单位：分）
    total_amount BIGINT NOT NULL,
    discount_amount BIGINT DEFAULT 0,
    payable_amount BIGINT NOT NULL,
    paid_amount BIGINT DEFAULT 0,
    refund_amount BIGINT DEFAULT 0,

    -- 服务上下文（JSON，存储场景特定信息）
    service_context JSONB,
    -- 字段名以 camelCase 序列化，结构见 02-交易域服务设计.md ServiceContext：
    -- 堂食：{"tableCode": "A05", "dinerCount": 4}
    -- 外卖：{"deliveryAddress": {"province":..., "city":..., "street":..., "detail":..., "lat":..., "lng":...}, "deliveryTime": "ISO8601"}
    -- 自取：{"pickupCode": "018", "expectedPickupTime": "ISO8601"}

    -- 状态与时间
    status VARCHAR(32) NOT NULL,
    -- PENDING_PAYMENT, PAID, ACCEPTED, IN_PRODUCTION, PRODUCTION_DONE,
    -- HANDOFF_READY, FULFILLED, COMPLETED, CANCELLED, REJECTED

    -- 三维独立状态投影（与 status 正交，支持按支付/退款/履约状态独立查询）
    payment_status VARCHAR(32) NOT NULL DEFAULT 'UNPAID',
    -- UNPAID, PAID, PARTIAL_REFUNDING, PARTIALLY_REFUNDED, FULL_REFUNDING, FULLY_REFUNDED, REFUND_FAILED
    refund_status VARCHAR(32) NOT NULL DEFAULT 'NONE',
    -- NONE, PARTIAL_REFUNDING, PARTIALLY_REFUNDED, FULL_REFUNDING, FULLY_REFUNDED, REFUND_FAILED
    fulfillment_status_projection VARCHAR(32),
    -- 履约状态快照（从履约域同步，仅用于查询投影，真相源在 fulfillment_orders 表）
    -- PLAN_PENDING, PRODUCTION_PLANNED, IN_PRODUCTION, COMPLETED, DELIVERING, FULFILLED, CANCELLED

    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    paid_at TIMESTAMP,
    accepted_at TIMESTAMP,
    completed_at TIMESTAMP,
    cancelled_at TIMESTAMP,

    -- 版本号（乐观锁）
    version INT NOT NULL DEFAULT 0,

    -- 索引
    INDEX idx_store_created (store_id, created_at),
    INDEX idx_customer_created (customer_id, created_at),
    INDEX idx_status_created (status, created_at),
    INDEX idx_payment_status (payment_status),
    INDEX idx_refund_status (refund_status),
    INDEX idx_external_order (external_order_no),
    INDEX idx_channel_type (channel_type, created_at)
) PARTITION BY HASH (store_id);
```

## 8. 索引设计策略

### 8.1 高频查询索引

**订单查询**：
- `sale_orders(store_id, created_at)` -- 门店维度订单列表
- `sale_orders(customer_id, created_at)` -- 顾客维度订单历史
- `sale_orders(status, created_at)` -- 状态维度订单查询
- `sale_orders(external_order_no)` -- 外部订单号查询

**商品查询**：
- `products(store_id, status)` -- 门店商品列表
- `products(category_id)` -- 品类下商品列表

**履约查询**：
- `fulfillment_orders(order_id)` -- 按订单查履约单
- `fulfillment_orders(store_id, status)` -- 门店履约状态
- `work_units(workstation_id, status)` -- 工作站任务列表

**结算查询**：
- `settlement_facts(order_id)` -- 按订单查结算
- `settlement_facts(entity_id, order_completed_at)` -- 主体维度结算查询

### 8.2 唯一约束索引

- `sale_orders(order_no)` UNIQUE -- 订单号唯一
- `products(store_id, product_code)` UNIQUE -- 门店内商品编码唯一
- `tables(store_id, table_no)` UNIQUE -- 门店内桌号唯一
- `channel_shop_bindings(store_id, channel_type)` UNIQUE -- 门店渠道绑定唯一

- 活跃商品数据
- 活跃门店数据

**冷数据**（归档存储）：
- 3 个月以前的订单数据
- 已删除的商品数据
- 已关闭的门店数据

### 12.2 应用层读写分离

```java
@Configuration
public class DataSourceConfig {

    @Bean
    @ConfigurationProperties("spring.datasource.master")
    public DataSource masterDataSource() {
        return DataSourceBuilder.create().build();
    }

    @Bean
    @ConfigurationProperties("spring.datasource.slave")
    public DataSource slaveDataSource() {
        return DataSourceBuilder.create().build();
    }

    @Bean
    public DataSource routingDataSource() {
        Map<Object, Object> targetDataSources = new HashMap<>();
        targetDataSources.put("master", masterDataSource());
        targetDataSources.put("slave", slaveDataSource());

        DynamicRoutingDataSource routingDataSource = new DynamicRoutingDataSource();
        routingDataSource.setTargetDataSources(targetDataSources);
        routingDataSource.setDefaultTargetDataSource(masterDataSource());

        return routingDataSource;
    }
}

@Service
public class OrderService {

    // 写操作：使用主库
    @Transactional
    @DataSource("master")
    public void createOrder(Order order) {
        orderRepository.save(order);
    }

    // 读操作：使用从库
    @Transactional(readOnly = true)
    @DataSource("slave")
    public Order getOrder(String orderId) {
        return orderRepository.findById(orderId);
    }
}
```
