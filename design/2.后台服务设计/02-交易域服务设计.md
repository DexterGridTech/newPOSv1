# 02-交易域服务设计

## 1. 服务概述

交易域服务（transaction-service）是餐饮中台的核心服务，负责订单管理、支付管理、购物车管理等核心交易功能。

**核心职责**：
- 订单创建、查询、取消
- 支付处理、退款处理
- 购物车管理
- 订单状态机管理
- 订单事件发布
- 桌台会话管理

**核心聚合根**：
- SaleOrder（销售订单）
- Payment（支付）
- ShoppingCart（购物车）
- TableSession（桌台会话）

**服务端口**：8082
**数据库**：transaction_db

```java
    @Enumerated(EnumType.STRING)

    private String externalOrderNo;

    // 顾客信息
    private String customerId;
    private String customerName;
    private String customerPhone;
    private String customerPhoneSuffix;

    // 金额信息（单位：分）
    private Long totalAmount;
    private Long discountAmount;
    private Long payableAmount;
    private Long paidAmount;
    private Long refundAmount;

    // 服务上下文
    @Type(JsonBinaryType.class)
    @Column(columnDefinition = "jsonb")
    private ServiceContext serviceContext;

    // 订单明细
    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL)
    private List<OrderItem> items;

    // 状态与时间
    @Enumerated(EnumType.STRING)
    private OrderStatus status;  // 订单业务主状态

    // 多状态线（P1-2）：独立于主状态，各自有独立生命周期
    @Enumerated(EnumType.STRING)
    private PaymentStatus paymentStatus;         // 支付状态：PENDING/SUCCESS/FAILED/REFUNDED

    @Enumerated(EnumType.STRING)
    private PaymentRefundStatus refundStatus;    // 退款状态：NONE/PARTIAL_REFUNDING/PARTIALLY_REFUNDED/FULL_REFUNDING/FULLY_REFUNDED/REFUND_FAILED

    @Enumerated(EnumType.STRING)
    private FulfillmentStatusProjection fulfillmentStatusProjection; // 履约状态投影（只读，由履约域事件驱动更新）

    private LocalDateTime createdAt;
    private LocalDateTime paidAt;
    private LocalDateTime acceptedAt;
    private LocalDateTime completedAt;
    private LocalDateTime cancelledAt;

    // 版本号（乐观锁）
    @Version
    private Integer version;

    // 领域方法

    /**
     * 创建订单
     */
    public static SaleOrder create(CreateOrderCommand command) {
        SaleOrder order = new SaleOrder();
        order.orderId = IdGenerator.generateOrderId();
        order.orderNo = OrderNoGenerator.generate();
        order.storeId = command.getStoreId();
        order.tenantId = command.getTenantId();
        order.channelType = command.getChannelType();
        order.orderScene = command.getOrderScene();
        order.fulfillmentType = command.getFulfillmentType();
        order.customerId = command.getCustomerId();
        order.totalAmount = command.getTotalAmount();
        order.discountAmount = command.getDiscountAmount();
        order.payableAmount = command.getPayableAmount();
        order.serviceContext = command.getServiceContext();
        order.status = OrderStatus.PENDING_PAYMENT;
        order.createdAt = LocalDateTime.now();

        // 创建订单明细
        order.items = command.getItems().stream()
            .map(item -> OrderItem.create(order, item))
            .collect(Collectors.toList());

        // 发布领域事件
        order.addDomainEvent(new OrderCreatedEvent(order));

        return order;
    }

    /**
     * 支付成功
     */
    public void markAsPaid(Payment payment) {
        if (this.status != OrderStatus.PENDING_PAYMENT) {
            throw new IllegalStateException("订单状态不正确，无法支付");
        }

        this.status = OrderStatus.PAID;
        this.paidAmount = payment.getAmount();
        this.paidAt = LocalDateTime.now();

        // 发布领域事件
        this.addDomainEvent(new OrderPaidEvent(this));
    }

    /**
     * 接单
     */
    public void accept() {
        if (this.status != OrderStatus.PAID) {
            throw new IllegalStateException("订单状态不正确，无法接单");
        }

        this.status = OrderStatus.ACCEPTED;
        this.acceptedAt = LocalDateTime.now();

        // 发布领域事件
        this.addDomainEvent(new OrderAcceptedEvent(this));
    }

    /**
     * 完成订单
     */
    public void complete() {
        if (this.status != OrderStatus.FULFILLED) {
            throw new IllegalStateException("订单状态不正确，无法完成");
        }

        this.status = OrderStatus.COMPLETED;
        this.completedAt = LocalDateTime.now();

        // 发布领域事件
        this.addDomainEvent(new OrderCompletedEvent(this));
    }

    /**
     * 取消订单
     */
    public void cancel(String cancelReason) {
        if (!canCancel()) {
            throw new IllegalStateException("订单状态不允许取消");
        }

        this.status = OrderStatus.CANCELLED;
        this.cancelledAt = LocalDateTime.now();

        // 发布领域事件
        this.addDomainEvent(new OrderCancelledEvent(this, cancelReason));
    }

    /**
     * 是否可以取消
     */
    private boolean canCancel() {
        return this.status == OrderStatus.PENDING_PAYMENT
            || this.status == OrderStatus.PAID
            || this.status == OrderStatus.ACCEPTED;
    }
}
```

```java
        this.status = PaymentStatus.SUCCESS;
        this.transactionId = transactionId;
        this.paymentDetails = details;

        // 发布领域事件
        this.addDomainEvent(new PaymentConfirmedEvent(this));
    }

    /**
     * 退款
     */
    public void refund() {
        if (this.status != PaymentStatus.SUCCESS) {
            throw new IllegalStateException("支付状态不正确，无法退款");
        }

        this.status = PaymentStatus.REFUNDED;
        this.refundedAt = LocalDateTime.now();

        // 发布领域事件
        this.addDomainEvent(new PaymentRefundedEvent(this));
    }
}
```

```java
    private String tableId;

    private LocalDateTime closedAt;

    @Version
    private Integer version;

    public void changePartySize(int newSize) {
        if (this.sessionStatus != SessionStatus.OPEN) {
            throw new IllegalStateException("只有OPEN状态可以变更人数");
        }
        this.partySize = newSize;
    }

    public void hangBill() {
        if (this.sessionStatus != SessionStatus.OPEN) {
            throw new IllegalStateException("只有OPEN状态可以挂单");
        }
        this.sessionStatus = SessionStatus.HANG_BILL;
    }

    public void close() {
        this.sessionStatus = SessionStatus.CLOSED;
        this.closedAt = LocalDateTime.now();
    }
}
```

## 4. 应用服务层

### 4.1 OrderApplicationService

```java
@Service
@Transactional
public class OrderApplicationService {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private ProductServiceClient productServiceClient;

    @Autowired
    private DomainEventPublisher eventPublisher;

    /**
     * 创建订单
     */
    public CreateOrderResponse createOrder(CreateOrderRequest request) {
        // 1. 验证请求
        validateCreateOrderRequest(request);

        // 2. 查询商品信息（获取商品快照）
        List<ProductSnapshot> productSnapshots = productServiceClient
            .getProductSnapshots(request.getProductIds());

        // 3. 计算订单金额
        OrderAmountCalculator calculator = new OrderAmountCalculator();
        OrderAmount amount = calculator.calculate(request, productSnapshots);

        // 4. 创建订单命令
        CreateOrderCommand command = CreateOrderCommand.builder()
            .storeId(request.getStoreId())
            .tenantId(request.getTenantId())
            .channelType(request.getChannelType())
            .orderScene(request.getOrderScene())
            .fulfillmentType(request.getFulfillmentType())
            .customerId(request.getCustomerId())
            .totalAmount(amount.getTotalAmount())
            .discountAmount(amount.getDiscountAmount())
            .payableAmount(amount.getPayableAmount())
            .serviceContext(request.getServiceContext())
            .items(buildOrderItems(request, productSnapshots))
            .build();

        // 5. 创建订单聚合
        SaleOrder order = SaleOrder.create(command);

        // 6. 保存订单
        orderRepository.save(order);

        // 7. 发布领域事件
        eventPublisher.publish(order.getDomainEvents());

        // 8. 返回响应
        return CreateOrderResponse.builder()
            .orderId(order.getOrderId())
            .orderNo(order.getOrderNo())
            .payableAmount(order.getPayableAmount())
            .build();
    }

    /**
     * 支付订单
     */
    public PayOrderResponse payOrder(PayOrderRequest request) {
        // 1. 查询订单
        SaleOrder order = orderRepository.findById(request.getOrderId())
            .orElseThrow(() -> new OrderNotFoundException(request.getOrderId()));

        // 2. 创建支付记录
        Payment payment = Payment.create(CreatePaymentCommand.builder()
            .orderId(order.getOrderId())
            .storeId(order.getStoreId())
            .paymentMethod(request.getPaymentMethod())
            .amount(order.getPayableAmount())
            .build());

        // 3. 调用支付网关
        PaymentGatewayResponse gatewayResponse = paymentGateway.pay(payment);

        // 4. 返回支付参数
        return PayOrderResponse.builder()
            .paymentId(payment.getPaymentId())
            .paymentParams(gatewayResponse.getPaymentParams())
            .build();
    }

    /**
     * 支付回调
     */
    public void handlePaymentCallback(PaymentCallbackRequest request) {
        // 1. 验证签名
        if (!paymentGateway.verifySignature(request)) {
            throw new InvalidSignatureException();
        }

        // 2. 查询支付记录
        Payment payment = paymentRepository.findByTransactionId(request.getTransactionId())
            .orElseThrow(() -> new PaymentNotFoundException(request.getTransactionId()));

        // 3. 更新支付状态
        payment.markAsSuccess(request.getTransactionId(), request.getPaymentDetails());
        paymentRepository.save(payment);

        // 4. 查询订单
        SaleOrder order = orderRepository.findById(payment.getOrderId())
            .orElseThrow(() -> new OrderNotFoundException(payment.getOrderId()));

        // 5. 更新订单状态
        order.markAsPaid(payment);
        orderRepository.save(order);

        // 6. 发布领域事件
        eventPublisher.publish(payment.getDomainEvents());
        eventPublisher.publish(order.getDomainEvents());
    }

    /**
     * 取消订单
     */
    public void cancelOrder(CancelOrderRequest request) {
        // 1. 查询订单
        SaleOrder order = orderRepository.findById(request.getOrderId())
            .orElseThrow(() -> new OrderNotFoundException(request.getOrderId()));

        // 2. 取消订单
        order.cancel(request.getCancelReason());

        // 3. 保存订单
        orderRepository.save(order);

        // 4. 发布领域事件
        eventPublisher.publish(order.getDomainEvents());
    }

    /**
     * 验证创建订单请求
     */
    private void validateCreateOrderRequest(CreateOrderRequest request) {
        if (request.getStoreId() == null || request.getStoreId().isEmpty()) {
            throw new IllegalArgumentException("门店ID不能为空");
        }
        if (request.getItems() == null || request.getItems().isEmpty()) {
            throw new IllegalArgumentException("订单明细不能为空");
        }
        if (request.getChannelType() == null) {
            throw new IllegalArgumentException("渠道类型不能为空");
        }
        if (request.getOrderScene() == null) {
            throw new IllegalArgumentException("订单场景不能为空");
        }
        if (request.getFulfillmentType() == null) {
            throw new IllegalArgumentException("履约类型不能为空");
        }
    }

    /**
     * 构建订单明细项
     */
    private List<CreateOrderItemCommand> buildOrderItems(
            CreateOrderRequest request, List<ProductSnapshot> snapshots) {
        Map<String, ProductSnapshot> snapshotMap = snapshots.stream()
            .collect(Collectors.toMap(ProductSnapshot::getProductId, Function.identity()));

        return request.getItems().stream()
            .map(item -> {
                ProductSnapshot snapshot = snapshotMap.get(item.getProductId());
                return CreateOrderItemCommand.builder()
                    .productId(item.getProductId())
                    .productSnapshot(snapshot)
                    .quantity(item.getQuantity())
                    .unitPrice(snapshot.getBasePrice())
                    .subtotal(snapshot.getBasePrice() * item.getQuantity())
                    .discountAmount(0L)
                    .remark(item.getRemark())
                    .build();
            })
            .collect(Collectors.toList());
    }
}
```

```java
        // 构建查询条件
        Specification<SaleOrder> spec = (root, query, cb) -> {
            List<Predicate> predicates = new ArrayList<>();

            if (StringUtils.hasText(request.getStoreId())) {
                predicates.add(cb.equal(root.get("storeId"), request.getStoreId()));
            }

            if (StringUtils.hasText(request.getCustomerId())) {
                predicates.add(cb.equal(root.get("customerId"), request.getCustomerId()));
            }

            if (request.getStatus() != null) {
                predicates.add(cb.equal(root.get("status"), request.getStatus()));
            }

            if (request.getStartDate() != null) {
                predicates.add(cb.greaterThanOrEqualTo(root.get("createdAt"), request.getStartDate()));
            }

            if (request.getEndDate() != null) {
                predicates.add(cb.lessThanOrEqualTo(root.get("createdAt"), request.getEndDate()));
            }

            return cb.and(predicates.toArray(new Predicate[0]));
        };

        // 分页查询
        Pageable pageable = PageRequest.of(
            request.getPageNum() - 1,
            request.getPageSize(),
            Sort.by(Sort.Direction.DESC, "createdAt")
        );

        Page<SaleOrder> page = orderRepository.findAll(spec, pageable);

        // 转换为 DTO
        List<OrderListDTO> dtos = page.getContent().stream()
            .map(OrderListDTO::from)
            .collect(Collectors.toList());

        return PageResult.of(dtos, page.getTotalElements());
    }
```

```java
@Component
public class OrderAmountCalculator {

    /**
     * 计算订单金额
     */
    public OrderAmount calculate(CreateOrderRequest request, List<ProductSnapshot> snapshots) {
        Map<String, ProductSnapshot> snapshotMap = snapshots.stream()
            .collect(Collectors.toMap(ProductSnapshot::getProductId, Function.identity()));

        long totalAmount = 0L;
        long discountAmount = 0L;

        for (CreateOrderRequest.OrderItemRequest item : request.getItems()) {
            ProductSnapshot snapshot = snapshotMap.get(item.getProductId());
            if (snapshot == null) {
                throw new ProductNotFoundException(item.getProductId());
            }

            long itemTotal = snapshot.getBasePrice() * item.getQuantity();

            // 加上规格加价
            if (item.getSpecifications() != null) {
                for (String specName : item.getSpecifications()) {
                    Long specPrice = snapshot.getSpecificationPrice(specName);
                    if (specPrice != null) {
                        itemTotal += specPrice * item.getQuantity();
                    }
                }
            }

            totalAmount += itemTotal;
        }

        // 计算优惠金额（后续对接优惠券系统）
        discountAmount = 0L;

        long payableAmount = totalAmount - discountAmount;

        return OrderAmount.builder()
            .totalAmount(totalAmount)
            .discountAmount(discountAmount)
            .payableAmount(payableAmount)
            .build();
    }
}
```

```java
/**
 * 订单号生成器 — 使用 Snowflake 算法，保证多 Pod 部署下全局唯一。
 *
 * 原设计使用 static AtomicLong，在 K8s 多副本部署时各 Pod 独立计数，

 *   [符号位 1] [时间戳差 41位] [数据中心ID 5位] [机器ID 5位] [序列号 12位]

@Component

                            @Value("${app.snowflake.machine-id:1}") long machineId) {
        this.snowflake = new SnowflakeIdGenerator(datacenterId, machineId);
    }

    /**
     * 生成订单号：门店编码(4位) + Snowflake ID(18位)
     * 示例：00012025041500000000001
     * machineId 从 K8s Pod 环境变量 POD_INDEX 注入（StatefulSet 场景）
```

### 5.4 DomainEventPublisher（领域事件发布器）

> **架构红线（来自 00-技术架构总览 R12）：** 业务数据修改与事件写入必须在**同一本地事务**中完成，禁止在事务提交后直接向 MQ 发送消息。
>
> **正确的 Outbox 实现流程：**
> 1. 领域对象在状态机方法中调用 `addDomainEvent()` 记录待发布事件（内存暂存）
> 2. 应用服务将聚合根写入 DB + 将事件插入 `domain_event_outbox` 表，在**同一事务**中提交
> 3. `OutboxRelayService`（独立进程/定时任务）轮询 `domain_event_outbox` 表，读取未发送记录
> 4. 将事件发布到 RocketMQ，成功后更新 outbox 记录状态为 `SENT`
>
> 本类的 `publish()` 方法用于在 Spring 进程内触发 `@EventListener`，监听器负责将事件写入 Outbox 表（步骤 2），而不是直接推送 MQ。

```java
@Component
public class DomainEventPublisher {

    @Autowired
    private ApplicationEventPublisher applicationEventPublisher;

    /**
     * 发布领域事件列表（进程内路由，最终由 @EventListener 写入 Outbox 表）
     * 注意：调用方必须在 @Transactional 方法内，确保事件写入与业务操作在同一事务
     */
    public void publish(List<DomainEvent> events) {
        if (events == null || events.isEmpty()) {
            return;
        }

        for (DomainEvent event : events) {
            applicationEventPublisher.publishEvent(event);
        }
    }

    /**
     * 发布单个领域事件
     */
    public void publish(DomainEvent event) {
        applicationEventPublisher.publishEvent(event);
    }
}
```

```java
    /**
     * 检查订单取消幂等性
     */
    public boolean checkCancelIdempotency(String orderId) {

        Boolean success = redisTemplate.opsForValue().setIfAbsent(
            key,
            "1",
            24,
            TimeUnit.HOURS
        );
        return Boolean.TRUE.equals(success);
    }
}
```

## 12. 支付后改单（加菜/退菜）完整流程

### 12.1 业务场景与约束

支付后改单发生在订单已支付（PAID/ACCEPTED/IN_PRODUCTION）但尚未完成的阶段，主要场景：
- 堂食顾客加菜（最常见）
- 顾客要求退掉某道菜（需退款）
- 换菜（退一道加一道）

**核心约束：**
- 已进入生产的工作单不可撤回（`work_unit.status = IN_PROGRESS/COMPLETED`）
- 加菜必须重新锁定库存
- 退菜必须触发部分退款流程
- 改单操作必须幂等（网络重试安全）
- 改单记录必须完整保留（审计）

### 12.2 状态机扩展

```
订单状态允许改单的窗口：
  PAID          → 可加菜、可退菜
  ACCEPTED      → 可加菜、可退菜（未开始生产的菜）
  IN_PRODUCTION → 可加菜；退菜仅限未开始生产的工作单
  READY         → 不允许改单
  COMPLETED     → 不允许改单（走售后退款）
```

### 12.3 加菜流程

```
POST /api/v1/orders/{orderId}/items
Body: { items: [{productId, variantId, quantity, modifiers}] }

1. 校验订单状态（PAID/ACCEPTED/IN_PRODUCTION）
2. 幂等检查（idempotency_key）
3. 获取商品快照（当前价格，非下单时价格）
4. 锁定库存
5. 创建新 OrderItem（status=ADDED，关联 parent_order_id）
6. 计算增量金额，更新 sale_order.total_amount
7. 发起增量支付（微信/支付宝补差价）
8. 支付确认后：
   a. OrderItem.status → CONFIRMED
   b. 发布 OrderItemAdded 事件 → 履约域创建新工作单
9. 写入 order_modification_log
```

```java
@Transactional
public AddItemResult addOrderItems(String orderId, AddItemCommand cmd) {
    // 幂等检查
    String idempotentKey = "order:add-item:" + orderId + ":" + cmd.getIdempotencyKey();
    if (redisTemplate.hasKey(idempotentKey)) {
        return getExistingAddItemResult(orderId, cmd.getIdempotencyKey());
    }

    SaleOrder order = orderRepository.findByIdWithLock(orderId)
        .orElseThrow(() -> new OrderNotFoundException(orderId));

    order.validateModifiable(); // 校验状态窗口

    // 获取商品快照（改单时用当前价格）
    List<OrderItem> newItems = cmd.getItems().stream()
        .map(item -> productSnapshotService.createSnapshot(item, order.getStoreId()))
        .collect(toList());

    long incrementAmount = newItems.stream().mapToLong(OrderItem::getSubtotal).sum();

    // 创建增量支付单
    Payment incrementPayment = Payment.createIncrement(orderId, incrementAmount, cmd.getPaymentMethod());
    paymentRepository.save(incrementPayment);

    // 暂存新订单行（待支付确认后激活）
    newItems.forEach(item -> item.setPendingConfirmation(incrementPayment.getPaymentId()));
    orderItemRepository.saveAll(newItems);

    // 记录改单日志
    orderModificationLogRepository.save(OrderModificationLog.addItems(orderId, newItems, incrementAmount));

    redisTemplate.opsForValue().set(idempotentKey, incrementPayment.getPaymentId(), 24, TimeUnit.HOURS);

    return AddItemResult.of(incrementPayment);
}
```

### 12.4 退菜流程

```
POST /api/v1/orders/{orderId}/items/{itemId}/remove
Body: { reason, quantity }

1. 校验订单状态
2. 校验工作单状态（work_unit.status 必须为 PENDING/DISPATCHED，不能是 IN_PROGRESS/COMPLETED）
3. 撤回对应工作单（发 CancelWorkUnit 命令到履约域）
4. 释放库存
5. 计算退款金额（按比例退，含优惠分摊）
6. 发起部分退款（见第13节）
7. 更新 OrderItem.status = REMOVED
8. 更新 sale_order.total_amount
9. 写入 order_modification_log
```

## 13. 部分退款状态机

### 13.1 退款类型

| 退款类型 | 触发场景 | 退款金额 |
|---|---|---|
| FULL_REFUND | 整单取消 | 实付金额全额 |
| PARTIAL_ITEM_REFUND | 退菜 | 该菜品金额（含优惠分摊） |
| PARTIAL_AMOUNT_REFUND | 商家补偿（少量） | 指定金额 |
| CHANNEL_REFUND | 渠道平台发起退款 | 渠道账单金额 |

### 13.2 退款状态机

```
REFUND_PENDING（退款申请已创建）
    ↓ 调用支付渠道退款接口
REFUND_PROCESSING（退款处理中）
    ↓ 支付渠道异步回调
REFUND_SUCCESS（退款成功）
    ↓ 或
REFUND_FAILED（退款失败）
    ↓ 人工介入 / 重试
REFUND_PROCESSING（重新发起）
```

**订单级退款状态（payment_refund_status）：**
```
NONE → PARTIAL_REFUNDING → PARTIALLY_REFUNDED
NONE → FULL_REFUNDING    → FULLY_REFUNDED
PARTIALLY_REFUNDED → FULL_REFUNDING → FULLY_REFUNDED
```

### 13.3 优惠分摊退款计算

退菜时，优惠券折扣需按比例分摊到各订单行：

```java
public long calculateRefundAmount(OrderItem item, SaleOrder order) {
    // 该行原始金额占总原始金额的比例
    BigDecimal itemRatio = BigDecimal.valueOf(item.getOriginalAmount())
        .divide(BigDecimal.valueOf(order.getOriginalAmount()), 6, HALF_UP);

    // 该行承担的优惠金额
    long itemDiscount = itemRatio.multiply(BigDecimal.valueOf(order.getDiscountAmount()))
        .setScale(0, HALF_UP).longValue();

    // 实际退款 = 原始金额 - 分摊优惠
    return item.getOriginalAmount() - itemDiscount;
}
```

### 13.4 退款幂等与并发控制

```java
@Transactional
public RefundResult initiateRefund(RefundCommand cmd) {
    // 幂等键：orderId + refundId（业务方生成）
    String lockKey = "refund:lock:" + cmd.getOrderId();
    boolean locked = redisTemplate.opsForValue()
        .setIfAbsent(lockKey, cmd.getRefundId(), 30, TimeUnit.SECONDS);
    if (!locked) throw new ConcurrentRefundException(cmd.getOrderId());

    try {
        // 检查是否已存在相同 refundId 的退款记录
        if (refundRepository.existsByRefundId(cmd.getRefundId())) {
            return refundRepository.findByRefundId(cmd.getRefundId()).toResult();
        }

        SaleOrder order = orderRepository.findByIdWithLock(cmd.getOrderId())
            .orElseThrow();

        // 校验退款金额不超过可退金额
        long refundable = order.getPaidAmount() - order.getTotalRefundedAmount();
        if (cmd.getAmount() > refundable) {
            throw new RefundAmountExceededException(cmd.getAmount(), refundable);
        }

        // 创建退款记录（与状态更新同一事务）
        RefundRecord record = RefundRecord.create(cmd);
        refundRepository.save(record);

        // 更新订单退款状态
        order.addRefund(cmd.getAmount());
        orderRepository.save(order);

        // 异步调用支付渠道退款接口
        refundAsyncService.submitToPaymentChannel(record);

        return record.toResult();
    } finally {
        redisTemplate.delete(lockKey);
    }
}
```
