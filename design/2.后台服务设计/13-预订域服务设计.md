# 预订域服务设计

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05
> 服务名：`booking-service` | 端口：8089 | 数据库：`booking_db`

---

## 1. 服务定位

预订域服务管理正餐预订场景的完整业务闭环，从顾客预订到到店开台。它是独立的限界上下文微服务，通过领域事件与交易域、租户与组织域解耦。

**核心聚合根：**
- `BookingOrder`（预订单）
- `BookingPeriod`（餐段配置）

## 2. 核心聚合设计

### 2.1 BookingOrder 聚合根

```java
@Entity
@Table(name = "booking_orders")
public class BookingOrder {

    @Id
    private String bookingId;
    private String bookingNo;
    private String storeId;
    private String tenantId;

    // 预订类型与来源
    @Enumerated(EnumType.STRING)
    private BookingType bookingType;       // B_END | C_END
    @Enumerated(EnumType.STRING)
    private BookingSourceChannel sourceChannel; // MINI_PROGRAM | PHONE | THIRD_PARTY | POS

    // 顾客信息
    private String customerName;
    private String customerPhone;          // 加密存储
    private String customerPhoneSuffix;    // 后4位，用于展示
    private Integer partySize;

    // 预订时间
    private LocalDate bookingDate;
    private String periodId;
    private String periodName;
    private LocalTime arrivalTime;

    // 桌台（可选，支持先预订后分桌）
    private String tableId;
    private String tableNo;

    // 订金
    private Long depositAmount;            // 单位：分，0表示无需订金
    @Enumerated(EnumType.STRING)
    private DepositStatus depositStatus;   // UNPAID | PAID | REFUNDED
    private String depositPaymentId;       // 关联支付记录
    @Enumerated(EnumType.STRING)
    private DepositRefundPolicy depositRefundPolicy; // NO_REFUND | FULL_REFUND | PARTIAL_REFUND

    // 状态
    @Enumerated(EnumType.STRING)
    private BookingStatus status;

    // 到店后关联
    private String sessionId;             // 到店后关联的 TableSession（主锚点）
    private String linkedOrderId;         // 首次点菜后由交易域回填，可为空

    // 时间戳
    private LocalDateTime createdAt;
    private LocalDateTime confirmedAt;
    private LocalDateTime arrivedAt;
    private LocalDateTime completedAt;
    private LocalDateTime cancelledAt;
    private LocalDateTime expireAt;       // 逾期判定时间

    private String remark;
    private String cancelReason;
    private String cancelInitiator;       // CUSTOMER | MERCHANT | SYSTEM

    @Version
    private Integer version;
}
```

**BookingStatus 状态机：**

```
PENDING（待确认）
    │ 门店确认
    ▼
CONFIRMED（已确认）
    │ 顾客到店
    ▼
ARRIVED（已到店）—— 必须已有 session_id，linked_order_id 可为空
    │ 完成就餐
    ▼
COMPLETED（已完成）

PENDING/CONFIRMED ──取消──► CANCELLED
CONFIRMED ──逾期15分钟──► OVERDUE（系统定时任务）
```

## 3. 应用服务

### 3.1 BookingApplicationService

```java
@Service
@Transactional
public class BookingApplicationService {

    /**
     * 创建预订单
     * 幂等键：clientBookingId（前端生成）
     */
    public BookingOrder createBooking(CreateBookingCommand cmd) {
        // 1. 检查餐段容量
        bookingAvailabilityService.checkCapacity(cmd.getStoreId(), cmd.getPeriodId(), cmd.getPartySize());
        // 2. 创建预订单
        BookingOrder booking = BookingOrder.create(cmd);
        bookingRepository.save(booking);
        // 3. 发布事件
        eventPublisher.publish(new BookingCreatedEvent(booking));
        return booking;
    }

    /**
     * 门店确认预订
     */
    public void confirmBooking(String bookingId, String operatorId) {
        BookingOrder booking = bookingRepository.findById(bookingId);
        booking.confirm(operatorId);
        bookingRepository.save(booking);
        eventPublisher.publish(new BookingConfirmedEvent(booking));
    }

    /**
     * 确认顾客到店 — 预订域与交易域的关键衔接点
     * 到店后：预订域发布 BookingArrivedEvent，交易域消费后创建 TableSession
     * session_id 由交易域通过 BookingSessionLinkedEvent 回填
     */
    public void confirmArrival(String bookingId, String operatorId) {
        BookingOrder booking = bookingRepository.findById(bookingId);
        booking.arrive(operatorId);
        bookingRepository.save(booking);
        // 发布到店事件，交易域消费后创建 TableSession
        eventPublisher.publish(new BookingArrivedEvent(booking));
    }

    /**
     * 取消预订
     * 取消后：预订域发布 BookingCancelledEvent
     * 交易域消费后判断退款规则，调用外部集成域支付网关执行退款
     */
    public void cancelBooking(String bookingId, String reason, String initiator) {
        BookingOrder booking = bookingRepository.findById(bookingId);
        booking.cancel(reason, initiator);
        bookingRepository.save(booking);
        eventPublisher.publish(new BookingCancelledEvent(booking));
    }

    /**
     * 回填 session_id（由交易域通过事件驱动调用）
     */
    public void linkSession(String bookingId, String sessionId) {
        BookingOrder booking = bookingRepository.findById(bookingId);
        booking.linkSession(sessionId);
        bookingRepository.save(booking);
    }

    /**
     * 回填 linked_order_id（首次点菜后由交易域回填）
     */
    public void linkOrder(String bookingId, String orderId) {
        BookingOrder booking = bookingRepository.findById(bookingId);
        booking.linkOrder(orderId);
        bookingRepository.save(booking);
    }
}
```

## 4. 领域事件

| 事件名 | 触发时机 | 消费方 |
|--------|----------|--------|
| `BookingCreatedEvent` | 预订单创建成功 | 外部集成域（通知顾客）、运营治理域（审计） |
| `BookingConfirmedEvent` | 门店确认预订 | 外部集成域（发送确认通知）、租户与组织域（桌台变 RESERVED） |
| `BookingArrivedEvent` | 顾客到店确认 | 交易域（创建 TableSession）、租户与组织域（桌台变 OCCUPIED） |
| `BookingCancelledEvent` | 预订取消 | 租户与组织域（释放桌台）、外部集成域（通知顾客）、交易域（退订金） |
| `BookingNoShowEvent` | 逾期未到店 | 运营治理域（爽约统计）、外部集成域（通知顾客） |

## 5. 订金支付/退款边界

```
订金支付流程：
  顾客创建预订 → booking-service 生成 depositPaymentId
  → 调用 transaction-service 发起支付
  → 支付成功后 transaction-service 发布 PaymentConfirmedEvent
  → booking-service 消费事件，更新 depositStatus = PAID

订金退款流程：
  booking-service 发布 BookingCancelledEvent（携带 depositPaymentId 和 depositRefundPolicy）
  → transaction-service 消费事件，根据 depositRefundPolicy 判断退款金额
  → transaction-service 调用 external-service 支付网关执行退款
  → 退款完成后 booking-service 更新 depositStatus = REFUNDED
```

> **边界说明：** booking-service 不直接调用支付网关，退款决策和执行由 transaction-service 负责，booking-service 只负责发布事件和更新订金状态。

## 6. 到店转 TableSession 协作流程

```
1. POS 操作员点击"到店确认"
   → booking-service.confirmArrival()
   → 发布 BookingArrivedEvent（含 bookingId, tableId, partySize）

2. transaction-service 消费 BookingArrivedEvent
   → 创建 TableSession（关联 bookingId）
   → 发布 TableSessionCreatedEvent（含 sessionId, bookingId）

3. booking-service 消费 TableSessionCreatedEvent
   → 调用 linkSession(bookingId, sessionId)
   → BookingOrder.status 变为 ARRIVED，session_id 填入

4. 顾客首次点菜提交订单
   → transaction-service 创建 SaleOrder（关联 sessionId）
   → 发布 OrderCreatedEvent（含 orderId, sessionId, bookingId）

5. booking-service 消费 OrderCreatedEvent（当 bookingId 不为空时）
   → 调用 linkOrder(bookingId, orderId)
   → linked_order_id 填入
```

## 7. 数据库表设计

```sql
-- 预订单表
CREATE TABLE booking_orders (
    booking_id          VARCHAR(36) PRIMARY KEY,
    booking_no          VARCHAR(32) NOT NULL UNIQUE,
    store_id            VARCHAR(36) NOT NULL,
    tenant_id           VARCHAR(36) NOT NULL,
    booking_type        VARCHAR(16) NOT NULL,       -- B_END | C_END
    source_channel      VARCHAR(32) NOT NULL,
    customer_name       VARCHAR(64) NOT NULL,
    customer_phone      VARCHAR(256) NOT NULL,      -- 加密存储
    customer_phone_suffix VARCHAR(4) NOT NULL,
    party_size          INTEGER NOT NULL,
    booking_date        DATE NOT NULL,
    period_id           VARCHAR(36) NOT NULL,
    period_name         VARCHAR(64) NOT NULL,
    arrival_time        TIME NOT NULL,
    table_id            VARCHAR(36),
    table_no            VARCHAR(32),
    deposit_amount      BIGINT NOT NULL DEFAULT 0,
    deposit_status      VARCHAR(16) NOT NULL DEFAULT 'UNPAID',
    deposit_payment_id  VARCHAR(36),
    deposit_refund_policy VARCHAR(32),
    status              VARCHAR(32) NOT NULL DEFAULT 'PENDING',
    session_id          VARCHAR(36),               -- 到店后关联 TableSession
    linked_order_id     VARCHAR(36),               -- 首次点菜后回填
    remark              TEXT,
    cancel_reason       TEXT,
    cancel_initiator    VARCHAR(16),
    expire_at           TIMESTAMP WITH TIME ZONE,
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    confirmed_at        TIMESTAMP WITH TIME ZONE,
    arrived_at          TIMESTAMP WITH TIME ZONE,
    completed_at        TIMESTAMP WITH TIME ZONE,
    cancelled_at        TIMESTAMP WITH TIME ZONE,
    version             INTEGER NOT NULL DEFAULT 0
);

CREATE INDEX idx_booking_store_date ON booking_orders(store_id, booking_date);
CREATE INDEX idx_booking_status ON booking_orders(store_id, status);
CREATE INDEX idx_booking_session ON booking_orders(session_id) WHERE session_id IS NOT NULL;

-- 餐段配置表
CREATE TABLE booking_periods (
    period_id               VARCHAR(36) PRIMARY KEY,
    store_id                VARCHAR(36) NOT NULL,
    period_name             VARCHAR(64) NOT NULL,
    start_time              TIME NOT NULL,
    end_time                TIME NOT NULL,
    max_capacity            INTEGER NOT NULL,
    advance_booking_days    INTEGER NOT NULL DEFAULT 7,
    cutoff_time             TIME,
    deposit_required        BOOLEAN NOT NULL DEFAULT FALSE,
    deposit_amount          BIGINT NOT NULL DEFAULT 0,
    refund_policy           VARCHAR(32),
    is_active               BOOLEAN NOT NULL DEFAULT TRUE,
    created_at              TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at              TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_period_store ON booking_periods(store_id, is_active);
```

## 8. 幂等与补偿

- 创建预订：前端传 `clientBookingId`，服务端以此做幂等去重（窗口期 24h）
- 确认到店：操作幂等，重复调用检查 `status == CONFIRMED` 才执行
- 逾期检查：定时任务每分钟扫描 `status = CONFIRMED AND expire_at < NOW()`，批量标记 `OVERDUE`
- 事件消费：所有事件消费均基于 `bookingId` 做幂等，防止重复处理
