# 自助点餐机详细设计

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

---

## 一、产品定位与边界

### 1.1 定位

自助点餐机是**门店落地式自助终端**，让顾客无需排队等待服务员，独立完成点餐和支付。

核心价值：
- 分流收银台压力，减少高峰期排队
- 支持扫码/刷脸支付，全程无接触
- 打印取餐号，配合叫号屏使用

### 1.2 技术形态

- React + Electron，Android Kiosk 模式（锁定全屏，禁止退出）
- 屏幕：21-27 寸竖屏触摸屏
- 硬件：扫码枪（付款码）、热敏打印机（取餐号）、摄像头（刷脸，可选）
- 设备级 Token，无需员工登录

## 二、页面流程

```
待机页（广告轮播）
      │ 触摸屏幕
      ▼
就餐方式选择
  ├── 堂食（扫桌台码）
  └── 自取（直接点餐）
      │
      ▼
菜单页（选菜）
      │
      ▼
购物车确认页
      │
      ▼
支付页
  ├── 微信扫码支付
  ├── 支付宝扫码支付
  └── 刷脸支付（可选）
      │
      ▼
支付结果页
      │ 打印取餐号
      ▼
待机页
```

## 三、核心页面详细设计

### 3.1 待机页

无操作 60 秒后自动进入待机状态：
- 全屏广告轮播（图片/视频）
- 底部显示"触摸屏幕开始点餐"提示
- 广告内容由商户后台配置

### 3.2 就餐方式选择页

```
┌─────────────────────────────┐
│                             │
│      欢迎光临 [门店名]       │
│                             │
│  ┌──────────┐ ┌──────────┐ │
│  │          │ │          │ │
│  │   堂食   │ │   自取   │ │
│  │  扫桌台码 │ │  到店取餐 │ │
│  │          │ │          │ │
│  └──────────┘ └──────────┘ │
│                             │
└─────────────────────────────┘
```

选择"堂食"后，展示扫码提示，顾客扫桌台码确认桌台。

### 3.3 菜单页

自助机菜单页为竖屏大字体设计，适合站立操作：

```
┌─────────────────────────────┐
│ ← [门店名]  自取            │
├──────────┬──────────────────┤
│          │                  │
│  分类    │  商品列表         │
│  导航    │                  │
│  （左侧） │  ┌─────────────┐ │
│          │  │  [大图]      │ │
│  热销    │  │  红烧肉      │ │
│  招牌菜  │  │  ¥38        │ │
│  主食    │  │  [+]        │ │
│  饮品    │  └─────────────┘ │
│          │                  │
├──────────┴──────────────────┤
│  [购物车] 3件  ¥44  [去结算] │
└─────────────────────────────┘
```

**自助机特殊设计：**
- 商品图片更大（适合远距离查看）
- 字体比 H5 大 20%
- 按钮点击区域更大（≥ 60px，适合手指触摸）
- 无优惠券选择（简化流程，优惠自动应用）

### 3.4 支付页

```
┌─────────────────────────────┐
│  请选择支付方式              │
│  实付：¥44.00               │
├─────────────────────────────┤
│  ┌──────────┐ ┌──────────┐ │
│  │  微信    │ │  支付宝  │ │
│  │  扫码    │ │  扫码    │ │
│  └──────────┘ └──────────┘ │
│                             │
│  ┌──────────────────────┐  │
│  │  刷脸支付（可选）     │  │
│  └──────────────────────┘  │
├─────────────────────────────┤
│  [← 返回修改]               │
└─────────────────────────────┘
```

选择支付方式后：
- 扫码支付：展示收款二维码，顾客用手机扫码
- 付款码支付：提示顾客出示付款码，扫码枪扫描

**支付超时处理：**
- 展示收款码后，120 秒内未支付自动返回支付方式选择页
- 倒计时显示在页面底部

### 3.5 支付结果页

```
┌─────────────────────────────┐
│                             │
│         ✅ 支付成功          │
│                             │
│      取餐号：018             │
│                             │
│  正在打印取餐号...           │
│                             │
│  请在取餐区等待叫号          │
│                             │
│  [5秒后自动返回首页]         │
│                             │
└─────────────────────────────┘
```

支付成功后：
1. 自动打印取餐号小票
2. 5 秒后自动返回待机页
3. 取餐号小票内容：

```
================================
        [门店名称]
================================
取餐号：018
时间：12:40:15
--------------------------------
红烧肉（中份/微辣）× 1
白米饭 × 2
================================
请在取餐区等待叫号
================================
```

## 四、异常场景处理

| 场景 | 处理方式 |
|------|----------|
| 打印机缺纸 | 提示"打印机暂时无法使用，请记住您的取餐号：018" |
| 支付失败 | 展示失败原因，提供重试入口 |
| 网络断开 | 展示"系统维护中，请前往收银台点餐"，禁用所有操作 |
| 商品售罄 | 提交时提示售罄，引导替换 |
| 长时间无操作 | 60 秒后返回待机页，购物车清空 |

## 五、API 接口清单

与顾客小程序基本一致，额外接口：

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 菜单 | GET | `/api/v1/stores/{storeId}/menu` | `scene=KIOSK` |
| 提交订单 | POST | `/api/v1/orders` | — |
| 展示收款码 | POST | `/api/v1/payments/qr-pay` | 被扫模式 |
| 查询支付状态 | GET | `/api/v1/orders/{orderId}/payment-status` | 轮询 |
| 打印取餐号 | POST | `/api/v1/print-tasks` | `type=PICKUP_TICKET` |
