## 一、体系概述

### 1.1 定位

打印体系是**贯穿所有终端的基础能力**，不是独立终端，而是被 POS、打包台、KDS 等调用的公共服务。

核心能力：
- 统一的打印任务管理（队列、重试、状态跟踪）
- 多种打印机类型支持（小票、厨房、标签）
- 小票模板管理（可配置）
- 离线打印队列（网络恢复后补打）

### 1.2 打印机类型

| 类型 | 纸宽 | 典型用途 | 接入方式 |
|------|------|----------|----------|
| 小票打印机 | 80mm | 顾客小票、收银凭证 | USB/网络 |
| 厨房打印机 | 80mm | 厨房工单（替代KDS） | USB/网络 |
| 标签打印机 | 40-60mm | 外卖打包标签 | USB/网络 |

### 1.3 打印架构

```
前台终端（POS/打包台/KDS）
        │ 创建打印任务
        ▼
   后台打印服务
        │ 推送打印指令
        ▼
   本地打印 Agent（门店内网）
        │ 驱动打印机
        ▼
      打印机
```

本地打印 Agent 是部署在门店内网的轻量服务（Node.js 或 Java），负责：
- 接收后台推送的打印指令（WebSocket 长连接）
- 调用本地打印机驱动（ESC/POS 指令集）
- 上报打印结果

**打印机通信协议：**

| 接入方式 | 协议 | 说明 |
|----------|------|------|
| USB 直连 | ESC/POS over USB | Agent 直接写入 USB 设备文件 |
| 网络打印机 | TCP Socket（端口 9100） | Agent 建立 TCP 连接发送 ESC/POS 指令 |
| 蓝牙打印机 | BLE + ESC/POS | 仅自助机/PDA 场景 |

**ESC/POS 关键指令：**
```
ESC @          // 初始化打印机
ESC a n        // 对齐方式（0=左，1=居中，2=右）
ESC ! n        // 字体大小（bit0=加宽，bit4=加高）
GS V n         // 切纸（n=0 全切，n=1 半切）
ESC p m t1 t2  // 开钱箱
```

**打印任务重试策略：**
- 首次失败：立即重试 1 次
- 仍失败：等待 5s 后重试
- 第 3 次失败：进入离线队列，标记为 `FAILED`，等待打印机恢复后补打
- 最大重试次数：3 次

## 二、打印任务模型

```typescript
interface PrintTask {
  taskId: string;
  type: PrintTaskType;
  orderId?: string;
  workUnitId?: string;
  printerId: string;
  template: string;          // 模板名称
  data: Record<string, any>; // 模板渲染数据
  status: 'PENDING' | 'PRINTING' | 'DONE' | 'FAILED';
  retryCount: number;
  createdAt: string;
  doneAt?: string;
  errorMsg?: string;
}

type PrintTaskType =
  | 'ORDER_RECEIPT'      // 顾客小票
  | 'KITCHEN_TICKET'     // 厨房工单
  | 'PACK_LABEL'         // 打包标签
  | 'REFUND_RECEIPT'     // 退款凭证
  | 'SHIFT_REPORT'       // 交班报表
  | 'REPRINT';           // 补打（任意类型）
```

## 四、打印触发时机

| 触发场景 | 打印类型 | 触发终端 | 自动/手动 |
|----------|----------|----------|-----------|
| 订单支付成功 | 顾客小票 | POS | 自动 |
| 门店接单 | 厨房工单 | POS/后台 | 自动（可配置） |
| 打包完成 | 打包标签 | 打包台 | 手动触发 |
| 退款完成 | 退款凭证 | POS | 自动 |
| 交班 | 交班报表 | POS | 手动触发 |
| 补打 | 任意类型 | POS/打包台 | 手动触发 |

## 五、打印机管理

### 5.1 打印机注册

```
POST /api/v1/printers/register
{
  "storeId": "store_001",
  "printerName": "前台小票机",
  "printerType": "RECEIPT",
  "connectionType": "NETWORK",
  "ipAddress": "192.168.1.100",
  "port": 9100
}
```

### 5.2 打印机状态监控

本地 Agent 每 30 秒上报打印机状态：
- `ONLINE`：正常
- `OFFLINE`：无法连接
- `PAPER_LOW`：纸张不足
- `PAPER_OUT`：缺纸
- `ERROR`：硬件故障

前台终端收到打印机离线推送时，展示警告提示。

### 5.3 离线打印队列

打印机离线时，打印任务进入本地队列：
- 队列存储在本地 Agent 内存 + 磁盘（防止 Agent 重启丢失）
- 打印机恢复后，自动按顺序补打
- 超过 2 小时的厨房工单不补打（已过时）
- 顾客小票和退款凭证始终补打
