# 预订前台与门店协同设计

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

---

## 一、业务概述

### 1.1 预订链路全景

```
顾客侧                          门店侧
  │                               │
  ├─ 查询可预订餐段                │
  ├─ 创建预订单                    │
  ├─ 支付订金（可选）              │
  │                               ├─ POS 收到预订通知
  │                               ├─ 门店确认预订
  ├─ 收到确认通知                  │
  │                               │
  │（到店当天）                    │
  ├─ 收到到店提醒                  │
  ├─ 到店                         ├─ POS 确认顾客到店
  │                               ├─ 分配桌台，建立 TableSession
  │                               ├─ 衔接正常点餐流程
  │                               │
  │（逾期未到）                    │
  │                               ├─ 标记 OVERDUE，释放桌台
  │                               │
  ├─ 取消预订                      ├─ 收到取消通知，释放桌台
```

### 1.2 预订对象说明

| 对象 | 说明 |
|------|------|
| `BookingOrder` | 预订单，独立于 `SaleOrder`，有自己的状态机 |
| `BookingSlot` | 餐段（如"午市 11:00-13:30"），门店在后台配置 |
| `BookingDeposit` | 订金支付记录，可抵扣正餐消费 |

## 二、顾客侧（小程序）

### 2.1 预订入口

顾客小程序门店详情页增加"预订"入口，与"点餐"并列：

```
┌─────────────────────────────┐

│  ⭐4.8  营业中  距您1.2km    │
├─────────────────────────────┤
│  [点餐]  [预订]  [外卖]      │
└─────────────────────────────┘
```

### 2.2 餐段查询页

```
┌─────────────────────────────┐
│  选择就餐时间                │
├─────────────────────────────┤
│  [< 4月5日(今天) >]          │

│  ┌──────┐ ┌──────┐           │
│  │12:30 │ │13:00 │           │
│  │已满  │ │可预订│           │
│  └──────┘ └──────┘           │
│                              │
│  晚市                        │
```

**餐段状态：**
| 状态 | 展示 | 可操作 |
|------|------|--------|
| AVAILABLE | "可预订" | ✓ |
| LIMITED | "剩N位" | ✓（N≤5时展示） |
| FULL | "已满" | — |
| CLOSED | "不可预订" | — |

### 2.3 创建预订页

选择餐段后进入：

```
┌─────────────────────────────┐
│  确认预订信息                │
├─────────────────────────────┤
│  门店：XX火锅                │
│  时间：4月5日 12:00          │
│  餐段：午市                  │
├─────────────────────────────┤
│  就餐人数  [-] 2 [+]         │
│  联系姓名  [输入姓名]         │
│  联系电话  [输入手机号]       │
│  备注      [特殊需求...]      │
├─────────────────────────────┤
│  订金说明                    │
│  需支付订金 ¥50，可抵扣消费  │
│  取消预订退还订金（24h前）   │
├─────────────────────────────┤
│  [确认预订]                  │
└─────────────────────────────┘
```

提交后：
```
POST /api/v1/bookings
Idempotency-Key: {clientBookingId}
{
  "storeId": "store_001",
  "bookingType": "C_END",               // C_END=顾客自助，B_END=代客预订
  "sourceChannel": "MINI_PROGRAM",
  "periodId": "period_lunch_001",       // 餐段ID，替代旧的 slotId
  "bookingDate": "2024-04-05",
  "arrivalTime": "12:00",
  "partySize": 2,
  "customerName": "张三",
  "customerPhone": "138xxxx0001",
  "remark": "靠窗位置"
}
→ 返回 bookingId、depositAmount（订金金额，0表示无需订金）
→ 若 depositAmount > 0，跳转订金支付页
→ 若 depositAmount = 0，直接跳转预订详情页
```

### 2.4 订金支付页

```
┌─────────────────────────────┐
│  支付订金                    │
│  ¥50.00                     │
├─────────────────────────────┤
│  [微信支付]  [支付宝]        │
├─────────────────────────────┤
│  订金可抵扣正餐消费          │
│  24小时前取消全额退还        │
├─────────────────────────────┤
│  [立即支付]                  │
└─────────────────────────────┘
```

支付成功后状态变为 `CONFIRMED`（若门店设置自动确认）或 `PENDING`（需门店手动确认）。

### 2.5 预订详情页

```
┌─────────────────────────────┐
│  预订详情                    │
│  预订号：#BK20240405001      │
│  状态：已确认 ✅              │
├─────────────────────────────┤
│  XX火锅                      │
│  4月5日（周六）12:00         │
│  2人  午市                   │
├─────────────────────────────┤
│  联系人：张三                │
│  手机：138xxxx0001           │
├─────────────────────────────┤
│  订金：¥50（已支付）         │
│  到店后可抵扣消费            │
├─────────────────────────────┤
│  [取消预订]  [联系门店]      │
└─────────────────────────────┘
```

**状态展示规则：**
| 状态 | 展示文案 | 可操作 |
|------|----------|--------|
| PENDING | "待门店确认" | 取消 |
| CONFIRMED | "已确认 ✅" | 取消（24h前） |
| ARRIVED | "已到店" | — |
| COMPLETED | "已完成" | 再次预订 |
| CANCELLED | "已取消" | 再次预订 |
| OVERDUE | "逾期未到" | 再次预订 |

### 2.6 取消预订流程

```
点击"取消预订"
  → 展示取消规则：
    "24小时前取消：全额退还订金 ¥50"
    "24小时内取消：订金不退"
  → 选择取消原因（必填）
  → 确认取消
  → POST /api/v1/bookings/{bookingId}/cancel
  → 若符合退款条件，自动发起订金退款
  → 展示"预订已取消，订金将在3-5个工作日退回"
```

### 2.7 到店提醒

预订时间前 2 小时，系统推送小程序服务通知：
> "您在 XX火锅 的预订将于今日 12:00 开始，请准时到店。"

逾期超过 15 分钟未到店，推送提醒：
> "您的预订即将超时，请尽快到店或联系门店。"

## 三、门店侧（POS）

### 3.1 预订列表页

POS 左侧导航新增"预订管理"入口：

```
┌──────────────────────────────────────────────────────────────┐
│ 预订管理  [今日] [明日] [本周]  [日期筛选]                    │
├──────────────────────────────────────────────────────────────┤
│ 时间     姓名   人数  状态      操作                          │
│ 12:00   张三    2人  待确认    [确认] [拒绝]                  │
│ 12:30   李四    4人  已确认    [到店] [取消]                  │
│ 18:00   王五    6人  已确认    [到店] [取消]                  │
│ 18:30   赵六    3人  已到店    [分配桌台]                     │
└──────────────────────────────────────────────────────────────┘
```

### 3.2 确认预订

```
点击"确认"
  → 弹出确认框：
    "确认 张三 4月5日12:00 2人 的预订？"
    [确认]  [拒绝]
  → 确认：POST /api/v1/bookings/{bookingId}/confirm
  → 拒绝：需填写拒绝原因，POST /api/v1/bookings/{bookingId}/reject
  → 系统自动推送通知给顾客
```

### 3.3 顾客到店确认

```
点击"到店"
  → 弹出到店确认框：
    "确认 张三（2人）已到店？"
    桌台分配：[自动分配] 或 [手动选择桌台]
  → 确认：POST /api/v1/bookings/{bookingId}/arrive
  → 系统自动创建 TableSession，关联 bookingId
  → 跳转到桌台点单页，预填桌台信息
  → 订金自动记录为"待抵扣"状态
```

**订金抵扣规则：**
- 结账时，POS 收款页自动展示"预订订金抵扣 ¥50"
- 实付金额 = 消费金额 - 订金
- 若消费金额 < 订金，差额退还顾客

### 3.4 预订取消与桌台释放

```
点击"取消"
  → 选择取消原因
  → POST /api/v1/bookings/{bookingId}/cancel
  → 若该预订已分配桌台，自动释放桌台（状态回到 AVAILABLE）
  → 若顾客已支付订金且符合退款条件，自动发起退款
```

### 3.5 新预订实时提醒

新预订到达时（WebSocket 推送 `BOOKING_NEW`）：
- 顶部导航"预订管理"显示红点 + 数量
- 播放提示音
- 弹出悬浮卡片（5秒后收起）

### 3.6 代客预订（B_END）

收银员或店长在 POS 端代顾客创建预订（`bookingType: B_END`），适用于电话预订、现场预订等场景。

**代客预订入口：** 预订列表页右上角「新建预订」按钮。

**代客预订表单：**
```
┌──────────────────────────────────────────────────────────────┐
│ 新建预订                                                      │
├──────────────────────────────────────────────────────────────┤
│ 就餐日期   [选择日期]                                         │
│ 餐段       [午市 11:00-13:30 ▼]                              │
│ 到店时间   [12:00 ▼]                                         │
│ 就餐人数   [-] 2 [+]                                         │
│ 联系姓名   [输入姓名]                                         │
│ 联系电话   [输入手机号]                                       │
│ 指定桌台   [不指定 ▼]  ← 可选，选后锁定该桌台               │
│ 备注       [特殊需求...]                                      │
├──────────────────────────────────────────────────────────────┤
│ 订金       ¥50（按餐段配置）                                  │
│ 收款方式   [现场收取] [免收订金（需店长授权）]                │
├──────────────────────────────────────────────────────────────┤
│ [取消]                          [确认创建]                    │
└──────────────────────────────────────────────────────────────┘
```

**提交请求：**
```
POST /api/v1/bookings
Idempotency-Key: {clientBookingId}
{
  "storeId": "store_001",
  "bookingType": "B_END",
  "sourceChannel": "POS",
  "periodId": "period_lunch_001",
  "bookingDate": "2024-04-05",
  "arrivalTime": "12:00",
  "partySize": 2,
  "customerName": "张三",
  "customerPhone": "138xxxx0001",
  "preferredTableCode": "A05",   // 可选，指定桌台
  "remark": "靠窗位置",
  "depositWaived": false          // true = 免收订金（需授权）
}
```

**指定桌台逻辑：**
- 选择指定桌台后，系统检查该桌台在目标时段是否空闲
- 若冲突，提示"该桌台在所选时段已有预订，请选择其他桌台"
- 确认后，桌台状态变为`预订中`，其他预订不可再选该桌台

**免收订金：**
- 需店长授权码（同退款授权流程，见文档 04 §4.4）
- 授权后 `depositWaived: true`，预订单直接进入 `CONFIRMED` 状态

## 四、商户后台预订配置

### 4.1 餐段管理

```
┌──────────────────────────────────────────────────────────────┐
│ 餐段管理  [新增餐段]                                          │
├──────────────────────────────────────────────────────────────┤
│ 餐段名称  时间段          最大人数  订金   状态   操作        │
│ 午市     11:00-13:30     60人      ¥50   启用   [编辑][停用] │
│ 晚市     17:30-21:00     80人      ¥100  启用   [编辑][停用] │
└──────────────────────────────────────────────────────────────┘
```

**餐段配置项：**
| 配置项 | 说明 |
|--------|------|
| 时间段 | 开始/结束时间 |
| 最大接待人数 | 超出后显示"已满" |
| 每时段间隔 | 如每30分钟一个时间点 |
| 订金金额 | 0 = 无需订金 |
| 订金退款规则 | 几小时前取消可退 |
| 提前预订天数 | 最多可提前几天预订 |
| 自动确认 | 开启后无需人工确认 |

### 4.2 预订管理页

商户后台"预订管理"模块，支持：
- 按日期/状态筛选预订列表
- 查看预订详情（含订金状态）
- 手动确认/拒绝/取消预订
- 导出预订数据（CSV）
- 预订统计（到店率、爽约率）

## 五、消息通知

| 消息类型 | 触发时机 | 接收方 |
|----------|----------|--------|
| `BOOKING_NEW` | 顾客创建预订 | POS |
| `BOOKING_CONFIRMED` | 门店确认预订 | 顾客小程序 |
| `BOOKING_REJECTED` | 门店拒绝预订 | 顾客小程序 |
| `BOOKING_CANCELLED` | 任意方取消 | 对方（顾客/门店） |
| `BOOKING_REMINDER` | 预订前2小时 | 顾客小程序 |
| `BOOKING_ARRIVED` | 顾客到店确认 | 商户后台 |
| `BOOKING_OVERDUE` | 逾期15分钟 | POS、商户后台 |

## 六、API 接口清单

> 以下路径以后台总册 `11-前台业务端API详细设计.md` 为唯一真相源，前台文档仅作引用。

| 接口 | 方法 | 路径 | 说明 |

| 到店确认（门店） | POST | `/api/v1/bookings/{bookingId}/arrive` | — |
| 预订列表（门店） | GET | `/api/v1/bookings` | 支持日期/状态筛选 |
| 餐段配置 | GET/PUT | `/api/v1/booking/periods/{periodId}` | 门店后台管理 |
