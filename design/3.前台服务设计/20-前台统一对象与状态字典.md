# 前台统一对象与状态字典

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

> **说明：** 本文档是前台所有终端的唯一状态/对象基准。各终端文档中的状态枚举、对象定义均以本文档为准，不得自行扩展或重命名。

---

## 一、核心对象词典

### 1.1 SaleOrder（销售订单）

顾客的一次购买行为，是前台最核心的业务对象。

| 字段 | 类型 | 说明 |
|------|------|------|
| orderId | string | 系统订单ID |
| orderNo | string | 展示用订单号（如 #20240405001） |
| storeId | string | 门店ID |
| tableSessionId | string? | 所属桌台会话（堂食时有值） |
| orderScene | enum | DINE_IN / TAKEAWAY_SELF / DELIVERY_ONESELF / DELIVERY_PLATFORM |
| status | enum | 见订单状态机 |
| items | OrderItem[] | 订单行 |
| surcharges | Surcharge[] | 附加费明细 |
| couponId | string? | 使用的优惠券 |
| totalAmount | number | 商品小计 |
| surchargeAmount | number | 附加费合计 |
| discountAmount | number | 优惠金额 |
| payableAmount | number | 实付金额 |
| clientOrderId | string | 前端幂等键 |
| createdAt | string | 创建时间 |

**OrderItem（订单行）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| orderItemId | string | 订单行ID |
| productId | string | 商品ID |
| skuId | string | SKU ID |
| name | string | 商品名称快照 |
| specDesc | string | 规格描述快照（如"中份/微辣"） |
| unitPrice | number | 单价快照 |
| quantity | number | 数量 |
| serveStatus | enum | PENDING_SERVE / SERVED |
| refundStatus | enum | NONE / PARTIAL_REFUNDING / PARTIALLY_REFUNDED / FULL_REFUNDING / FULLY_REFUNDED |

## 二、状态字典

### 2.1 订单状态机（SaleOrder.status）

```
PENDING_PAYMENT（待支付）
    │ 支付成功
    ▼
PAID（已支付/待接单）
    │ 门店接单
    ▼
ACCEPTED（已接单）
    │ 开始制作
    ▼
IN_PRODUCTION（制作中）
    │ 全部工单完成制作
    ▼
PRODUCTION_DONE（制作完成）
    │ 配送到达/上菜完成/顾客取餐
    ▼
FULFILLED（已履约）
    │ 顾客确认/系统自动确认
    ▼
COMPLETED（已完成）

任意状态 ──取消──► CANCELLED
部分退款或全额退款均不改变主状态；退款情况通过 `refundStatus` 字段反映。
> 注意：全额退款完成后，主状态仍为 `CANCELLED`，前端需通过 `refundStatus == FULLY_REFUNDED` 判断
> 是否展示"已退款"标签，而非将 FULLY_REFUNDED 作为主状态。
```

| 状态 | 顾客端展示 | POS 展示 | 可执行操作（顾客） | 可执行操作（POS） |
|------|-----------|---------|-----------------|-----------------|
| PENDING_PAYMENT | 待支付 | 待支付 | 支付、取消 | 收款、取消 |
| PAID | 待接单 | 待接单 | 取消申请 | 接单、拒单 |
| ACCEPTED | 已接单 | 已接单 | 催单 | 加菜、取消 |
| IN_PRODUCTION | 制作中 | 制作中 | 催单 | 加菜 |
| PRODUCTION_DONE | 待配送/待取餐/待上菜 | 制作完成 | — | — |
| FULFILLED | 已送达/已上菜 | 已履约 | 确认收货 | — |
| COMPLETED | 已完成 | 已完成 | 再来一单、售后 | 退款 |
| CANCELLED | 已取消（含全额退款场景） | 已取消 | 再来一单 | — |

---

### 2.2 退款状态（独立于订单主状态）

| 状态 | 说明 | 前端展示 |
|------|------|----------|
| NONE | 无退款 | — |
| PARTIAL_REFUNDING | 部分退款处理中 | "部分退款处理中…" |
| PARTIALLY_REFUNDED | 已部分退款 | "已部分退款 ¥X" |
| FULL_REFUNDING | 全额退款处理中 | "退款处理中…" |
| FULLY_REFUNDED | 已全额退款 | "已全额退款" |
| REFUND_FAILED | 退款失败 | "退款失败，请联系客服" |

---

### 2.3 桌台状态（Table.status）

> 以租户与组织域（`07-租户与组织域.md`）为 canonical 真相源，前台只做展示映射，不重新定义状态名。

| 状态（canonical） | 颜色 | 说明 | 可执行操作 |
|------|------|------|-----------|
| AVAILABLE | 绿色 | 空闲可用 | 开台点单 |
| OCCUPIED | 红色 | 就餐中（有进行中桌台会话） | 查看订单、加菜、结账、转台、并台 |
| CLEANING | 橙色 | 订单已完成，清台中 | 确认清台（完成后变为 AVAILABLE） |
| RESERVED | 蓝色 | 已被预订 | 查看预订 |
| DISABLED | 灰色 | 停用 | — |

---

### 2.4 桌台会话状态机（TableSession.status）

```
OPEN（开台）
    │ 所有订单完成/结账
    ▼
PENDING_CLEAR（待清台）
    │ 服务员确认清台
    ▼
CLOSED（已关闭）

任意状态 ──转台──► 原会话 CLOSED，目标桌台新建/合并 OPEN
```

---

### 2.5 预订状态机（BookingOrder.status）

```
PENDING（待确认）
    │ 门店确认
    ▼
CONFIRMED（已确认）
    │ 顾客到店（POS 操作"到店确认"，系统建立 TableSession）
    ▼
ARRIVED（已到店 / 已入座）
    │ 完成就餐
    ▼
COMPLETED（已完成）

PENDING/CONFIRMED ──取消──► CANCELLED
CONFIRMED ──逾期未到店──► OVERDUE（系统自动，到店时间+缓冲后触发）
```

---

### 2.6 工单状态机（WorkUnit.status）

```
PENDING（待制作）
    │ 厨师开始制作
    ▼
IN_PROGRESS（制作中）
    │ 厨师完成制作
    ▼
DONE（已完成）

任意状态 ──退菜/取消──► RECALLED（已撤回）
```

---

### 2.7 打印任务状态（PrintTask.status）

| 状态 | 说明 |
|------|------|
| PENDING | 待打印 |
| PRINTING | 打印中 |
| SUCCESS | 打印成功 |
| FAILED | 打印失败（进入失败队列） |

## 三、消息类型字典

所有 WebSocket 消息类型的完整清单，各终端只允许使用此表中定义的类型名称。

| 消息类型 | 接收终端 | 触发时机 |
|----------|----------|----------|
| `ORDER_NEW` | POS | 新订单支付完成 |
| `ORDER_STATUS_CHANGED` | POS、顾客App | 订单状态变更 |
| `ORDER_URGE` | POS、KDS | 顾客催单 |
| `REFUND_STATUS_CHANGED` | POS、顾客App | 退款状态变更 |
| `WORK_UNIT_NEW` | KDS | 新工单推送 |
| `WORK_UNIT_CANCELLED` | KDS | 工单撤销 |
| `WORK_UNIT_URGE` | KDS | 工单催单 |
| `PACK_UNIT_READY` | 打包台 | 厨房完成，可打包 |
| `RIDER_STATUS_CHANGED` | 打包台 | 骑手状态变更 |
| `PICKUP_CALL` | 叫号屏 | 叫号触发 |
| `TABLE_STATUS_CHANGED` | PDA、POS | 桌台状态变更 |
| `MENU_INVALIDATED` | 所有终端 | 菜单发布 |
| `PRINTER_STATUS_CHANGED` | POS | 打印机状态变更 |
| `PRINT_TASK_FAILED` | POS | 打印任务失败 |
| `INVENTORY_ALERT` | POS | 库存预警 |
| `BOOKING_NEW` | POS | 顾客创建新预订 |
| `BOOKING_CONFIRMED` | 顾客App | 门店确认预订 |
| `BOOKING_REJECTED` | 顾客App | 门店拒绝预订 |
| `BOOKING_CANCELLED` | POS / 顾客App | 任意方取消预订 |
| `BOOKING_REMINDER` | 顾客App | 预订前2小时提醒 |
| `BOOKING_ARRIVED` | POS、商户后台 | 顾客到店确认 |
| `BOOKING_OVERDUE` | POS | 预订逾期未到店（系统自动触发） |
| `CHANNEL_PUSH_FAILED` | POS | 渠道推单失败 |
| `DEVICE_OFFLINE` | POS | 设备离线超阈值 |

## 四、错误码字典

前台所有终端统一使用以下错误码，来源：`final-后台设计-claude/11-前台业务端API详细设计.md`。

| 错误码 | 含义 | 各终端处理方式 |
|--------|------|---------------|
| 10001 | 参数无效 | 展示具体字段错误 |
| 10002 | 未授权（Token 失效） | 跳转登录页 |
| 10003 | 权限不足 | 展示"无权限"提示 |
| 10004 | 资源不存在 | 展示"不存在"提示 |
| 10005 | 服务不可用 | 展示"系统繁忙"，上报错误 |
| 20001 | 订单不存在 | 提示"订单不存在" |
| 20002 | 订单状态无效 | 展示具体原因 |
| 20003 | 订单不可取消 | 展示不可取消原因 |
| 20004 | 订单金额异常 | 提示"价格已变动，请重新下单" |
| 21001 | 支付失败 | 展示失败原因，提供重试 |
| 21002 | 支付超时 | 展示超时提示，提供手动查询 |
| 21003 | 退款失败 | 展示失败原因，联系客服 |
| 22001 | 商品不存在 | 提示"商品已下架" |
| 22002 | 商品不可售 | 标记不可售 |
| 22003 | 商品售罄 | 标记购物车中售罄商品 |
| 23001 | 门店不存在 | 提示"门店不存在" |
| 23002 | 门店未营业 | 提示"门店暂未营业" |
| 23003 | 门店已停用 | 提示"门店已停用" |

## 五、前台对象到后台 DTO 映射

> 本节说明前台各终端提交请求时，前端字段与后台 DTO 字段的对应关系，以消除命名歧义。

### 5.1 创建订单（POST /api/v1/orders）

| 前台字段 | 后台 DTO 字段 | 类型 | 说明 |
|----------|--------------|------|------|
| `channelType` | `channelType` | enum | `MINI_PROGRAM` / `H5` / `POS_DIRECT` / `KIOSK` / `MEITUAN` / `ELEME` / `DOUYIN` |
| `orderScene` | `orderScene` | enum | `DINE_IN` / `TAKEAWAY_SELF` / `DELIVERY_ONESELF` / `DELIVERY_PLATFORM` |
| `fulfillmentType` | `fulfillmentType` | enum | `TABLE_SERVICE` / `SELF_PICKUP` / `THIRD_PARTY_DELIVERY` / `PLATFORM_DELIVERY` / `STORE_DELIVERY` |
| `tableCode` | `tableCode` | string | 堂食必填 |
| `sessionId` | `tableSessionId` | string | 堂食加菜时必填，关联桌台会话 |
| `items[].productId` | `items[].productId` | string | — |
| `items[].skuId` | `items[].skuId` | string | — |
| `items[].quantity` | `items[].quantity` | number | — |
| `items[].specSnapshot` | `items[].specSnapshot` | object | 规格快照，防止后续改价影响历史订单 |
| `items[].unitPrice` | `items[].unitPrice` | number | 前台传入，后台校验 |
| `couponId` | `couponId` | string | 可选 |
| `remark` | `remark` | string | 整单备注 |
| `clientOrderId` | `Idempotency-Key`（请求头） | string | UUID，幂等键 |

### 5.2 创建预订（POST /api/v1/bookings）

| 前台字段 | 后台 DTO 字段 | 类型 | 说明 |
|----------|--------------|------|------|
| `storeId` | `storeId` | string | — |

| `partySize` | `partySize` | number | 就餐人数 |
| `customerName` | `customerName` | string | — |
| `customerPhone` | `customerPhone` | string | 脱敏存储 |
| `remark` | `remark` | string | 可选 |
| `clientBookingId` | `Idempotency-Key`（请求头） | string | UUID，幂等键 |

### 5.3 桌台会话（TableSession）关键字段

| 前台字段 | 后台字段 | 说明 |
|----------|----------|------|

### 5.4 WebSocket 消息（WsMessage）
