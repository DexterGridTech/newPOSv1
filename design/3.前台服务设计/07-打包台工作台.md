> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

---

## 一、产品定位与边界

### 1.1 定位

打包台工作台是**外卖和自取订单的出餐核验终端**，确保每单出餐准确，并驱动后续叫号/配送流程。

核心职责：
- 接收厨房完成的外卖/自取工单
- 核验打包完整性（扫码或手动确认）
- 触发自取叫号
- 触发外卖骑手通知
- 补打标签/小票

### 1.2 使用角色

| 角色 | 主要操作 |
|------|----------|
| 打包员 | 核验打包、确认出餐 |
| 收银员 | 兼任打包（小店场景） |

### 1.3 技术形态

- React Web，Android 平板（10-13 寸），横屏
- 硬件：扫码枪（核验取餐码）、标签打印机

## 三、核心流程

### 3.1 工单到达

厨房完成工单后，打包台自动收到推送：
- 新卡片插入列表顶部，高亮 3 秒
- 播放提示音
- 外卖订单显示骑手状态（待派单/已到店）

### 3.2 扫码核验

打包员用扫码枪扫描取餐码（顾客小程序上的二维码）：

```
扫码枪输入取餐码
  → 前端解析取餐码，提取 orderId
  → 高亮对应订单卡片
  → 弹出确认框：
    "订单 #0023
     红烧肉×1  白米饭×2
     [确认打包完成]"
  → 确认后调用接口
```

### 3.3 手动确认打包

不使用扫码时，直接点击卡片上的"确认打包"：

```
点击"确认打包"
  → 弹出确认框（展示订单商品清单）
  → 确认
  → POST /api/v1/fulfillment-orders/{fulfillmentOrderId}/pack-complete
  → 自取订单：自动触发叫号
  → 外卖订单：通知骑手可取餐
  → 卡片移入"已完成"区域
```

### 3.4 打印标签

每个外卖袋需贴打包标签，标签内容：

```
┌─────────────────────┐
│ [门店名]             │
│ 订单：#0023          │
│ 美团外卖             │
│ ─────────────────── │
│ 红烧肉（中份/微辣）  │
│ 白米饭 × 2          │
│ ─────────────────── │
│ 备注：少放盐         │
│ 打包时间：12:40      │
└─────────────────────┘
```

```
POST /api/v1/print-tasks
{
  "type": "PACK_LABEL",
  "orderId": "order_001",
  "printerId": "printer_pack_001"
}
```

### 3.5 骑手状态展示

外卖订单卡片显示骑手实时状态（来自三方平台推送）：

| 骑手状态 | 展示文字 | 卡片颜色 |
|----------|----------|----------|
| 待派单 | 骑手：待派单 | 正常 |
| 已接单 | 骑手：前往取餐（预计 X 分钟） | 正常 |
| 已到店 | 骑手：已到店等待 | 橙色高亮 |
| 已取走 | 骑手：已取走 | 灰色 |

骑手已到店时，打包台播放提示音，提醒打包员尽快出餐。

## 四、API 接口清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 待打包工单列表 | GET | `/api/v1/fulfillment-orders` | `status=PRODUCTION_DONE` |
| 确认打包完成 | POST | `/api/v1/fulfillment-orders/{id}/pack-complete` | — |
| 打印标签 | POST | `/api/v1/print-tasks` | — |
| 补打小票 | POST | `/api/v1/print-tasks` | `type=REPRINT` |

**WebSocket 订阅：**
- `/topic/store.{storeId}.pack` — 新打包工单推送
- `/topic/store.{storeId}.rider` — 骑手状态变更推送
