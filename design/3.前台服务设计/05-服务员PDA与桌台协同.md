# 服务员 PDA 与桌台协同终端详细设计

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

---

## 一、产品定位与边界

### 1.1 定位

服务员 PDA 是**服务员随身携带的移动操作终端**，让服务员在桌边完成点单、加菜、桌台管理等操作，无需往返收银台。

核心价值：
- 桌边点单，减少顾客等待
- 实时掌握桌台状态，提升翻台效率
- 上菜确认，确保出餐准确

### 1.2 使用角色

| 角色 | 主要操作 |
|------|----------|
| 服务员 | 点单、加菜、桌台管理、上菜确认 |
| 店长 | 全部服务员操作 + 查看全局桌台状态 |

### 1.3 PDA 与 POS 操作权限边界

| 操作 | PDA（服务员） | POS（收银员） | 说明 |
|------|--------------|--------------|------|
| 桌边点单/加菜 | ✓ | ✓ | PDA 更便捷 |
| 收款/支付 | — | ✓ | 收款只在 POS 完成 |
| 扫码收款 | — | ✓ | — |
| 现金收款 | — | ✓ | — |
| 退款 | — | ✓（需权限） | PDA 无退款入口 |
| 取消订单 | — | ✓ | — |
| 上菜确认 | ✓ | — | PDA 专属 |
| 桌台开台 | ✓ | ✓ | — |
| 桌台清台 | ✓ | ✓ | — |
| 转台/并台 | ✓ | ✓ | — |
| 催单（通知厨房） | ✓ | ✓ | — |
| 查看本店订单列表 | ✓（只读） | ✓ | PDA 不可改单 |
| 接单/拒单（外卖） | — | ✓ | 外卖接单在 POS |
| 班次管理 | — | ✓ | — |
| 打印小票 | — | ✓ | — |

> 核心原则：**PDA 负责"跑厅"操作（点单、上菜、桌台），POS 负责"收银"操作（收款、退款、班次）**。两者通过 WebSocket 实时同步桌台和订单状态。

### 1.4 技术形态

- React Native（Android），运行在专用 PDA 设备上
- 屏幕尺寸：5-6 寸，竖屏为主
- 网络：门店 WiFi，需覆盖全场
- 硬件：内置扫码头（扫桌台码、商品码）

## 三、核心页面详细设计

### 3.1 首页（桌台总览）

**布局（竖屏）：**
```
┌─────────────────────┐
│ [门店] [张三] [12:30]│
├─────────────────────┤
│ [大厅▼]  [地图|列表] │
├─────────────────────┤
│ ┌──────┐ ┌──────┐  │
│ │ A01  │ │ A02  │  │
│ │ 空闲 │ │就餐中│  │
│ │      │ │ 4人  │  │
│ └──────┘ └──────┘  │
│ ┌──────┐ ┌──────┐  │
│ │ A03  │ │ A04  │  │
│ │就餐中│ │待清台│  │
│ │ 2人  │ │      │  │
│ └──────┘ └──────┘  │
├─────────────────────┤
│ [扫码开台]  [刷新]   │
└─────────────────────┘
```

**桌台状态颜色与 POS 端一致**（见文档 04 第六节）。

**扫码开台：**
- 点击"扫码开台"，调起摄像头扫描桌台二维码
- 扫码成功后，自动识别桌台号，进入点单页

### 3.2 点单页

PDA 点单页为竖屏单列布局，与 POS 三栏布局不同：

```
┌─────────────────────┐
│ ← A区05号桌  [购物车🛒3]│
├─────────────────────┤
│ [热销][招牌][主食][饮品]│  ← 横向滚动分类
├─────────────────────┤
│ ┌───────────────┐   │
│ │[图] 红烧肉     │   │
│ │ ¥38  月售88   │   │
│ │ 微辣/中份      │   │
│ │           [+] │   │
│ └───────────────┘   │
│ ┌───────────────┐   │
│ │[图] 白米饭     │   │
│ │ ¥3            │   │
│ │           [+] │   │
│ └───────────────┘   │
├─────────────────────┤
│ 已选3件  ¥44  [提交] │
└─────────────────────┘
```

**与 POS 点单的差异：**
- 无挂单功能（PDA 是单任务）
- 无现金收款（PDA 不连接钱箱）
- 支付方式：顾客扫码支付（展示收款二维码）或记账（后续在 POS 结账）
- 提交后自动返回桌台总览

### 3.3 订单详情页

服务员查看桌台订单，确认上菜：

```
┌─────────────────────┐
│ ← A区05号桌订单      │
├─────────────────────┤
│ 订单 #20240405001   │
│ 状态：制作中         │
├─────────────────────┤
│ 待上菜              │
│ ┌───────────────┐   │
│ │ 红烧肉（中/微辣）│  │
│ │ × 1    [已上菜]│   │
│ └───────────────┘   │
│ ┌───────────────┐   │
│ │ 白米饭         │   │
│ │ × 2    [已上菜]│   │
│ └───────────────┘   │
├─────────────────────┤
│ 已上菜              │
│ 可乐 × 2  ✓        │
├─────────────────────┤
│ [加菜]  [催单]  [结账]│
└─────────────────────┘
```

**上菜确认逻辑：**

> **重要边界：** 服务员"上菜确认"是**交付动作**，不是"完成制作"动作。
> - KDS：`完成制作` → 触发 `WorkUnitCompleted` 事件（厨房职责）
> - 打包台：`确认打包` → 触发 `PackCompleted` 事件（打包职责）
> - PDA：`确认上桌` → 触发 `OrderItemServed` 事件（服务员职责）

三段动作相互独立，PDA 不得调用 `work-units/complete` 接口。

**逐项上菜：**
```
服务员点击某道菜的"已上菜"
  → POST /api/v1/orders/{orderId}/items/{itemId}/served
    { "staffId": "staff_001" }
  → 该菜品标记为 SERVED（划线显示）
  → 记录上菜时间和操作员
```

**整单上菜完成：**
```
所有菜品均标记 SERVED
  → 订单状态自动推进为 FULFILLED
  → 顾客小程序收到"菜品已上齐"通知
```

**部分上菜场景：**
- 允许部分菜品先上（如凉菜先上，热菜后上）
- 未上菜的菜品保持 PENDING_SERVE 状态
- 订单状态保持 IN_PRODUCTION 直到全部上菜

**误上菜/上错桌补救：**
```
服务员发现上错桌
  → 点击已上菜菜品旁的"撤回"
  → POST /api/v1/orders/{orderId}/items/{itemId}/unserved
    { "staffId": "staff_001", "reason": "上错桌" }
  → 菜品状态回到 PENDING_SERVE
  → 记录撤回操作到审计日志
  → 服务员重新上菜到正确桌台后再次确认
```

### 3.4 桌台操作

长按桌台卡片或进入桌台详情后，可执行：

**转台：**
```
选择"转台"
  → 展示空闲桌台列表
  → 选择目标桌台
  → 确认转台
  → POST /api/v1/table-sessions/{sessionId}/transfer
    { "targetTableCode": "A06" }
  → 原桌台变为"待清台"，目标桌台变为"就餐中"
```

**并台：**
```
选择"并台"
  → 展示其他就餐中桌台
  → 选择要合并的桌台
  → 确认并台（两桌订单合并到主桌）
  → POST /api/v1/table-sessions/{sessionId}/merge
    { "mergeSessionId": "session_002" }
```

**清台：**
```
选择"清台"
  → 确认桌台已清理完毕
  → POST /api/v1/table-sessions/{sessionId}/close
  → 桌台状态变为"空闲"
```

## 四、实时推送处理

PDA 通过 WebSocket 接收推送，处理以下事件：

| 事件 | 处理方式 |
|------|----------|
| `ORDER_NEW` | 桌台卡片状态更新，震动提醒 |
| `ORDER_STATUS_CHANGED` | 更新对应桌台/订单状态 |
| `WORK_UNIT_STATUS_CHANGED` | 更新订单详情页的上菜状态 |
| `TABLE_STATUS_CHANGED` | 刷新桌台地图 |

**断线处理：**
- WiFi 信号弱时，顶部显示"网络不稳定"提示条
- 断线后自动重连，重连成功后拉取增量数据
- 操作失败时展示 Toast，不自动重试

## 五、离线能力

PDA 在弱网环境下的降级策略：

| 功能 | 离线支持 | 说明 |
|------|----------|------|
| 查看桌台状态 | 展示缓存数据（最后同步时间） | 不保证实时 |
| 点单 | 不支持 | 需要网络提交 |
| 上菜确认 | 本地队列，网络恢复后同步 | 允许离线操作 |
| 查看订单详情 | 展示缓存数据 | — |
