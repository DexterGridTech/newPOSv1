## 一、角色体系

### 1.1 角色分层

系统角色分为三层：**平台层**、**商户层**、**顾客层**。

```
平台层
├── 平台超级管理员      // 平台最高权限，管理所有项目/商户
├── 平台运营           // 日常运营，查看数据，处理投诉
└── 平台财务           // 结算对账，资金管理

商户层
├── 品牌管理员         // 管理品牌下所有门店，配置品牌商品库
├── 门店店长           // 管理单门店，查看报表，处理异常
├── 收银员             // POS 收银，接单，退款
├── 服务员             // PDA 点单，上菜确认，桌台管理
├── 厨师/厨房员工      // KDS 查看工单，标记完成
├── 打包员             // 打包台核验，出单
└── 门店运营           // 查看报表，管理菜单（只读）

顾客层
└── 顾客（匿名/会员）  // 点餐，支付，查看订单
```

| 角色 | 菜单管理 | 订单查看 | 订单操作 | 退款 | 报表 | 系统配置 |

| 平台超管 | ✓ | ✓（全平台） | ✓ | ✓ | ✓ | ✓ |
| 平台运营 | — | ✓（全平台） | 部分 | — | ✓ | — |
| 平台财务 | — | ✓（结算视图） | — | ✓ | ✓ | — |
| 品牌管理员 | ✓（品牌库） | ✓（品牌内） | ✓ | ✓ | ✓ | 部分 |
| 门店店长 | ✓（门店） | ✓（本店） | ✓ | ✓ | ✓ | 部分 |
| 收银员 | — | ✓（本店） | ✓（收银相关） | ✓（限额内） | — | — |
| 服务员 | — | ✓（本店） | ✓（点单/上菜） | — | — | — |
| 厨师 | — | ✓（工单视图） | ✓（标记完成） | — | — | — |
| 打包员 | — | ✓（打包视图） | ✓（核验出单） | — | — | — |
| 顾客 | — | ✓（本人） | ✓（下单/取消） | ✓（申请） | — | — |

详细权限矩阵见文档 13。

## 二、终端清单

### 2.1 顾客侧终端

#### 顾客小程序（微信/支付宝）
- **定位**：购物中心统一入口，覆盖所有餐饮商户
- **核心功能**：浏览商户、点餐、在线支付、订单跟踪、会员积分、优惠券
- **技术形态**：微信小程序 + 支付宝小程序（uni-app 双端）
- **网络要求**：4G/WiFi，支持弱网降级
- **认证方式**：微信/支付宝 OAuth，匿名浏览

#### 堂食扫码点餐 H5
- **定位**：桌台扫码后的轻量点餐入口，嵌入微信/支付宝内置浏览器
- **核心功能**：菜单浏览、加购、提交、支付（微信/支付宝 JSAPI）
- **技术形态**：Vue 3 H5，移动端响应式
- **特殊要求**：必须在 500ms 内完成首屏渲染（顾客扫码后的第一印象）
- **认证方式**：扫码携带桌台 token，自动识别桌台和门店

#### 自助点餐机
- **定位**：门店落地式自助终端，减少排队
- **核心功能**：菜单浏览、加购、支付（扫码/刷脸）、打印取餐号
- **技术形态**：React + Electron，Android Kiosk 模式
- **硬件依赖**：触摸屏（≥21寸）、扫码枪、热敏打印机、摄像头（刷脸可选）
- **认证方式**：设备级 Token，无需员工登录

#### 取餐叫号屏
- **定位**：展示型终端，展示叫号信息
- **核心功能**：叫号展示、语音播报、广告轮播
- **技术形态**：React Web，全屏展示，Android 浏览器
- **硬件依赖**：大屏显示器（≥43寸）、音响
- **认证方式**：设备级 Token，只读权限

### 2.2 门店执行侧终端

#### POS 收银工作台
- **定位**：门店核心操作终端，收银员/店长主要工作界面
- **核心功能**：
  - 点单（堂食/外带/自取）
  - 接单/拒单（三方外卖、在线订单）
  - 支付（扫码、现金、刷卡、会员余额）
  - 改单（加菜、退菜）
  - 退款
  - 桌台管理
  - 班次管理
- **技术形态**：React Web（推荐）或 React + Electron
- **硬件依赖**：触摸屏（15-21寸）、扫码枪、钱箱、小票打印机、副屏（顾客展示）
- **认证方式**：设备 Token + 操作员 Token（员工刷卡/扫码登录）

#### 服务员 PDA
- **定位**：移动点单和桌台管理，服务员随身携带
- **核心功能**：
  - 移动点单（代顾客点餐）
  - 加菜/退菜
  - 桌台状态管理（开台、并台、转台）
  - 上菜确认
  - 催单
- **技术形态**：React Native 或 uni-app，Android 手持设备
- **硬件依赖**：Android 手持 PDA（带扫码头）
- **认证方式**：设备 Token + 操作员 Token

#### KDS 厨房显示系统
- **定位**：厨房工作站专用展示终端，替代纸质出单
- **核心功能**：
  - 工单展示（按工作站分屏）
  - 工单优先级排序（催单高亮）
  - 工单完成确认
  - 工单撤销（改单/取消时）
- **技术形态**：React Web，Android 浏览器全屏
- **硬件依赖**：防水触摸屏（15-21寸），厨房环境
- **认证方式**：设备 Token，工作站级别（不需要员工登录）

#### 打包台工作台
- **定位**：外卖/自取订单的打包核验终端
- **核心功能**：
  - 待打包订单列表
  - 扫码核验（扫取餐码确认订单）
  - 打包完成确认
  - 触发叫号（自取订单）
  - 触发打印（补打标签）
- **技术形态**：React Web，Android 平板
- **硬件依赖**：Android 平板（10-13寸）、扫码枪
- **认证方式**：设备 Token + 操作员 Token

#### 打印机（受控设备）
- **定位**：不是独立终端，由其他终端控制
- **类型**：
  - 小票打印机（80mm 热敏，POS 出单、顾客小票）
  - 厨房打印机（80mm 热敏，厨房工单）
  - 标签打印机（外卖打包标签）
- **接入方式**：通过打印服务（本地 Agent 或云打印）接入，见文档 08

### 2.3 管理侧终端

#### 商户后台 Web
- **定位**：商户日常运营管理界面
- **核心功能**：
  - 菜单管理（商品、分类、规格、价格）
  - 订单管理（查询、导出）
  - 报表（营业额、商品销量、退款统计）
  - 门店配置（营业时间、桌台、打印机）
  - 员工管理（账号、权限）
  - 渠道管理（美团/饿了么菜单同步）
- **技术形态**：React + Ant Design Pro，PC 浏览器
- **认证方式**：账号密码 + 短信验证码

#### 平台运营后台 Web
- **定位**：平台级治理和运营界面
- **核心功能**：
  - 商户管理（入驻、审核、停用）
  - 全平台订单监控
  - 结算对账管理
  - 营销活动管理
  - 系统配置
- **技术形态**：React + Ant Design Pro，PC 浏览器
- **认证方式**：账号密码 + MFA

## 四、业态终端组合模式

不同规模和业态的门店，终端配置不同。

### 4.1 小型快餐（1-2 收银台）

```
必配：POS × 1，小票打印机 × 1，厨房打印机 × 1
可选：叫号屏 × 1，自助机 × 1
不需要：KDS（用打印机替代），PDA，打包台
```

### 4.2 中型正餐（3-5 收银台，有桌台服务）

```
必配：POS × 2-3，PDA × 2-4，KDS × 1-2，小票打印机 × 2
可选：叫号屏 × 1，打包台 × 1（如有外卖）
```

### 4.3 大型餐厅（5+ 收银台，多工作站）

```
必配：POS × 3+，PDA × 5+，KDS × 3+（按工作站分屏），打包台 × 1-2
可选：自助机 × 2-4，叫号屏 × 2，标签打印机 × 1
```

### 4.4 纯外卖门店

```
必配：POS × 1（接单用），打包台 × 1，标签打印机 × 1
不需要：KDS（用打印机），PDA，叫号屏，自助机
```

### 4.5 美食广场档口

```
必配：POS × 1，小票打印机 × 1
可选：自助机 × 1，叫号屏（共用广场叫号屏）
不需要：KDS，PDA，打包台
```

## 五、终端间协作关系

```
顾客小程序/H5/自助机
        │ 下单/支付
        ▼
   后台交易域
        │ 订单创建事件
        ▼
   POS（接单确认）────────────────────────────────┐
        │ 接单后                                   │
        ▼                                          │
   后台履约域（生产计划）                          │
        │                                          │
        ├──── 工单推送 ──► KDS（厨房制作）         │
        │                    │ 完成确认             │
        │                    ▼                     │
        ├──── 工单推送 ──► 打包台（核验打包）       │
        │                    │ 打包完成             │
        │                    ▼                     │
        │              叫号屏（叫号）               │
        │                                          │
        └──── 状态推送 ──► PDA（上菜确认）◄────────┘
                                │
                                ▼
                          订单完成 → 结算
```
