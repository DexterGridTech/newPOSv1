# 取餐叫号屏详细设计

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

---

## 一、产品定位与边界

### 1.1 定位

取餐叫号屏是**面向顾客的展示型终端**，展示自取订单的叫号信息，引导顾客取餐。

核心职责：
- 展示"制作中"和"可取餐"的订单号
- 语音播报叫号
- 广告/营销内容轮播（空闲时）

### 1.2 技术形态

- React Web，Android 浏览器全屏
- 屏幕：43 寸以上大屏，横屏，安装在取餐区
- 无需员工操作，纯展示终端
- 设备级 Token，只读权限

## 二、界面设计

### 2.1 主界面布局

```
┌──────────────────────────────────────────────────────────────┐
│                    [门店名称]  取餐叫号                       │
├─────────────────────────┬────────────────────────────────────┤
│                         │                                    │
│      制作中              │         请取餐                     │
│                         │                                    │
│   018  023  031         │    ★ 015  ★ 019  ★ 022           │
│   035  041  048         │      027  033                      │
│                         │                                    │
│   （灰色小字）           │   （绿色大字，★=新叫号）           │
│                         │                                    │
├─────────────────────────┴────────────────────────────────────┤
│  [广告轮播区域]  当前优惠：红烧肉套餐 ¥48，限时特惠！         │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 叫号动效

新叫号到达时：
- 号码以动画形式从"制作中"区域飞入"请取餐"区域
- 新叫号显示 ★ 标记，持续 30 秒后消除
- 语音播报："请 015 号顾客取餐"（TTS 合成）

### 2.3 号码展示规则

| 区域 | 内容 | 最多显示 | 字体大小 |
|------|------|----------|----------|
| 制作中 | 正在制作的取餐号 | 12 个 | 中 |
| 请取餐 | 已完成可取餐的号 | 8 个 | 大 |

超出数量时，最早的号码滚动消失。

## 三、叫号触发机制

叫号由打包台确认打包完成后自动触发：

```
打包台确认打包完成
  → POST /api/v1/fulfillment-orders/{id}/pack-complete
  → 后台触发 PickupCallCreated 事件
  → WebSocket 推送到叫号屏
  → 叫号屏更新展示 + 语音播报
```

**手动叫号（补叫）：**
- POS 端可手动触发补叫（顾客未听到时）
- `POST /api/v1/pickup-calls/{callId}/replay`

---

## 四、语音播报

- 使用设备本地 TTS（Android TextToSpeech API）
- 播报文案：`"请 {取餐号} 号顾客到取餐台取餐"`
- 新叫号立即播报，多个叫号排队播报（不并发）
- 音量可在设备设置中调整

---

## 五、广告轮播配置管理

广告轮播区域在叫号屏底部展示，由商户后台配置。

### 5.1 广告素材类型

| 类型 | 格式 | 尺寸建议 | 说明 |
|------|------|----------|------|
| 图片 | JPG/PNG | 1920×200px | 静态横幅 |
| 视频 | MP4（H.264） | ≤30s，≤10MB | 循环播放 |
| 文字公告 | 纯文本 | ≤50字 | 滚动字幕 |

### 5.2 轮播配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 每张展示时长 | 8s（图片）/ 视频时长 | 视频播完后切换 |
| 切换动效 | 淡入淡出 | — |
| 空内容时 | 展示门店名称 + 当前时间 | 兜底展示 |
| 生效时间段 | 全天（可配置时段） | 如仅午餐时段展示午市优惠 |

### 5.3 配置接口

```
GET /api/v1/stores/{storeId}/ad-banners?scene=PICKUP_SCREEN
```

返回有效期内、按排序字段排列的广告列表。叫号屏启动时拉取一次，之后每小时刷新（广告内容变化不频繁，无需 WebSocket 推送）。

---

## 六、API 接口清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 当前叫号列表 | GET | `/api/v1/pickup-calls/current` | 初始化加载 |
| 补叫 | POST | `/api/v1/pickup-calls/{callId}/replay` | POS 触发 |

**WebSocket 订阅：**
- `/topic/store.{storeId}.pickup-call` — 新叫号推送
