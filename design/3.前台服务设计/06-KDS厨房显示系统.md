# KDS 厨房显示系统详细设计

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

---

## 一、产品定位与边界

### 1.1 定位

KDS（Kitchen Display System）是**厨房工作站的专用展示与操作终端**，替代传统纸质出单，实现数字化厨房管理。

核心价值：
- 实时接收工单，无需等待打印
- 按工作站分屏展示，各岗位只看自己的工单
- 催单高亮，优先级可视化
- 工单完成确认，驱动履约状态流转

### 1.2 工作站模型

一个门店可配置多个工作站，每个 KDS 绑定一个或多个工作站：

| 工作站类型 | 示例 | 负责菜品 |
|------------|------|----------|
| 热菜区 | 炒锅1、炒锅2 | 所有热菜 |
| 凉菜区 | 凉菜台 | 凉菜、沙拉 |
| 主食区 | 主食台 | 米饭、面条、包子 |
| 饮品区 | 饮品台 | 饮料、甜品 |
| 打包区 | 打包台 | 外卖打包（见文档07） |

工作站配置在商户后台完成，每个商品归属一个工作站。

### 1.3 技术形态

- React Web，Android 浏览器全屏（Kiosk 模式）
- 屏幕尺寸：15-21 寸触摸屏，横屏
- 网络：门店有线网络（优先）或 WiFi
- 无需员工登录，设备级 Token

## 二、工单数据模型

```typescript
interface WorkUnit {
  workUnitId: string;
  orderId: string;
  orderNo: string;           // 展示用订单号（末4位）
  stationId: string;         // 工作站 ID
  scene: 'DINE_IN' | 'DELIVERY' | 'TAKEOUT';
  tableLabel?: string;       // 堂食：桌台标签 "A05"
  channelLabel?: string;     // 外卖：渠道 "美团"
  pickupNo?: string;         // 自取：取餐号 "018"
  items: WorkUnitItem[];
  priority: 'NORMAL' | 'URGENT'; // URGENT = 催单
  status: 'PENDING' | 'IN_PROGRESS' | 'DONE' | 'CANCELLED';
  createdAt: string;         // 工单创建时间
  acceptedAt?: string;       // 开始制作时间
  doneAt?: string;           // 完成时间
  estimatedDoneAt?: string;  // 预计完成时间
  urgeCount: number;         // 催单次数
}

interface WorkUnitItem {
  itemId: string;
  productName: string;
  specDesc: string;          // "中份/微辣"
  quantity: number;
  remark?: string;           // 单品备注
  status: 'PENDING' | 'DONE';
}
```

### 3.2 工单卡片设计

**卡片状态与样式：**

| 状态 | 边框颜色 | 背景 | 说明 |
|------|----------|------|------|
| PENDING（待制作） | 灰色 | 白色 | 新到工单 |
| IN_PROGRESS（制作中） | 蓝色 | 浅蓝 | 已开始制作 |
| URGENT（催单） | 红色闪烁 | 浅红 | 顾客催单 |
| 超时（等待 > 阈值） | 橙色 | 浅橙 | 超出预计时间 |

**等待时间计算：**
- 从工单创建时间到当前时间
- 超过门店配置的预警阈值（默认 15 分钟）时，变为橙色
- 超过严重阈值（默认 25 分钟）时，变为红色

**卡片操作按钮：**
- `PENDING` 状态：显示"开始制作"按钮
- `IN_PROGRESS` 状态：显示"完成"按钮（绿色，大按钮）
- 支持整单完成（点击"完成"）和逐项完成（点击每道菜旁的勾选）

### 3.3 工单排序规则

工单默认按以下优先级排序：
1. 催单（URGENT）优先
2. 等待时间长的优先（FIFO）
3. 外卖订单略微提前（可配置）

排序规则可在商户后台配置。

## 四、操作流程

### 4.1 开始制作

```
厨师点击"开始制作"
  → 卡片状态变为 IN_PROGRESS（蓝色）
  → POST /api/v1/work-units/{workUnitId}/start
  → 记录开始时间，用于计算实际制作时长
```

### 4.2 完成工单

**整单完成：**
```
厨师点击"完成"
  → 卡片从看板移除（动画滑出）
  → POST /api/v1/work-units/{workUnitId}/complete
  → 后台触发履约域事件：WorkUnitCompleted
  → 如该订单所有工单均完成，触发 ProductionDone 事件
  → 打包台/叫号屏收到推送
```

**逐项完成（可选模式）：**
```
厨师点击单道菜旁的"✓"
  → 该菜品标记为完成（划线显示）
  → 所有菜品完成后，整单自动完成
  → POST /api/v1/work-units/{workUnitId}/items/{itemId}/complete
```

### 4.3 工单撤销

当订单被取消或退菜时，后台推送撤销事件：
- 工单卡片顶部显示红色"已撤销"标签
- 3 秒后卡片自动从看板移除
- 厨师无需操作

### 4.4 催单处理

顾客在小程序催单后：
- 对应工单卡片边框变为红色闪烁
- 播放催单提示音（与新工单提示音不同）
- 卡片顶部显示"催单 × N"（N 为催单次数）
- 厨师完成工单后，催单标记自动消除

## 五、多屏分工模式

大型厨房可配置多台 KDS，每台绑定不同工作站：

```
门店配置：
  KDS-1（热菜区）→ 工作站：炒锅1、炒锅2
  KDS-2（凉菜区）→ 工作站：凉菜台
  KDS-3（主食区）→ 工作站：主食台、饮品台

工单分发规则：
  订单中的菜品按工作站拆分为多个工单
  每个工单只推送到对应工作站的 KDS
  所有工单完成后，订单状态才推进
```

**KDS 设备注册：**
```
POST /api/v1/devices/register
{
  "deviceType": "KDS",
  "storeId": "store_001",
  "stationIds": ["station_001", "station_002"],
  "deviceName": "热菜区KDS"
}
```

## 六、显示配置

商户可在后台配置 KDS 显示参数：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 每行卡片数 | 3 | 根据屏幕尺寸调整 |
| 预警时间（分钟） | 15 | 超时变橙色 |
| 严重超时（分钟） | 25 | 超时变红色 |
| 提示音 | 开启 | 新工单/催单提示音 |
| 自动完成 | 关闭 | 超时后自动标记完成 |
| 字体大小 | 中 | 小/中/大 |

## 七、API 接口清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 工单列表 | GET | `/api/v1/work-units` | 按工作站筛选 |
| 开始制作 | POST | `/api/v1/work-units/{id}/start` | — |
| 完成工单 | POST | `/api/v1/work-units/{id}/complete` | — |
| 完成单项 | POST | `/api/v1/work-units/{id}/items/{itemId}/complete` | — |
| 设备注册 | POST | `/api/v1/devices/register` | — |
| KDS 配置 | GET | `/api/v1/devices/{deviceId}/config` | — |

**WebSocket 订阅：**
- `/topic/station.{stationId}.workunit` — 工作站工单推送
- `/topic/station.{stationId}.cancel` — 工单撤销推送
- `/topic/station.{stationId}.urge` — 催单推送

## 八、断线降级策略

> 完整降级规则见文档 00 §9.5，本节说明 KDS 端的具体行为。

**断线时（KDS 本地）：**
- 顶部显示红色警告条："网络断开，工单可能不完整，已切换为打印机出单"
- 已显示的工单仍可正常操作（完成、开始制作）
- 新工单不再推送到此 KDS，由后台自动切换为厨房打印机出单

**重连后（KDS 本地）：**
1. 警告条消失
2. 调用增量接口补显断线期间工单：
   ```
   GET /api/v1/work-units?stationId={stationId}&since={lastEventTimestamp}&status=PENDING,IN_PROGRESS
   ```
3. 补显工单卡片顶部显示"补显"灰色标签，提示厨师核查
4. 后台自动切回 KDS 推送模式
