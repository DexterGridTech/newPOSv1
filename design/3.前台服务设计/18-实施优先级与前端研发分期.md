# 实施优先级与前端研发分期

> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

---

## 一、分期原则

- **P0 期（MVP）**：能跑通最核心的堂食点单→收银→出品→完成闭环，可上线试运营
- **P1 期（增强）**：补齐外卖、预订、附加费、异常治理等经营增强能力
- **P2 期（完善）**：渠道同步、自助点餐机、高级报表、平台治理等完善能力

---

## 二、P0 期：核心经营闭环

**目标：** 堂食扫码点餐 + POS 收银 + KDS 出品 + 基础后台可用。

### 2.1 前台终端

| 终端 | P0 必须交付的能力 |
|------|-----------------|
| 顾客小程序 | 商户列表、菜单浏览、堂食扫码点单、支付、订单详情、订单状态跟踪 |
| 堂食 H5 | 扫码建立桌台会话、菜单浏览、点单、支付 |
| POS 收银 | 点单、收款（微信/支付宝/现金）、订单管理、外卖接单、桌台管理、班次管理 |
| KDS | 工单接收、开始制作、完成制作、催单响应 |
| 打包台 | 打包任务接收、确认打包、叫号触发 |
| 叫号屏 | 叫号展示、自动轮播 |

### 2.2 后台管理

| 后台 | P0 必须交付的能力 |
|------|-----------------|
| 商户后台 | 商品管理、菜单发布、订单查询、营业时间配置、桌台管理、员工管理 |
| 平台后台 | 商户入驻审核、全平台订单监控、基础数据大盘 |

### 2.3 基础设施

- WebSocket 实时推送（订单、工单、桌台状态）
- 打印服务（小票、厨房单）
- 微信支付/支付宝接入
- JWT 认证与权限控制

### 2.4 P0 验收标准

- [ ] 顾客扫码 → 点单 → 支付 → KDS 出单 → 服务员上菜 → 完成，全链路跑通
- [ ] POS 人工点单 → 收款 → 出单，全链路跑通
- [ ] 外卖接单 → 接单 → 制作 → 打包 → 叫号，全链路跑通
- [ ] 商户后台可发布菜单，各终端实时刷新

---

## 三、P1 期：经营增强

**目标：** 补齐预订、附加费、同桌多单、异常治理等能力，支持正式商业运营。

### 3.1 新增能力

| 模块 | 交付内容 |
|------|----------|
| 预订链路 | 顾客小程序预订、订金支付、POS 预订管理、到店确认、桌台自动衔接 |
| 附加费 | 商户后台附加费规则配置、顾客端展示、POS 收款页展示与减免 |
| 同桌多单 | 堂食加菜（增量订单）、桌台汇总账单、部分退款 |
| 服务员 PDA | 桌边点单、上菜确认、转台/并台/清台 |
| 异常治理 | 商户后台异常记录、打印失败队列、改单审计 |
| 渠道治理 | 平台后台渠道推单失败补偿重放、Token 到期预警 |
| 营业报表 | 商户后台营业概览、商品销量、数据导出 |

### 3.2 P1 验收标准

- [ ] 预订全链路：顾客预订 → 门店确认 → 顾客到店 → 衔接点餐 → 订金抵扣，跑通
- [ ] 附加费：门店配置服务费 → 顾客端展示 → POS 收款扣除/减免，跑通
- [ ] 加菜：首单支付后加菜 → 差价支付 → 厨房收到加菜工单，跑通
- [ ] 打印失败：打印机离线 → 失败队列记录 → 重打，跑通

---

## 四、P2 期：平台完善

**目标：** 渠道同步、自助点餐机、高级治理、平台营销等完善能力。

### 4.1 新增能力

| 模块 | 交付内容 |
|------|----------|
| 渠道菜单同步 | 美团/饿了么菜单同步、差异对比、渠道价格策略 |
| 自助点餐机 | Kiosk 点单、支付、自取码打印 |
| 平台营销 | 跨商户优惠券、积分活动、满减活动 |
| 会员体系 | 会员等级、积分兑换、储值 |
| 设备大盘 | 平台后台设备在线状态监控 |
| 预订治理 | 平台后台爽约率统计、强制介入 |
| 高级报表 | 复购率、新客数、渠道分布分析 |

---

## 五、研发排期建议

```
P0 期（约 8-10 周）
  Week 1-2：基础设施（认证、WebSocket、打印服务）
  Week 3-4：顾客小程序 + 堂食 H5
  Week 5-6：POS 收银 + KDS + 打包台
  Week 7-8：商户后台基础版 + 叫号屏
  Week 9-10：联调、压测、上线准备

P1 期（约 6-8 周）
  Week 1-2：预订链路（顾客端 + POS 端）
  Week 3-4：附加费 + 同桌多单 + PDA
  Week 5-6：异常治理 + 渠道治理 + 报表
  Week 7-8：联调、验收

P2 期（约 6-8 周）
  Week 1-3：渠道同步 + 自助点餐机
  Week 4-5：平台营销 + 会员体系
  Week 6-8：设备大盘 + 高级报表 + 联调
```

---

## 六、跨期依赖说明

| 依赖关系 | 说明 |
|----------|------|
| P1 预订依赖 P0 桌台会话 | 到店确认后需创建 TableSession |
| P1 附加费依赖 P0 支付链路 | 附加费在支付时扣除 |
| P1 PDA 依赖 P0 WebSocket | PDA 通过 WS 接收桌台/订单状态 |
| P2 渠道同步依赖 P0 菜单管理 | 需要先有内部菜单才能同步 |
| P2 营销依赖 P0 订单链路 | 优惠券在下单时核销 |
