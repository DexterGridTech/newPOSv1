## 一、单终端功能验收

### 1.1 顾客小程序

| 场景 | 步骤 | 预期结果 |
|------|------|----------|
| 堂食扫码点单 | 扫桌台码 → 选菜 → 提交 → 微信支付 | 支付成功，订单状态变为 PAID，KDS 收到工单 |
| 外卖下单 | 选外卖 → 填地址 → 下单 → 支付 | 支付成功，POS 收到新订单提醒 |
| 自取下单 | 选自取 → 下单 → 支付 | 支付成功，生成取餐号 |
| 商品售罄 | 点击已售罄商品 | 不可加购，显示"售罄"标签 |
| 附加费展示 | 门店配置10%服务费，顾客下单 | 确认页显示服务费明细 |
| 堂食加菜 | 首单支付后点"继续加菜" | 进入加菜模式，提交后创建增量订单 |
| 取消订单 | 待支付状态点取消 | 订单取消，无需退款 |
| 取消已支付订单 | 已支付未接单点取消 | 订单取消，自动退款 |
| 催单 | 制作中状态点催单 | POS/KDS 收到催单提醒 |
| 预订流程 | 选餐段 → 填信息 → 支付订金 | 预订单创建，状态 PENDING，POS 收到提醒 |
| 取消预订（24h前） | 已确认预订点取消 | 预订取消，订金退还 |

### 1.2 POS 收银

| 场景 | 步骤 | 预期结果 |
|------|------|----------|
| 人工点单收款 | 选菜 → 微信扫码收款 | 支付成功，打印小票，KDS 出单 |
| 现金收款 | 选菜 → 现金收款 → 输入金额 | 显示找零，打开钱箱，打印小票 |
| 外卖接单 | 新外卖订单到达 → 点接单 | 订单状态变 ACCEPTED，KDS 收到工单 |
| 外卖拒单 | 新外卖订单到达 → 点拒单 → 选原因 | 订单状态变 REJECTED，通知三方平台 |
| 加菜（收差价） | 就餐中桌台点加菜 → 选菜 → 收取差价 | 创建加菜单，发起支付 |
| 免费加菜 | 就餐中桌台点加菜 → 免费加菜 | 需店长授权，授权后创建金额为0的加菜单 |
| 退菜 | 订单详情点退菜 → 选商品 | 退款原路退回，订单主状态不变，refundStatus 变为 PARTIALLY_REFUNDED |
| 附加费减免 | 收款页点减免服务费 | 需店长授权，授权后服务费移除 |
| 预订到店 | 预订列表点"到店" → 分配桌台 | 创建 TableSession，跳转点单页，订金记为待抵扣 |
| 转台 | 就餐中桌台点转台 → 选目标桌台 | 原桌台变待清台，目标桌台变就餐中 |
| 交班 | 点交班 → 确认 | 生成交班单，打印，班次关闭 |

### 1.3 KDS

| 场景 | 步骤 | 预期结果 |
|------|------|----------|
| 接收新工单 | 顾客下单支付 | KDS 显示新工单，播放提示音 |
| 完成制作 | 点击工单"完成" | 工单移出队列，触发打包/叫号流程 |
| 退菜撤回 | POS 退菜后 | KDS 对应工单行显示撤回标记 |
| 整单取消 | POS 取消订单 | KDS 对应工单整体撤销 |
| 催单响应 | 顾客催单 | 工单卡片显示催单标记，提示音加强 |

### 1.4 服务员 PDA

| 场景 | 步骤 | 预期结果 |
|------|------|----------|
| 扫码开台 | 扫桌台二维码 | 识别桌台号，进入点单页 |
| 桌边点单 | 选菜 → 提交 | 订单创建，POS 同步显示 |
| 上菜确认 | 订单详情点"已上菜" | 菜品标记 SERVED，记录上菜时间 |
| 误上菜撤回 | 点已上菜菜品旁"撤回" | 菜品状态回到 PENDING_SERVE |
| 清台 | 待清台桌台点清台 | 桌台状态变空闲 |

## 二、跨终端联调场景

### 2.1 堂食全链路

```
顾客扫码 → 小程序点单支付
  → POS 收到 ORDER_NEW 推送
  → KDS 收到 WORK_UNIT_NEW 推送
  → KDS 完成制作
  → 打包台收到 PACK_UNIT_READY（外卖/自取）
    或 PDA 收到上菜任务（堂食）
  → PDA 确认上菜
  → 订单状态变 FULFILLED
  → 顾客小程序状态更新
```

验收检查点：
- [ ] 各终端状态同步延迟 < 2s
- [ ] 任一终端断线重连后状态正确恢复
- [ ] 打印小票内容与订单一致

### 2.2 预订到就餐全链路

```
顾客小程序预订 → 支付订金
  → POS 收到 BOOKING_NEW 推送
  → 门店确认预订
  → 顾客收到 BOOKING_CONFIRMED 通知
  → 顾客到店，POS 点"到店"
  → 自动创建 TableSession
  → 正常点餐流程
  → 结账时订金自动抵扣
```

验收检查点：
- [ ] 订金抵扣金额正确
- [ ] 预订状态机流转正确（PENDING→CONFIRMED→ARRIVED→COMPLETED）

### 2.3 退款全链路

```
POS 发起退款（需店长授权）
  → 后台处理退款
  → 订单主状态不变，refundStatus 变为 FULL_REFUNDING → FULLY_REFUNDED
  → 顾客小程序收到退款状态推送（WS 消息类型：REFUND_STATUS_CHANGED）
  → 微信/支付宝退款到账
  → 顾客收到退款到账通知
```

验收检查点：
- [ ] 退款金额与订单金额一致
- [ ] 退款中间态（FULL_REFUNDING）期间禁止重复发起
- [ ] 退款失败时展示失败原因

## 三、异常场景演练

| 异常场景 | 触发方式 | 预期处理 |
|----------|----------|----------|
| 支付超时 | 下单后不支付，等待15分钟 | 订单自动关闭，库存释放 |
| 支付结果未知 | 支付后断网 | 轮询查询，最终确认状态 |
| KDS 断线 | 拔掉 KDS 网线 | POS 显示设备离线告警，工单积压 |
| 打印机离线 | 关闭打印机 | 打印失败队列记录，POS 提示重打 |
| 渠道推单失败 | 模拟美团接口超时 | 自动重试3次，失败后进入异常中心 |
| WebSocket 断线 | 断网30秒后恢复 | 自动重连，拉取断线期间增量数据 |
| 商品下架 | 后台下架商品，顾客购物车有该商品 | 提交订单时提示"商品已下架" |
| 桌台已被占用 | 两个顾客同时扫同一桌台码 | 第二个顾客提示"该桌台已有订单，是否加入" |

## 四、性能验收基准

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 菜单页首屏加载 | < 2s（4G网络） | Lighthouse 测试 |
| 下单接口响应 | < 500ms（P99） | 压测工具 |
| WebSocket 消息延迟 | < 1s | 端到端时间戳对比 |
| KDS 工单到达延迟 | < 2s（支付后） | 端到端测试 |
| POS 并发接单 | 50单/分钟不丢单 | 压测 |
| 菜单缓存命中率 | > 90% | 监控统计 |
