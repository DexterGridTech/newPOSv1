> 版本：1.1 | 状态：正式稿 | 日期：2026-04-05

---

## 一、产品定位与边界

### 1.1 定位

平台运营后台是**购物中心平台方的管理界面**，供平台超管、平台运营、平台财务使用。

核心职责：
- 商户入驻审核与管理
- 全平台订单监控与干预
- 结算对账管理
- 营销活动管理（跨商户优惠券、积分活动）
- 系统配置（费率、规则）
- 数据大盘

### 1.2 与商户后台的边界

| 能力 | 商户后台 | 平台后台 |
|------|----------|----------|
| 菜单管理 | ✓（本店） | 只读查看 |
| 订单管理 | ✓（本店） | ✓（全平台） |
| 退款 | ✓（本店，限额） | ✓（全平台，无限额） |
| 结算对账 | 查看本店账单 | 管理全平台结算 |
| 商户管理 | — | ✓ |
| 营销活动 | 本店活动 | 跨商户活动 |

### 3.2 商户管理

**商户列表字段：**

| 字段 | 说明 |
|------|------|
| 商户名称 | 品牌名 |
| 门店数量 | 该商户下的门店数 |
| 入驻时间 | — |
| 状态 | 正常/暂停/注销 |
| 今日GMV | — |
| 合同到期 | — |
| 操作 | 查看详情、暂停、配置费率 |

**入驻审核流程：**
```
商户提交入驻申请
  → 平台运营审核资质材料
  → 审核通过：创建商户账号，配置费率
  → 审核拒绝：填写拒绝原因，通知商户
```

**商户详情页包含：**
- 基本信息（营业执照、联系人）
- 门店列表
- 合同信息
- 费率配置
- 历史操作记录

### 3.3 全平台订单监控

**异常订单类型：**

| 异常类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 超时未接单 | 支付后 15 分钟未接单 | 平台介入，强制接单或退款 |
| 支付状态异常 | 支付回调异常，状态不一致 | 手动核查，修正状态 |
| 退款超时 | 退款申请 24 小时未处理 | 平台代为退款 |
| 投诉订单 | 顾客投诉 | 查看详情，协调处理 |

**平台介入操作：**
```typescript
// 平台强制退款（绕过商户审批）
POST /api/v1/platform/orders/{orderId}/force-refund
{
  "reason": "商户超时未处理",
  "operatorId": "platform_staff_001"
}
```

### 3.4 结算管理

**结算周期：** 默认 T+1（每日结算），可配置为周结/月结。

**结算账单列表：**

| 字段 | 说明 |
|------|------|
| 结算周期 | 2026-04-04 |
| 商户名称 | — |
| 交易金额 | 该周期内总交易额 |
| 退款金额 | — |
| 平台服务费 | 按费率扣除 |
| 应结金额 | 交易额 - 退款 - 服务费 |
| 状态 | 待结算/已结算/异常 |

**对账流程：**
```
系统自动生成结算账单（每日凌晨）
  → 商户确认账单（可提出异议）
  → 平台审核（有异议时）
  → 打款（银行转账/余额）
  → 标记已结算
```

### 3.5 营销管理

**跨商户优惠券：**
```typescript
interface PlatformCoupon {
  couponId: string;
  name: string;              // "购物中心美食节满50减10"
  type: 'CASH' | 'DISCOUNT';
  value: number;
  minOrderAmount: number;
  scope: {
    type: 'ALL' | 'CATEGORY' | 'STORES';
    storeIds?: string[];     // 指定商户
    categories?: string[];   // 指定品类
  };
  budget: number;            // 活动总预算
  perUserLimit: number;      // 每人限领数量
  totalLimit: number;        // 总发放数量
  validFrom: string;
  validTo: string;
  distributeChannel: 'MANUAL' | 'AUTO_REGISTER' | 'ACTIVITY';
}
```

**营销活动生命周期：**

```
草稿（DRAFT）
    │ 提交审核
    ▼
审核中（REVIEWING）
    │ 审核通过          审核拒绝
    ▼                    ▼
待生效（SCHEDULED）   已拒绝（REJECTED）→ 可修改后重新提交
    │ 到达 validFrom
    ▼
进行中（ACTIVE）
    │ 到达 validTo 或手动终止
    ▼
已结束（ENDED）
```

**各状态操作权限：**

| 状态 | 可执行操作 | 说明 |
|------|-----------|------|
| DRAFT | 编辑、提交审核、删除 | — |
| REVIEWING | 撤回（回到草稿） | 审核中不可编辑 |
| SCHEDULED | 查看、提前终止 | 不可修改内容 |
| ACTIVE | 查看实时数据、提前终止 | 进行中不可修改规则 |
| ENDED | 查看报告、复制为新活动 | 只读 |

**活动实时数据看板（ACTIVE 状态）：**
- 已发放券数 / 总限额
- 已核销券数 / 核销率
- 活动带来的 GMV
- 预算消耗进度（已用 / 总预算）
- 参与商户数 / 订单数

### 3.6 异常中心

#### 3.6.1 全平台异常记录

汇聚所有门店的异常事件，供平台运营排查和干预。异常记录对象结构见文档 `15-异常场景与前台补救机制.md §5`（`ExceptionRecord`）。

| exceptionType | 触发条件 | 严重级别 | 处理方式 |
|---------------|----------|----------|----------|
| `PAYMENT_UNKNOWN` | 支付状态不一致超过 5 分钟 | P0 | 手动核查，修正状态 |
| `REFUND_FAILED` | 退款请求被支付渠道拒绝 | P1 | 平台代为处理 |
| `CHANNEL_PUSH_FAILED` | 美团/饿了么订单接收异常 | P1 | 触发补偿重放 |
| `ORDER_TIMEOUT_UNACCEPTED` | 支付后 15 分钟未接单 | P1 | 平台介入强制接单或退款 |
| `DEVICE_OFFLINE` | KDS/PDA 离线超过 30 分钟 | P2 | 通知商户 |
| `CHANNEL_TOKEN_EXPIRING` | 渠道授权 Token 7 天内到期 | P2 | 自动提醒商户刷新 |

**异常记录列表视图：**
```
┌──────────────────────────────────────────────────────────────┐
│ 异常中心  [P0(2)] [P1(8)] [P2(15)]  [全部] [未处理]          │
├──────────────────────────────────────────────────────────────┤
│ 级别  时间     商户      异常类型        状态    操作          │
│ P0   12:30   XX火锅   支付回调异常    未处理  [处理] [详情]   │
│ P1   12:28   XX快餐   渠道推单失败    处理中  [重放] [详情]   │
│ P1   12:15   XX饮品   退款失败        未处理  [处理] [详情]   │
└──────────────────────────────────────────────────────────────┘
```

#### 3.6.2 渠道推单失败与补偿重放

三方外卖订单（美团/饿了么）推送到门店失败时，平台提供补偿重放机制：

```
渠道推单失败
  → 记录到异常中心（类型：渠道推单失败）
  → 自动重试 3 次（间隔 1min/3min/5min）
  → 3 次均失败 → 标记为"需人工介入"
  → 平台运营点击 [重放]
    → POST /api/v1/platform/channel-orders/{orderId}/replay
    → 重新推送到门店 POS
    → 记录重放操作到审计日志
```

**重放操作约束：**
- 仅限状态为 `PUSH_FAILED` 的渠道订单
- 重放前检查订单是否已超时（超时则直接退款，不重放）
- 每个订单最多人工重放 3 次，超出需升级处理

### 3.7 预订治理

#### 3.7.1 全平台预订列表

平台视角查看所有商户的预订数据：

| 筛选维度 | 说明 |
|----------|------|
| 日期范围 | 默认今日 |
| 商户/门店 | 下拉选择 |
| 预订状态 | PENDING/CONFIRMED/ARRIVED/OVERDUE 等 |
| 是否有订金 | 有订金/无订金 |

**预订治理操作（平台介入）：**
- 强制取消预订（商户无法处理时）
- 强制退还订金（商户拒绝退款时）
- 查看预订详情和操作历史

### 3.8 渠道授权管理

管理美团/饿了么的 ISV 授权：

```
┌──────────────────────────────────────────────────────────────┐
│ 美团外卖授权管理                                              │
├──────────────────────────────────────────────────────────────┤
│ 商户名称    授权状态   Token到期时间   操作                   │
│ XX火锅      ✅ 有效    2026-07-01     [刷新Token] [撤销]     │
│ XX快餐      ⚠️ 即将到期 2026-04-10   [刷新Token]            │
│ XX饮品      ❌ 已失效  2026-03-01    [重新授权]              │
└──────────────────────────────────────────────────────────────┘
```

Token 即将到期（7天内）时，系统自动发送提醒给商户。

**Token到期预警配置：**

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 预警提前天数 | 7 天 | 到期前几天开始提醒 |
| 提醒频率 | 每日一次 | 避免骚扰 |
| 提醒渠道 | 短信 + 平台站内信 | 商户管理员接收 |
| 自动刷新 | 支持（需商户预授权） | 部分渠道支持静默刷新 |

平台运营可在"系统配置 → Token到期预警"中查看所有即将到期的授权，并批量触发刷新通知。

## 四、操作日志

所有平台后台操作均记录审计日志：

```typescript
interface AuditLog {
  logId: string;
  operatorId: string;
  operatorName: string;
  action: string;            // "FORCE_REFUND"
  targetType: string;        // "ORDER"
  targetId: string;          // orderId
  before: Record<string, any>; // 操作前状态
  after: Record<string, any>;  // 操作后状态
  reason: string;
  ip: string;
  createdAt: string;
}
```

操作日志不可删除，保留 3 年。
