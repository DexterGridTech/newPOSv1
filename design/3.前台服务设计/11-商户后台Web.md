> 版本：1.0 | 状态：正式稿 | 日期：2026-04-05

## 一、产品定位与边界

### 1.1 定位

商户后台是**商户日常运营管理的 PC 端界面**，供店长、品牌管理员、门店运营使用。

核心职责：
- 菜单管理（商品、分类、规格、价格）
- 订单查询与导出
- 营业数据报表
- 门店配置（营业时间、桌台、打印机、设备）
- 员工账号与权限管理
- 渠道管理（美团/饿了么菜单同步）

### 1.2 技术形态

- React + Ant Design Pro，PC 浏览器（Chrome 90+）
- 响应式布局，最低支持 1280×768
- 认证：账号密码 + 短信验证码

### 3.2 商品管理

**商品列表页：**

| 字段 | 说明 |
|------|------|
| 商品图片 | 缩略图 |
| 商品名称 | 可点击进入编辑 |
| 分类 | 所属分类 |
| 价格 | 基础售价 |
| 状态 | 上架/下架/售罄 |
| 渠道 | 已同步的渠道图标 |
| 操作 | 编辑、上下架、删除、复制 |

**商品编辑页（关键字段）：**

```typescript
interface ProductForm {
  name: string;                    // 商品名称
  categoryId: string;              // 分类
  description: string;             // 描述
  images: string[];                // 图片（最多5张）
  basePrice: number;               // 基础价格
  originalPrice?: number;          // 划线价
  unit: string;                    // 单位（份/个/杯）
  specs: SpecGroup[];              // 规格组
  tags: string[];                  // 标签（招牌/新品/辣）
  availableScenes: Scene[];        // 可售场景（堂食/外卖/自取）
  availableTimeRanges: TimeRange[]; // 可售时段
  stockControl: {
    enabled: boolean;
    dailyLimit?: number;           // 每日限量
    currentStock?: number;         // 当前库存
  };
  channelPrices: {                 // 渠道差价
    channelType: string;
    price: number;
  }[];
}
```

**规格管理：**
- 规格组（如"辣度"）和规格选项（如"微辣"、"中辣"）分开管理
- 规格选项可设置加价
- 规格组可设置是否必选、是否多选

### 3.3 菜单发布

菜单发布是将商品组织成可售菜单的操作：

```
菜单发布流程：
1. 编辑菜单（拖拽排序分类和商品）
2. 预览菜单（模拟顾客视角）
3. 选择生效时间（立即/定时）
4. 发布确认
5. 发布成功（各终端自动刷新菜单）
```

**菜单版本管理：**
- 每次发布生成新版本
- 保留最近 10 个版本
- 可回滚到历史版本

### 3.4 渠道菜单同步

将门店菜单同步到美团/饿了么：

```
同步流程：
1. 查看同步状态（上次同步时间、差异项）
2. 映射商品（内部商品 ↔ 平台商品）
3. 配置渠道价格（可与门店价格不同）
4. 执行同步
5. 查看同步结果（成功/失败项）
```

**差异对比视图：**
```
┌──────────────────────────────────────────────────────────────┐
│ 美团外卖菜单同步  上次同步：2026-04-04 18:00                 │
├──────────────────────────────────────────────────────────────┤
│ 状态    商品名称      本地价格  美团价格  操作               │
│ ✅ 已同步 红烧肉      ¥38      ¥40      [编辑]              │
│ ⚠️ 价格差异 白米饭   ¥3       ¥4       [同步] [忽略]       │
│ ❌ 未同步 新品：可乐  ¥8       —        [同步]              │
│ 🗑 已下架 旧品：汤    —        ¥12      [下架]              │
└──────────────────────────────────────────────────────────────┘
```

**商品自动映射算法：**

首次同步时，系统尝试自动匹配内部商品与渠道平台商品，减少人工操作：

```
自动映射规则（按优先级）：
1. 精确匹配：商品名称完全一致 → 自动映射，置信度 100%
2. 模糊匹配：名称相似度 ≥ 85%（编辑距离算法）→ 建议映射，需人工确认
3. 条码匹配：内部商品有 SKU 条码，与平台商品条码一致 → 自动映射
4. 无匹配：标记为"未映射"，需人工处理
```

映射结果分三类展示：
- **自动映射（绿色）**：直接进入同步队列
- **建议映射（黄色）**：展示匹配理由，商户一键确认或手动调整
- **未映射（红色）**：必须人工处理后才能同步

**渠道价格策略：**

| 策略 | 说明 |
|------|------|
| 与门店同价 | 默认，渠道价格跟随门店价格自动更新 |
| 固定溢价 | 渠道价格 = 门店价格 × 系数（如 1.1，覆盖平台佣金） |
| 手动定价 | 每个商品单独设置渠道价格，不随门店价格联动 |

### 3.5 订单管理

**订单列表筛选条件：**
- 日期范围（默认今日）
- 订单状态
- 就餐方式（堂食/外卖/自取）
- 渠道（本平台/美团/饿了么）
- 订单号/手机号搜索

**订单详情操作（后台视角）：**
- 查看完整订单信息（含操作历史）
- 手动退款（需权限）
- 导出订单（CSV/Excel）

### 3.6 营业时间配置

```typescript
interface BusinessHours {
  regularHours: {
    dayOfWeek: number;    // 0=周日，1=周一...
    open: string;         // "10:00"
    close: string;        // "22:00"
    enabled: boolean;
  }[];
  specialDates: {         // 特殊日期（节假日）
    date: string;         // "2026-02-01"
    open?: string;        // 不填=全天关闭
    close?: string;
    label: string;        // "春节"
  }[];
  lastOrderOffset: number; // 最后下单时间（关店前N分钟）
}
```

### 3.7 桌台管理

**桌台配置：**
- 新增/编辑/删除桌台
- 设置桌台区域（大厅/包厢/露台）
- 设置最大就餐人数
- 生成/更新桌台二维码
- 批量导出二维码（用于打印贴桌）

**桌台二维码内容：**
```
https://h5.platform.com/menu?storeId={storeId}&tableCode={tableCode}&scene=DINE_IN
```

### 3.8 员工管理

**员工账号：**
- 手机号作为登录账号
- 分配角色（收银员/服务员/厨师/打包员/店长）
- 设置有效期（临时员工）
- 重置密码

**角色权限配置：**
- 基于预设角色，不支持自定义权限（避免配置错误）
- 详细权限矩阵见文档 13

## 四、报表设计

### 4.1 营业概览报表

支持按日/周/月/自定义时间段查看：

| 指标 | 说明 |
|------|------|
| 营业额 | 实收金额（扣除退款） |
| 订单数 | 完成订单数 |
| 客单价 | 营业额 / 订单数 |
| 退款金额 | 退款总额 |
| 退款率 | 退款订单数 / 总订单数 |
| 新客数 | 首次下单顾客数 |
| 复购率 | 有2次以上订单的顾客占比 |

### 4.2 数据导出

- 订单明细：CSV，包含所有字段
- 营业汇总：Excel，按日汇总
- 商品销量：Excel，按商品汇总
- 导出任务异步处理，完成后发送下载链接

## 六、附加费规则管理

商户可在"门店配置 → 附加费规则"中配置：

| 配置项 | 说明 |
|--------|------|
| 费用名称 | 如"服务费"、"包间费" |
| 计算方式 | 按比例（如10%）或固定金额 |
| 适用场景 | 堂食/外卖/自取/全部 |
| 适用时段 | 全天或指定时段 |
| 是否可减免 | 开启后收银员可申请减免（需店长授权） |
| 最低消费触发 | 消费满X元才收取 |

## 七、异常治理

### 7.1 异常记录

展示门店内发生的异常事件，供店长排查。异常记录对象结构见文档 `15-异常场景与前台补救机制.md §5`（`ExceptionRecord`）。

**异常类型枚举（`exceptionType`）：**

| exceptionType | 说明 |
|---------------|------|
| `PAYMENT_UNKNOWN` | 支付结果未知（超时未回调） |
| `REFUND_FAILED` | 退款请求被支付渠道拒绝 |
| `PRINTER_OFFLINE` | 打印机离线或缺纸导致打印失败 |
| `CHANNEL_PUSH_FAILED` | 美团/饿了么订单接收异常 |
| `DEVICE_OFFLINE` | KDS/PDA/打包台设备断线 |

**异常记录列表视图：**
```
┌──────────────────────────────────────────────────────────────┐
│ 异常记录  [全部] [未处理] [需跟进]  [日期筛选]                │
├──────────────────────────────────────────────────────────────┤
│ 时间     异常类型        关联订单   处理人   状态    操作      │
│ 12:30   支付结果未知    #0023      张三     已处理  [详情]    │
│ 12:28   渠道推单失败    #0021      —        未处理  [处理]    │
└──────────────────────────────────────────────────────────────┘
```

**处理操作：** 点击「处理」后填写 `resolution`（处理结果描述）并标记是否需要后续跟进（`requiresFollowUp`），提交后记录 `handledBy` 和 `handledAt`。
